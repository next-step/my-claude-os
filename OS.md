# 📱 내 파이어 투자 앱 (OS)

> **한 줄:** 40대 파이어(FIRE)를 목표로, 내 장기 포트폴리오를 중심으로 굴러가는 **나만의 투자 앱**이다.
> 시판 앱과 달리, 내 목표·내 포폴에 맞는 기능을 내가 필요할 때마다 하나씩 붙여 만든다.

## 🎯 목표 (북극성)

**40대 파이어(FIRE).** 지금의 포트폴리오를 기반으로 한 **장기투자**로 그 시점까지 자산을 키우는 것.
기준 시점은 **지금(26살, 한국 나이)부터 20년 뒤 — 약 46살.** (목표 금액·저축 기록 등 구체 숫자는 [portfolio.md](portfolio.md) / 실제 금액은 비공개 장부)
이 파이어 목표가 **모든 판단의 기준점**이다 — 기간이 길다는 것, 단기 출렁임보다 장기 방향이 중요하다는 것, 리스크를 얼마나 감당할지가 여기서 정해진다.

이 앱으로 키우려는 것:
1. **시장 감각** — 내 보유 종목·섹터에 무슨 일이 벌어지는지 매일 파악한다.
2. **리밸런싱 자립** — 포트폴리오 *구성*은 아빠가 잡아줬지만, **비중 조정과 주기는 내가 스스로 판단**할 수 있게 학습한다.
3. **주식·경제 공부** — 브리핑·Q&A로 만난 용어와 배경을 익혀 투자 판단의 토대를 쌓는다.

> ⚠️ AI는 분석·설명 도우미일 뿐, 투자 권유가 아니다. 판단과 실제 매수/매도는 내가 한다.

## 📱 앱 구조

앱 하나 = 스킬 하나가 아니다. **메뉴 + 부품 + 저장공간 + 기록**이 모여 하나의 앱이 된다.

### 메뉴 (내가 여는 기능 = 스킬)
| 메뉴               | 언제  | 하는 일                                     |
| ---------------- | --- | ---------------------------------------- |
| **`study-loop`** | 매일  | 시장 공부 루프 (브리핑 → Q&A → 정리·퀴즈). **메인 화면.** |
| **`rebalance`**  | 가끔  | 목표비중 대비 갭 진단 + 매매 수량 계산                  |
| **`market-now`** | 수시  | 포폴과 무관하게 시장 전반이 궁금할 때                    |

### 부품 (메뉴 뒤에서 일하는 엔진) — 여러 메뉴가 공유
- **`market-researcher`** (에이전트) — 시장·종목 조사 (출처·날짜·교차검증)
- **`study-tutor`** (에이전트) — 초보 눈높이 개념 설명
- **`rebalance.py`** (스크립트) — 평가액·갭·매매수량 결정론적 계산

### 저장공간 (데이터)
- **`portfolio.md`** — 목표비중 등 설계값 (공개)
- **`portfolio.private.md`** — 보유수량·평단·저축 등 실제 금액 (비공개, git 제외)

### 기록 (로그)
- **`journal/`** — 그날 학습 노트(브리핑 요약·Q&A·퀴즈)를 날짜별로 쌓음

## 🔁 메인 기능: 매일 공부 루프 (study-loop)
```
1. [AI]  시장 배경 → 그 안에서 내 포폴 브리핑
2. [나]  궁금한 것 물어보기  (여러 개면 병렬로 답)
3. [AI]  그날 대화 정리 + 복습 퀴즈 → journal/에 저장   ← 기록·체득
↻  다음 날 또 반복
```
비중 조정(리밸런싱)은 매일 루프에 넣지 않는다 — 가끔만 하는 일이라 `rebalance` 메뉴로 분리(과잉매매 방지).

## 🧭 설계 원칙 (기능 추가할 때 지키는 것)
- **필요할 때 추가.** 미리 다 만들지 않는다. "이게 필요하네" 할 때 메뉴를 붙인다.
- **부품은 재사용.** 같은 일(조사·계산)은 엔진 하나에 두고 메뉴들이 나눠 쓴다. 메뉴끼리 서로 호출하지 않는다.
- **계산은 스크립트, 설명은 AI.** 숫자는 결정론적 스크립트에, 판단·해설은 AI에.
- **민감정보 분리.** 실제 금액은 비공개 파일에만. 계산되는 값(현재비중·평가액)은 저장하지 않고 그때 계산.
- **투자 권유 아님.** 사실·계산·배경만. 실제 매매는 내가.
