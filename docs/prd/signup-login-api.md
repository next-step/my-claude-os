# PRD — 회원가입·로그인 API (FastAPI)

## 문제 (왜 필요한가)
아직 구체적인 서비스가 정해진 것은 아니며, **회원가입·로그인 흐름 자체를 만들어보는 것**이 목적이다.
이메일이 아니라 **아이디/비밀번호** 기반으로 사용자를 식별하는 최소 인증 API를 구축한다.

## 목표 (이 기능이 해결하는 것)
- 아이디/비밀번호/이름으로 신규 사용자가 회원가입할 수 있다.
- 가입한 사용자가 아이디/비밀번호로 로그인하여 JWT를 발급받을 수 있다.
- 사용자 저장·비밀번호 보관·토큰 발급의 최소 기반을 만든다.

## 사용자 시나리오 (기대 동작)
- **회원가입** `POST /signup`: 필수값 **아이디 · 비밀번호 · 이름**을 입력하면 계정이 생성된다.
  - 이미 존재하는 아이디면 가입을 막고 **409 Conflict** + `errorCode: DUPLICATE_USERNAME`.
- **로그인** `POST /login`: **아이디 · 비밀번호**로 로그인하면 **JWT**가 발급된다.
  - 실패 시(아이디 없음/비밀번호 불일치) 보안상 **구분하지 않고 동일 응답**: **401** + `errorCode: INVALID_CREDENTIALS`.

## 범위 밖 (이번에 안 하는 것)
내 정보 조회 · 로그아웃 · 토큰 갱신 · 소셜 로그인 · 비밀번호 찾기 · 이메일 인증 · 속도 제한, 그 외 위에 명시되지 않은 기능.

## 제약 / 조건 (확정)
- **식별자**: 아이디(username) 단일, 이메일 미사용.
  - 길이 **최대 20자**, 공백 불가. 중복 판정은 **앞뒤 공백 제거 후 대소문자 구분**(`Abc`≠`abc`)으로 단순화.
- **인증 방식**: JWT, 서명 알고리즘 **HS256**, payload에는 **user_id만** 담는다.
  - 토큰 만료 **60분**(확정). 시크릿 키는 **환경변수**로 주입(하드코딩 금지).
- **비밀번호 저장**: **SHA-512 + 사용자별 랜덤 솔트**(`secrets`로 생성, DB에 솔트 컬럼 별도 저장).
  - ⚠️ 잔여 리스크(수용됨): SHA-512는 빠른 해시라 솔트가 있어도 대량 대입에 취약. 이번 범위는 그대로 가되 추후 **bcrypt/Argon2** 교체 권장.
- **비밀번호 규칙**: **영문 소문자 + 숫자 포함, 8자 이상.**
- **DB**: SQLite.
- **런타임 버전**: **Python 3.12**(요구사항의 "3.1.2"는 FastAPI 0.111과 비호환이라 호환 버전으로 확정), FastAPI 0.111.1.
- **응답/에러 규격**:
  - 실패 응답은 **`errorCode`** 필드로 정의. 상태코드 매핑: **입력검증 실패 400/422, 로그인 실패 401, 중복 가입 409.**
  - 성공 응답의 상세 바디 스키마는 **설계(Plan) 단계에서 확정.**
- 이메일 인증 없음, 속도 제한 없음.

## 설계(Plan) 단계로 넘길 상세
- 성공 응답 바디 스키마함(필드 구성).
- 솔트 길이·저장 컬럼 형태 등 구현 세부.
