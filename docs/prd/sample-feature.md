# PRD — 사용자 텍스트 메모 (Quick Notes)

## 문제 (왜 필요한가)
- 사용자는 떠오른 짧은 생각이나 할 일을 즉시 기록할 가벼운 수단이 없으면, 외부 도구(메신저, 종이)로 흩어져 기록을 잃어버린다.
- 이 OS 개발 흐름(PRD → Plan → 사람 검토 → 구현)을 처음부터 끝까지 시연하려면, 도메인 복잡도가 낮고 자족적인 "작동하는 기능"이 필요하다.
- 복잡한 외부 의존성(인증, 결제, 외부 API) 없이 입력·저장·조회의 기본 흐름만으로 OS 흐름의 가치를 검증할 수 있는 예시가 필요하다.

## 목표 (이 기능이 해결하는 것)
- 사용자가 한 줄~여러 줄의 텍스트 메모를 빠르게 작성해 저장할 수 있다.
- 저장한 메모를 목록으로 한눈에 확인할 수 있다.
- 더 이상 필요 없는 메모를 삭제할 수 있다.
- 데이터가 한 곳에 모여, 사용자가 기록을 잃지 않는다.
- (부수 목표) OS 개발 흐름을 검증할 수 있을 만큼 단순하면서도, 테스트(단위/통합)로 동작을 보증할 수 있는 범위를 제공한다.

## 사용자 시나리오 (기대 동작)
1. 메모 작성
   - 사용자가 메모 내용을 입력하고 "저장"을 실행하면, 메모가 저장되고 목록 최상단(최신순)에 나타난다.
   - 빈 내용으로 저장을 시도하면 저장되지 않고, 내용이 필요하다는 안내를 받는다.
2. 메모 목록 조회
   - 사용자가 목록을 열면, 저장된 모든 메모가 최신순으로 표시된다.
   - 각 메모에는 내용과 작성 시각이 함께 보인다.
   - 메모가 하나도 없으면 "아직 메모가 없습니다" 같은 빈 상태 안내가 보인다.
3. 메모 삭제
   - 사용자가 특정 메모의 삭제를 실행하면, 해당 메모가 목록에서 사라지고 저장소에서도 제거된다.
4. 영속성
   - 사용자가 앱(또는 화면)을 다시 열어도, 이전에 저장한 메모가 그대로 남아 있다.

## 범위 밖 (이번에 안 하는 것)
- 메모 수정/편집 (작성·조회·삭제만 다룬다)
- 다중 사용자 / 로그인 / 권한 (단일 사용자, 단일 로컬 컨텍스트 가정)
- 검색, 정렬 옵션, 태그/카테고리, 즐겨찾기
- 외부 동기화, 클라우드 백업, 공유
- 첨부파일/이미지, 서식(마크다운 렌더링 등)
- 페이지네이션/무한 스크롤 (초기에는 단순 전체 목록)

## 열린 질문 (사람에게 확인이 필요한 점)
> ⚠️ 각 질문의 "**답변:**" 줄을 사람이 이 파일에 직접 채웁니다.
> (채팅이 아니라 이 md 파일에 적습니다.) 비워두면 ② Plan이 합리적으로 가정합니다.

1. 실행 형태: CLI, 웹, 데스크톱 중 무엇을 가정하나? (저장소·UI 방식이 달라진다)
   - **답변:** CLI로 실행합니다.
2. 저장 방식: 로컬 파일(JSON 등), 인메모리(데모용), SQLite 중 무엇을 쓰나?
   - **답변:** 로컬 파일로 저장합니다.
3. 메모 1건의 길이 제한이 필요한가? (예: 최대 글자 수)
   - **답변:** 100,000자로 합니다.
4. 작성 시각은 어떤 형식/시간대로 보여줄까? (절대 시각 vs 상대 시각)
   - **답변:** 상대 시간으로 보여줍니다. (예: "3분 전")
5. 삭제 시 확인 절차가 필요한가, 아니면 즉시 삭제인가?
   - **답변:** 확인 절차가 필요합니다.
6. 사용할 언어/런타임/프레임워크에 제약이 있나? (기존 OS 코드베이스와의 정합성)
   - **답변:** 일단 Python으로 진행합니다.
