# 온보딩: 나만의 클로드 코드 OS 만들기

> 4주 과정을 시작하기 전, 코스의 배경 지식을 정리한 문서다.
> 출처: 넥스트스텝 "나만의 클로드 코드 OS 만들기" 온보딩 자료.

## 한 줄 요약

이론을 넘어, **4주 동안 "나만의 클로드 코드 OS"를 직접 만든다.**
설계자의 시각으로 OS를 만드는 것이 빠르게 변하는 AI 시대에 적응하는 가장 좋은 방법이라는 전제 위에 설계된 과정이다.

## OS란?

- **이론적 정의**: Operating System(윈도우·리눅스·안드로이드처럼)의 약어.
- **이 과정에서의 정의**: **어떤 목표를 달성하기 위해 AI와 사람이 상호 협력하는 체계(시스템).**
- 쉽게 말하면, 내 문제를 해결하거나 일상을 편하게 만드는 AI 체계. 구조가 단순해도 나만의 OS가 될 수 있다.
  - 예) 매일 내 입맛에 맞게 AI가 뉴스를 선별·코멘트해 배달 = 나만의 '뉴스 OS'

## 공통 골격

OS의 목표는 자유롭게 정할 수 있다(개발·글쓰기·주식 등 무엇이든). 하지만 모든 OS는 아래 골격의 변주다.

```
사람+AI 의도 공유  →  사람+AI 실행  →  사람+AI 피드백
        └──────────────── ↻ 반복 ────────────────┘
```

## OS 예시 (모두 위 골격의 변주)

| 주제 | 흐름 | 반복 단위 |
|------|------|-----------|
| **개발** | 스펙 리뷰 → 태스크 분해 → 승인 → 코딩 → 테스트 → 최종 승인 → 배운 것 기록·활용 | 태스크마다 |
| **글쓰기** | 아이디어 탐색 → 인터뷰로 방향 설정 → 초안 → 피드백 → 학습 후 재초안 → 최종 승인 → 기록·활용 | 글 한 편마다 |
| **주식** | 투자 성향 인터뷰 → 투자 조언 → 투자 진행 → 결과 학습·동향 데이터 확보 | 데이터 확보 후 다시 조언으로 |
| **영어 공부** | 실력 테스트 → 매일 이메일 퀴즈 → 제출 → DB 저장 후 채점·피드백 → 분석 보고서 | 매일 |

> 표에서 **사람**은 의사결정, **AI**는 실행(작성자/분석가/튜터 등) 역할을 맡는다.

## 무엇을 배우고, 무엇을 얻는가

- **이론 강의**: 스킬·에이전트·오케스트레이터 등 기본 개념, 컨텍스트·루프·하네스 등 유용한 개념 정리.
- **실습·리뷰·과제**: 매주 배운 개념을 직접 내 OS에 적용하며 체득하고, 조별 리뷰로 같은 개념의 다른 접근을 나눈다.

4주 후 얻는 것:
1. **개념 따라잡기** — 지금까지 나온 AI 개념을 따라잡는다.
2. **설계자 경험·시각** — OS 실습·조별 리뷰로 설계 경험을 쌓고 다양한 시각을 얻는다.
3. **AI 변화 저항력** — 미래의 AI 변화에 흔들리지 않는 힘을 기른다.

## 진행 방식: 이론 → 실습 → 리뷰 반복

1. OS 주제 결정 및 주제별 조 편성
2. 강사의 개념 설명 후, 실습 시간에 내 OS에 적용
3. 실습 후 조별로 리뷰 (개념을 어떻게 활용해 OS를 업데이트했는지 나누기)
4. 과제로 부족한 실습을 보충하며 다음 주제 준비

## 실습·리뷰 가이드

**권장 ✅**
- 완벽하지 않아도 **동작하고 효능감 주는 OS를 일단 만들고 지속적으로 업그레이드** (강력 권장)
- 실습은 완성보다 **개념을 실제로 연결**해보는 것이 더 중요한 목표
- 완성하지 못했더라도 구상한 접근을 나누기

**비권장 ⚠️**
- 100% 완성하려고 조별 리뷰 불참
- 조별모임에서 혼자 너무 많이 얘기하기
- 실습과 무관한 얘기하기

## 준비하면 좋은 것

1. **열린 마음** — 열린 마음으로 말하고 들을 때 가장 많이 얻는다.
2. **과제 수행 시간 확보** — 배운 것과 배울 것을 연결하는 과제로 학습 효과가 극대화된다.
3. **동기 부여 되는 OS 결정** — 4주 동안 만들 OS를 미리 생각해오면 실습 시간을 효율적으로 쓸 수 있다(별도 설계 시간도 있음).
