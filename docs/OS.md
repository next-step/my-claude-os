# 내 OS 설계: 주식/투자 OS

> 4주 동안 만들 "나만의 클로드 코드 OS" 설계 문서. 진행하며 계속 업데이트하는 살아있는 문서다.
> 배경 지식은 [onboarding.md](./onboarding.md) 참고.

## 주제 & 동기

- **주제**: 주식/투자 OS
- **최종 목표**: 알아서 매매까지 해주는 **자동매매 에이전트**
- **동기**: 투자 판단과 실행을 AI에게 위임해, 내 투자 성향에 맞게 자동으로 굴러가는 체계를 갖고 싶다.

## 핵심 원칙: 동작하는 것부터, 신뢰는 단계적으로

자동매매는 **종착지이지 출발점이 아니다.** 온보딩의 권장사항("완벽하지 않아도 동작하는 OS를 먼저, 이후 지속 업그레이드")과 현실적 위험(실제 돈·실매매 API·규제)을 고려해, **사람이 승인하는 조언가(1단계)부터** 시작해 신뢰를 쌓으며 자동화 수준을 올린다.

## 단계별 로드맵

**이번 4주 강의의 범위는 1단계 완성**이다. 2·3단계는 강의가 끝난 뒤 혼자 얹어볼 **미래 로드맵**으로만 남겨둔다.

| 단계 | 사람의 역할 | AI의 역할 | 위험 | 상태 |
|------|------------|----------|------|------|
| **1단계 (MVP)** | 투자 성향 제공 + **매매 최종 승인** | 분석·조언·주문서 초안 작성 | 낮음 (모의/수동) | 🟢 4주 강의 목표 |
| **2단계** | 전략·한도 설정, 예외만 개입 | 조건 충족 시 주문 제안, 승인 후 실행 | 중간 | ⚪ 강의 후 미래 |
| **3단계 (목표)** | 가드레일·리뷰만 | 탐색→판단→**자동 매매**→결과 학습 | 높음 | ⚪ 강의 후 미래 |

## 공통 골격 적용 (1단계 기준)

온보딩의 공통 골격 `의도 공유 → 실행 → 피드백 → 반복`을 1단계에 매핑한 것이다.

```
[사람+AI] 의도 공유      투자 성향·목표·제약(종목군, 리스크 한도) 인터뷰
        ↓
[AI]      실행           시장·종목 분석 → 매매 조언 → 주문서 초안 작성
        ↓
[사람]    의사결정        조언 검토 후 매매 승인/거절 (실제 주문은 사람이)
        ↓
[AI]      피드백          결과 기록 → 동향 데이터 확보 → 다음 조언에 반영
        └──────────────── ↻ 반복 ────────────────┘
```

## 1단계 MVP 범위 (4주 안에 동작 목표)

- [ ] 투자 성향 인터뷰 → 프로필 저장 (리스크 성향, 관심 종목군, 한도 등)
- [ ] 종목/시장 데이터 수집 (어떤 소스를 쓸지 미정 — 아래 열린 질문)
- [ ] 분석 → 매매 조언 생성 (사유 포함)
- [ ] 주문서 초안 제시 (종목/수량/가격/근거)
- [ ] 사람이 승인/거절 → 결과 기록
- [ ] 결과·동향을 다음 조언에 반영하는 루프

> 실매매 대신 **모의투자/페이퍼트레이딩**으로 검증 후, 신뢰가 쌓이면 단계 상향.

## 4주 학습 계획 (1단계를 매주 한 겹씩 완성)

자동화 단계를 올리는 게 아니라, **하나의 1단계 조언가 OS**를 매주 그 주의 개념으로 채워 완성한다.
각 주차의 구체적 계획은 **해당 주차 강의를 듣고 나서** 이 섹션에 채운다. (미리 추측해 적지 않는다.)
4주가 끝나면 **완성된 1단계 조언가 OS** 하나가 남는 것이 목표.

| 주차 | 개념 | 1단계에서 만든 것 | 상태 |
|------|------|------------------|------|
| **1주차** | 에이전틱 | _강의 수강 후 작성_ | ⚪ 대기 |
| **2주차** | 컨텍스트 | _강의 수강 후 작성_ | ⚪ 대기 |
| **3주차** | 루프 | _강의 수강 후 작성_ | ⚪ 대기 |
| **4주차** | 하네스 | _강의 수강 후 작성_ | ⚪ 대기 |

## 가드레일 (안전 원칙)

- 1단계에서는 **실제 주문을 AI가 직접 실행하지 않는다.** (사람 승인 필수)
- 손실 한도·종목당 비중 등 **한도를 명시**하고 그 안에서만 조언.
- 모든 매매 조언에 **근거를 남겨** 나중에 복기 가능하게 한다.

## 회사 분석 스킬 (`analyze-company`)

1단계 MVP의 "종목/시장 데이터 수집 → 분석" 부분을 담당하는 첫 스킬.
회사 이름을 받아 서브 에이전트들이 데이터를 **병렬 수집**하고 메인이 **종합**한다.

- **구조**: 멀티 에이전트 (수집 전문 서브 에이전트 3 + 종합 메인 1)
  - `stock-trend-researcher` — 최근 1년 주가 흐름
  - `company-news-researcher` — 회사 관련 뉴스
  - `market-news-researcher` — 대한민국 경제(거시) 뉴스
- **수집 방식**: 웹검색(WebSearch/WebFetch) 기반. *왜:* API 키 발급 없이 바로 시작 가능, 1단계 검증에 충분. 정확도가 필요해지면 DART·증권사 API로 단계적 교체.
- **데이터 범위(현재)**: 위 3가지로 고정. *왜:* 동작하는 것부터 만들고, 아래 후보들은 **서브 에이전트를 추가하는 방식**으로 나중에 확장.

### 확장 후보 데이터 (우선순위순, 강의 후/필요 시 추가)

투자 분석의 고전적 5축은 **기술적 / 펀더멘털 / 수급 / 심리 / 거시**. 현재 스킬은 기술적(주가)·심리(뉴스)·거시(경제뉴스)를 다루고, **펀더멘털·수급**이 비어 있다. 채울 후보:

| 순위 | 데이터 | 왜 필요한가 | 메모 |
|------|--------|------------|------|
| 1 | **재무제표/펀더멘털** (매출·영업이익·ROE, PER·PBR·배당) | 주가는 "시장의 평가", 재무는 "그럴 만한가". 둘을 대조해야 고/저평가 판단이 선다. | 가장 큰 공백 |
| 2 | **공시 (DART)** (실적·증자·자사주·배당·내부자거래) | 무료+구조화+공신력 갖춘 1차 자료. 뉴스보다 신뢰도 높음. | DART API 후보 |
| 3 | **수급** (외국인/기관/개인 매매) | 한국 단기 주가의 핵심 동인. "왜 움직였나"의 절반. | |
| 4 | **동종업계/경쟁사 비교** | 단독 PER·성장률은 의미가 약함. 업계 평균이 기준점을 준다. | |
| 5 | (선택) **애널리스트 컨센서스/목표주가**, **구체 거시 수치**(환율·금리·유가) | 외부 시각·정량 거시 보강. | |

> 데이터 선택 기준: **"이 데이터가 매수/매도/보류 판단을 바꾸는가?"** — 아니면 넣지 않는다.
> 위 축들은 서로 직교(orthogonal)해서, 그대로 서브 에이전트 하나씩으로 매핑된다.
