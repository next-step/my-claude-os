02 큰 그림 — 팀장과 팀원

각 스킬은 오케스트레이터(팀장). 일을 잘게 쪼개 서브에이전트(팀원)에게 위임하고, 팀원은 독립 컨텍스트에서 한 가지 일만 해 결과만 돌려준다.

사용자 → 스킬(팀장) → 병렬 서브에이전트(팀원) → 검증(fact-checker) → 팀장이 조립·저장·전달.

06 안전장치 & 데이터 정책

🚫 자동매매 차단 훅

block-auto-trade.py — PreToolUse 훅. Bash 명령을 검사해 tossctl 실주문(order place/cancel/amend, --execute)을 하네스 차원에서 거부. 원칙 1을 기계적으로 강제.

📈 tossctl = 시세 READ 전용

지수·환율·시세 숫자만 tossctl로 조회. 계좌·주문은 안 씀. 명령 문법은 scripts/tossctl-commands.md 한 곳에서 관리 — 새 명령은 여기 추가.

🔕 출처 비노출 정책

출처는 내부 검증(fact-check)용으로만 확인하고, 사용자에게 보이는 출력엔 매체명·URL·tossctl을 적지 않는다. 대신 쉽게 풀어 쓰고 미확인은 "(확인필요)".

📒 보유의 단일 출처

보유는 오직 portfolio/holdings.md 원장. (토스 계좌는 비어 있어 tossctl로 계좌 조회 안 함.) 개인 금융정보라 git에서 제외.

이 문서는 사람이 읽기 위한 구성 요약이며, 개인 금융정보(보유·브리핑 내용)는 포함하지 않는다.
