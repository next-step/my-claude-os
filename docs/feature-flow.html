/verify 확장 (PR Checks·CI)
판단(/verify, 일꾼) 이 판정을 내면 → 결정(/impl, 제어) 이 분기한다. judging ≠ deciding.
① 통과 → 10 결과 확인  |  ② 구현 결함 → 07로 재구현 ↺ (루프)  |  ③ 설계 문제 → 루프 탈출, 04 /plan 으로 에스컬레이션 ↑ (이번 세션의 "구조 누락"이 바로 ③)
왜 /verify 분리 — PR Checks·CI 등 검증은 계속 자란다. 자라는 일꾼을 /impl 안에 넣으면 god-object. 검증은 /verify로, /impl은 루프 제어만. 흐름 강제 — CLAUDE.md 규칙(게이트·테스트·재활용)
