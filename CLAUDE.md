1. 클로드 OS 관련 모든 파일(예. .claude 하위 md)은 반드시 프로젝트 안에 만들 것
2. 클로드 OS 만들기 실습 중이기 때문에 대화 과정에서 AI와의 협업을 배울 수 있도록 양질의 설명 제공할 것
3. 이 프로젝트는 "개발 OS"(기능 개발 흐름)를 만드는 작업이다. 전체 청사진·개발 흐름·설계 원칙은 [OS.md](./OS.md)에 정의되어 있다. 기능을 개발할 때는 OS.md의 개발 흐름(PRD → Plan → 사람 검토 → 구현)을 따를 것. 특히 사람 검토(게이트)를 통과하기 전에는 구현을 시작하지 말고, 구현에는 단위/통합 테스트를 반드시 포함하며, 새로 짜기 전에 기존 코드 재활용을 우선할 것. OS 구성 요소(스킬·훅·문서 등)를 추가/수정할 때도 먼저 OS.md를 읽고 그 원칙에 맞출 것. 청사진과 어긋나는 변경을 하면 OS.md도 함께 갱신할 것
