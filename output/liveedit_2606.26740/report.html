1. 논문 분석

한 줄 요약

양방향 확산 비디오 편집 모델을 3단계 증류(distillation)로 4-step 인과 모델로 압축하고 AR-지향 마스크 캐시로 불필요 연산을 줄여, 배경 보존을 유지하면서 12.66 FPS의 실시간 스트리밍 비디오 편집을 달성한 프레임워크 LiveEdit.

  • 분야 / 키워드: 컴퓨터 비전, 확산 모델(Diffusion), 스트리밍 비디오 편집, 실시간 추론, 인과적(causal) 생성, Distribution Matching Distillation, 마스크 캐시

2. 해결하려는 문제 (Motivation)

스트리밍(프레임 단위, 실시간) 비디오 편집에서 두 가지 핵심 난제를 해결하고자 한다.

  • 시간적 안정성: 편집되지 않은(배경) 영역을 프레임이 흘러가도 일관되게 보존해야 함.
  • 실시간 응답성: 인과적 프레임 단위 처리로 낮은 지연시간(latency)을 확보해야 함.

구체적으로 (1) 양방향 모델을 인과적 추론으로 전환할 때 과거 프레임에 걸쳐 어텐션 가중치가 평탄(uniform)하게 퍼져 구조적 일관성이 깨지는 어텐션 분포 이동(attention distribution shift) 문제와, (2) 편집되지 않은 영역까지 모든 공간 토큰을 매 프레임 조밀하게 계산하는 공간-시간 중복(spatial-temporal redundancy) 문제를 다룬다.

3. 기존 접근의 한계

  • 양방향 오프라인 모델(LucyEdit, InsV2V, VideoCoF)은 지연시간이 커서 인터랙티브/실시간 응용에 부적합.
  • 기존 스트리밍 생성 기법(StreamV2V, StreamDiffusion 등)은 합성(synthesis)에 치중해 공간적 복원/배경 보존이 떨어짐.
  • 생성용 캐시를 편집에 그대로 적용하면 미편집 영역에서 구조 열화와 깜빡임(flickering)이 발생.
  • Self-Forcing의 ODE 초기화는 편집 작업에 대해 계산 비용이 과도함.

4. 제안 방법 (Method) — 핵심 아이디어와 동작 원리

(A) 3단계 증류 파이프라인

양방향 기반 모델의 편집 능력을 효율적 4-step 인과 모델로 점진 이전.

  • Stage 1 (Foundation Tuning): Diffusion Transformer 기반 양방향 편집 baseline 구축. 원본 latent와 노이즈 latent를 채널 방향으로 concat, MSE 손실(‖ϵ−ϵθ^bid(z_t,t,c)‖₂²)로 학습. (9K steps, lr 1e-5, batch 8)
  • Stage 2 (Teacher Forcing): 양방향→인과 구조로 전환. 미래 토큰 접근을 막는 chunk-wise causal attention mask 적용(chunk = latent 3프레임). Stage 1의 구조적 prior 유지. (20K steps)
  • Stage 3 (DMD, Distribution Matching Distillation): 생성을 4 denoising step으로 압축. Stage 2 가중치에서 직접 초기화(비싼 ODE 초기화 우회). frozen Real Score + trainable Fake Score 모델로 MSE+DMD gradient 결합 최적화. 샘플링 timestep [0, 250, 500, 750]. (10K steps)

(B) AR-지향 마스크 캐시 (AR-Oriented Mask Cache)

편집 활동에 따라 연산을 동적 라우팅.

  • 직전 chunk에서 L₂ 거리(‖z_edit^{k-1}−z_src^{k-1}‖₂ > τ)로 편집/정적 영역의 이진 마스크 추출.
  • 약 70% 중복 토큰을 가지치기(prune). 미편집 영역은 재계산 대신 캐시된 중간 feature를 재사용.
  • 캐시는 Self-Attention 층에만 적용(FFN 재사용 시 고주파 민감도로 열화 발생). 프레임당 약 79ms 지연.

5. 핵심 기여 (Contributions)

  • 양방향 기반 모델을 효율적 4-step 인과 모델로 이전하며 어텐션 분포 이동 문제를 푸는 3단계 증류 파이프라인.
  • 편집/정적 영역을 L₂ 거리로 동적 식별해 중복 연산을 줄이고 배경 보존을 보장하는 AR-지향 마스크 캐시.
  • 120개 video-text 쌍으로 구성한 스트리밍 비디오 편집 벤치마크 제시.
  • 12.66 FPS 실시간 추론으로 인터랙티브/AR 응용을 가능케 하며, 텍스트 정합·배경 일관성에서 SOTA 달성.

7. 한계 및 향후 과제

  • 저자는 이러한 증류 기법을 일반 비디오 편집에 직접 이전하면 고충실도(high-fidelity) 디테일이 종종 열화된다고 인정.

2. 상세 해설 — 실시간 스트리밍 비디오 편집의 원리

이 문서는 LiveEdit 논문의 핵심 개념들을, 사전지식이 없는 사람도 따라올 수 있게 직관 → 비유 → 단계별 전개 → 예시 순으로 풀어 쓴 상세 해설입니다. 원래 3개의 파트(3단계 증류 파이프라인 / AR-지향 마스크 캐시 / 스트리밍 인과 추론 및 평가)를 학습(training) → 추론 메커니즘(inference) → 평가(evaluation) 순으로 재배열했습니다. 원문에 없는 내용은 "직관적 이해를 위한 비유"라고 명시했습니다.

공통 사전 지식 (꼭 알아야 할 개념)

  • 확산 모델(Diffusion Model): 깨끗한 이미지/영상에 노이즈를 조금씩 더해 완전한 모래폭풍으로 만든 뒤, 그 과정을 거꾸로 학습하는 생성 모델. 추론 때는 노이즈에서 출발해 노이즈를 한 단계씩 걷어내며(=denoising) 그림을 만든다.
  • 확산 트랜스포머(Diffusion Transformer): 이미지/프레임을 작은 조각(패치)들로 잘라 각 조각을 하나의 "토큰" 벡터로 다루는 확산 모델 아키텍처.
  • denoising step: 노이즈를 한 번 걷어내는 한 단계. 보통 수십~수백 번 반복. 이 횟수가 곧 속도를 좌우한다(적을수록 빠름).
  • 프레임(frame): 동영상을 이루는 한 장 한 장의 정지 사진.
  • 토큰(token): 확산 트랜스포머가 다루는 단위. 한 프레임 = 여러 토큰 묶음.
  • latent(잠재 표현, z): 영상을 인코더로 압축한 작은 표현. 이 문서의 모든 z는 이 압축 표현이다.
  • chunk(청크): LiveEdit은 latent를 3프레임씩 묶어 한 덩어리 단위로 처리한다.
  • 양방향(bidirectional): 한 프레임을 만들 때 과거와 미래 프레임을 모두 참고. 품질은 좋지만 실시간 불가.
  • 인과적(causal / autoregressive, AR): 한 프레임을 만들 때 과거 프레임만 참고 → 실시간(스트리밍) 가능.
  • 스트리밍(streaming): 들어오는 프레임을 그때그때 바로 처리. 라이브 방송 필터를 떠올리면 된다.
  • 어텐션 / Self-Attention: 한 토큰이 다른 토큰들을 얼마나 참고할지 정하는 연산. 토큰 수 N이면 비용이 대략 N×N에 비례.
  • 마스크(mask): 특정 위치를 "보지 못하게" 차단하는 가림막. 인과 마스크는 미래 토큰으로 향하는 어텐션을 막는다.
  • FFN(Feed-Forward Network): 어텐션 뒤에 붙어 각 토큰을 독립적으로 한 번 더 변환하는 작은 신경망.
  • 캐시(cache): 한 번 계산한 결과를 저장해 두었다가 재사용하는 것.
  • 증류(Distillation): 크고 느린 "선생" 모델의 능력을 작고 빠른 "학생" 모델에 옮겨 담는 기법.
  • L₂ 거리 / MSE 손실: 두 벡터 차이의 크기("두 점 사이의 거리"). 작을수록 잘 맞춘 것.
  • τ(타우, threshold): "이 정도 이상 달라지면 편집된 걸로 본다"의 기준선.
  • NFE (Number of Function Evaluations): 모델을 몇 번 호출(=denoising step)하느냐. 작을수록 빠르다.
  • FPS: 초당 처리 프레임 수. 클수록 실시간에 가깝다.
  • CFG (Classifier-Free Guidance): 텍스트 지시를 더 강하게 따르려고 모델을 두 번 돌려 섞는 기법. 계산 2배. 불필요해지면 그만큼 빨라진다.

Part 1. 3단계 증류 파이프라인 (Foundation Tuning / Teacher Forcing / DMD)

큰 그림: 왜 "3단계"로 나누는가?

LiveEdit이 최종적으로 원하는 모델은 "과거만 보고(인과적), 단 4번의 denoising step만으로(빠름), 양방향 모델처럼 잘 편집하는" 모델입니다. 하지만 인과성 + 초고속 + 고품질을 한꺼번에 학습시키면 무너집니다. 그래서 목표를 세 조각으로 쪼개 한 번에 하나씩 달성합니다.

단계이름새로 추가하는 능력끝났을 때 상태
Stage 1Foundation Tuning편집을 잘하는 능력 (아직 느리고 양방향)100 NFE, CFG 필요, 비스트리밍
Stage 2Teacher Forcing과거만 보는 인과적 구조 (실시간 가능)스트리밍 가능, 아직 step 많음
Stage 3DMD4 step으로의 초압축 (초고속)4 NFE, CFG 불필요, 실시간

핵심 비유(직관용): 운전 교습과 같습니다. ① 운동장에서 운전 자체를 익히고(Stage 1), ② 앞만 보고 달리는 연습(Stage 2), ③ 훨씬 적은 조작으로 매끄럽게 주파(Stage 3). 앞 단계에서 익힌 실력을 버리지 않고 그 위에 다음 능력을 얹는 것이 핵심입니다.

Stage 1: Foundation Tuning (기초 편집 능력 만들기)

직관. 가장 먼저 "영상을 텍스트 지시대로 잘 편집하는" 양방향 편집 baseline을 제대로 만듭니다. 속도·실시간성은 아직 신경 쓰지 않습니다. 나중 단계들은 전부 이 모델을 출발점으로 삼으므로, 여기서 실력이 부실하면 뒤가 다 무너집니다.

비유. 요리로 치면 일단 "맛있게 만드는 법" 자체를 완벽히 익히는 단계. 서빙 속도는 이후 문제.

단계별 전개. 학습 목표(손실 함수):

Loss = ‖ ϵ − ϵθ^bid(z_t, t, c) ‖₂²
  • ϵ: 정답 노이즈. 깨끗한 영상에 일부러 더한 진짜 잡음.
  • z_t: 시각 t에서의 latent. 원본 latent와 노이즈 latent를 채널 방향으로 concat.
  • t: 노이즈 정도(timestep). c: 조건(텍스트 지시).
  • ϵθ^bid(...): 모델 예측. bid=양방향, θ=학습 파라미터.
  • ‖ ... ‖₂²: (정답 − 예측)을 제곱해 합산한 MSE. 이 값을 줄이도록 학습.

세부 설정(원문): Diffusion Transformer, 9K steps, lr 1e-5, batch 8.

예시(토이). 정답 ϵ=[0.5,-0.2], 예측 ϵθ=[0.3,-0.1] → 차이 [0.2,-0.1] → 제곱 [0.04,0.01] → 합 0.05가 그 샘플의 Loss.

이 단계가 끝나면: 편집은 잘하지만 약 100 NFE, CFG 필요, 미래를 봐야 하므로 실시간 불가.

Stage 2: Teacher Forcing (양방향 → 인과적 구조로 전환)

직관. Stage 1 모델은 미래 프레임까지 훔쳐봅니다. 실시간에선 불가능하므로 "미래를 못 보게 가림막을 치는" 구조 개조를 합니다. 단, Stage 1에서 익힌 편집 실력(구조적 prior)은 그대로 물려받습니다.

비유. 시험에서 뒷장(미래 문제)을 보는 버릇이 든 학생에게, 지금 문제와 이미 지나온 앞 문제만 보이도록 뒷부분을 가리는 것. "Teacher Forcing"은 정답(선생)의 흐름을 따라가며 인과 구조에 적응시키는 훈련.

단계별 전개. 핵심 장치는 chunk-wise causal attention mask. latent 3프레임씩 묶음(chunk) 단위로 처리하며, 같은 청크 안은 서로 보되 미래 청크는 못 본다. 양방향을 그냥 인과적으로 바꾸면 과거 프레임들에 어텐션 가중치가 고르게(uniform) 퍼지는 어텐션 분포 이동 문제가 생기는데, Teacher Forcing이 이 부적응을 학습으로 메운다. (20K steps)

예시(토이). 청크A={1,2,3}, 청크B={4,5,6}. 프레임 5를 만들 때 → 1,2,3,4,5는 보되 6은 미래라서 차단. "프레임 6이 아직 안 들어와도 프레임 5를 즉시 출력"이 가능.

이 단계가 끝나면: 스트리밍이 가능. 하지만 denoising step 수는 아직 많아 충분히 빠르진 않음 → Stage 3가 해결.

Stage 3: DMD (4 step으로의 분포 매칭 증류)

직관. 잘 편집하고(S1) 실시간 구조도 갖췄지만(S2) 여전히 수십~백 번 denoising이 필요. DMD는 이를 단 4번으로 압축. 핵심: 장면을 똑같이 베끼는 대신 "학생이 만드는 결과물들의 전체 분포가 선생의 분포와 같아지도록" 맞춘다.

비유. 화가가 50번 붓질로 완성하는 그림을 견습생이 4번 붓질로 비슷하게. "픽셀 단위로 베껴라"가 아니라 "화풍·분위기 전체가 같은 느낌이 되게 하라".

단계별 전개. 두 보조 모델 사용:

  • Real Score — 고정(frozen): 진짜 분포의 방향을 알려주는 기준 모델.
  • Fake Score — 학습(trainable): 현재 학생이 만드는 분포를 추적.
  • 학습 신호 = 두 분포 차이(DMD gradient) + 기본 MSE 손실 결합.

빠르게 만드는 장치: 초기화는 Stage 2 가중치에서 직접(비싼 ODE 초기화 우회). 샘플링 timestep을 4개로 고정 [0, 250, 500, 750] = 4 NFE. 그 결과 CFG도 불필요. (10K steps)

예시(토이). Stage1: 100번 촘촘히(NFE 100). Stage3: 750→500→250→0의 4개 지점만. 효과: 81프레임을 약 7.89초에 처리, 최종 12.66 FPS.

이 단계가 끝나면: 4 NFE, CFG 불필요, 실시간. LiveEdit이 목표한 "빠르고 인과적이며 잘 편집하는" 모델 완성.

전체 흐름 한눈에 보기 (증류 파이프라인)

            [ 느리지만 잘 편집하는 양방향 모델을 만든다 ]
   Stage 1 ───────────────────────────────────────────────┐

   · Diffusion Transformer, 원본+노이즈 latent concat       │  (100 NFE, CFG 필요,

FAQ: 증류 파이프라인

Q. 3단계를 한 번에 합쳐 학습하면 안 되나요?
편집 능력·인과성·4-step 압축을 동시에 요구하면 학습이 불안정해 무너집니다. 목표를 분리해 단계별로 점진 이전하며, 각 단계는 앞 단계 결과 위에서 시작해 실력이 누적됩니다.

Q. "증류"인데 선생/학생 모델이 따로 있나요?
넓게 보면 "느린 양방향 → 빠른 4-step 인과"로 옮기는 것 전체가 증류. 특히 Stage 3 DMD는 명시적으로 고정된 Real Score(선생 기준)와 학습되는 Fake Score(학생 추적)를 씁니다.

Q. step을 줄이면(4 step) 품질이 떨어지지 않나요?
DMD는 "한 장 베끼기"가 아니라 "결과 분포 전체를 선생과 일치"시키는 방식이라 적은 step으로도 품질을 유지합니다. (단 일반 고충실도 편집엔 디테일 열화 가능 — 한계.)

Q. ODE 초기화를 "우회"한다는 게?
무거운 사전 준비(ODE 초기화)를 생략하고 Stage 2 가중치에서 곧바로 DMD 학습을 시작한다는 뜻.

Q. CFG가 "불필요"해지면 왜 좋은가요?
CFG는 모델을 두 번 돌려 계산이 2배. Stage 3 후엔 필요 없어져 그만큼 빨라집니다(12.66 FPS 기여).

Part 2. AR-지향 마스크 캐시 (AR-Oriented Mask Cache)

앞의 3단계 증류가 모델을 학습하는 방법이라면, 이 파트는 학습된 모델이 추론할 때 속도와 일관성을 끌어올리는 메커니즘입니다.

핵심 개념 — AR-지향 마스크 캐시

직관. 영상 편집에서 바꾸고 싶은 부분은 보통 화면의 일부입니다("강아지를 고양이로"). 배경은 그대로 두어야 하는데, 일반 모델은 매 프레임 모든 토큰을 똑같이 비싸게 재계산 → (1) 느리고 (2) 배경이 깜빡입니다(flickering). 핵심 아이디어: "실제로 편집이 일어나는 영역만 새로 계산하고, 변하지 않는 영역은 직전 캐시를 그대로 재사용하자." 어디가 편집 영역인지는 직전 chunk의 L₂ 거리로 자동 판별. 약 70% 중복 토큰을 건너뛰어 빨라지고(12.66 FPS), 배경은 캐시값을 써서 일관성도 좋아집니다.

아래는 직관적 이해를 위한 비유이며, 논문 본문 표현이 아닙니다.

비유. 벽화를 그리는 화가. "벽 가운데 강아지만 고양이로" 요청. 어리석은 화가는 매번 벽 전체를 다시 칠해 느리고 얼룩진다. 똑똑한 화가는 강아지 부분만 마스킹 테이프로 표시해 그 영역만 새로 칠하고 나머지는 그대로 둔다. "테이프 영역"=편집 마스크, "그대로 두기"=캐시 재사용. 테이프 위치는 L₂ 거리로 정한다.

단계별 전개. 기호: z_src^{k-1}=직전 원본 latent, z_edit^{k-1}=직전 편집 latent, τ=임계값, M=토큰별 0/1 마스크(1=편집·새 계산, 0=정적·캐시).

  1. 차이 측정: d = ‖ z_edit^{k-1} − z_src^{k-1} ‖₂ — 토큰이 직전에 실제로 고쳐졌는지 수치화.
  2. 이진화: M = 1 (편집) if d > τ, else M = 0 (정적).
  3. 가지치기(prune): 다음 chunk에서 M=0(약 70%) 토큰은 Self-Attention 연산에서 제외 → N×N 비용 급감.
  4. 캐시 재사용: M=0 토큰 출력은 직전 저장 feature로 채움 → 깜빡임 사라지고 시간적 일관성 유지.
  5. 적용 위치 제한: 캐시는 Self-Attention 층에만, FFN에는 미적용(FFN 캐시 시 고주파 민감으로 심한 열화).

결과: 프레임당 약 79ms, 12.66 FPS.

예시 (작은 토이 입력)

화면을 토큰 6개짜리 1차원 줄로 축소. 직전 chunk 측정값(설명용 가상):

토큰 위치z_src (원본)z_edit (편집본)차이 d = ‖edit−src‖₂
1 (하늘)2.02.00.0
2 (하늘)2.12.10.0
3 (강아지→고양이)5.09.04.0
4 (강아지→고양이)4.08.54.5
5 (바닥)1.01.050.05
6 (바닥)1.01.00.0

τ = 0.5이면: 토큰 3,4 → M=1(새 계산), 토큰 1,2,5,6 → M=0(캐시). 마스크 M = [0,0,1,1,0,0] → 6개 중 2개만 새 계산, 4/6 ≈ 67% pruning("약 70%"와 같은 맥락).

τ를 너무 크게(5.0) 잡으면 토큰 3,4마저 캐시로 덮여 편집이 안 먹고, 너무 작게(0.01) 잡으면 토큰 5의 0.05까지 편집으로 잡아 느려집니다. 그래서 논문도 "콘텐츠에 따라 τ의 세심한 튜닝이 필요"라고 한계로 언급합니다.

전체 흐름 한눈에 보기 (마스크 캐시)

직전 chunk (k-1)
   z_src^{k-1}  ─┐
                ├─▶ d = ‖z_edit − z_src‖₂  (토큰별 차이 측정)
   z_edit^{k-1} ─┘
                      │
                      ▼
             d > τ ?  (임계값 비교)
             ┌────────┴─────────┐
        예(편집,M=1)        아니오(정적,M=0)
             │                    │
             ▼                    ▼
   다음 chunk(k)에서       직전 캐시 feature
   Self-Attention 새로 계산  그대로 재사용 (계산 skip)
             │                    │
             └────────┬───────────┘
                      ▼
        합쳐서 다음 chunk 출력 구성
   (≈70% 토큰 prune → 79ms/프레임, 12.66 FPS)
   ※ 캐시는 Self-Attention 층에만, FFN에는 미적용

FAQ: 마스크 캐시

Q. 마스크를 "지금 프레임"에서 계산하나요?
아니요. 직전 chunk(k−1)의 차이로 마스크를 만들어 다음 chunk(k)에 적용합니다. 미래를 안 보는 인과(AR) 구조라 한 박자 이전 정보를 씁니다.

Q. 캐시를 쓰면 편집 품질이 떨어지지 않나요?
편집 영역(M=1)은 항상 새로 계산하므로 편집은 정상 반영. 캐시는 배경에만 적용되어 오히려 배경 일관성(BC 0.956)을 높입니다.

Q. 왜 FFN엔 캐시를 안 거나요?
FFN 캐시 시 고주파 디테일에 민감해 화질이 심하게 열화. Self-Attention 층에만 적용이 최적.

Q. 70% pruning이면 화면의 70%가 안 바뀐다는 뜻?
대략 그렇습니다. 화면 대부분이 배경(정적)이므로 평균 약 70% 토큰이 캐시 재사용 대상. 콘텐츠에 따라 비율은 달라집니다.

Q. "AR-지향"이 정확히 무슨 뜻?
과거→현재로 한 chunk씩 순서대로 생성하는 자기회귀(스트리밍) 구조에 맞춰 직전 결과를 다음에 이어받도록 캐시를 설계했다는 의미. 양방향 방식엔 쓸 수 없는 실시간 전용 설계.

Part 3. 스트리밍 인과 추론 및 평가

이 파트는 학습된 모델이 실시간으로 한 프레임씩 편집한다는 것의 의미와, 그 품질을 어떻게 측정했는가(평가 지표 · ablation · 사용자 연구)에 집중합니다.

스트리밍 인과(causal) 추론: "미래를 안 보고 한 덩어리씩 편집하기"

직관. 원래 편집 모델은 영상 전체(과거·현재·미래)를 한꺼번에 보고 편집합니다. 라이브 방송처럼 미래 프레임이 아직 도착하지 않으면 불가능하므로, LiveEdit 추론은 "지금까지 들어온 프레임만 보고 3프레임 chunk를 하나씩 순서대로 편집"하도록 바뀝니다.

비유. 양방향 모델은 시험지를 전부 받아놓고 서로 참고하며 푸는 학생. 인과 모델은 문제가 한 장씩 컨베이어 벨트로 넘어오는 퀴즈쇼 참가자. 다음 문제(미래)는 아직 안 보였고 이미 푼 문제와 지금 문제만 보고 즉답해야 합니다.

단계별 전개.

  1. 입력 받기: 새 chunk(latent 3프레임) + 편집 지시문 c.
  2. 인과 어텐션 적용: chunk-wise causal attention mask로 미래 chunk 토큰 참고 차단. 미래 위치 j > i이면 S[i, j] = -∞(→ softmax 후 가중치 0).
  3. 4-step 디노이징: 딱 4번(NFE=4), timestep [0, 250, 500, 750]. (양방향 baseline은 100번 → 25배 빠름)
  4. CFG 생략: 이미 증류돼 있어 CFG 끄고 1패스(연산 절반 절약).
  5. 출력 & 이어붙이기: 편집된 3프레임 출력, 다음 chunk가 오면 1번으로. 방금 결과는 과거 맥락으로 재사용.
⚠️ 여기서 어텐션 분포 이동이 등장. 미래를 막으면 과거 프레임 전체에 어텐션 가중치가 고르게 퍼져 구조가 무너집니다. LiveEdit이 3단계 증류로 잡으려는 게 바로 이 평탄화입니다. ("평평해진다"는 직관적 묘사.)

예시(토이). 프레임 5장, chunk 크기 3. chunkA={1,2,3}, chunkB={4,5}. 양방향이면 프레임3 편집 때 4·5도 봄. 인과(LiveEdit)면 프레임3 편집 때 1·2·3만 봄. chunk B 도착 시 이미 편집된 1·2·3을 과거 맥락으로 참고. 각 chunk 79ms → 1000÷79 ≈ 12.66 FPS.

평가 지표: "잘 편집됐는지를 6개 숫자로 재기"

직관. "편집이 잘 됐다"를 객관적 숫자로. 영상 품질 평가 도구 VBench + 텍스트-이미지 의미 일치 도구 CLIP으로 6측면 점수화. 핵심 긴장: "지시는 잘 따르되, 배경은 안 흔들리게" 두 마리 토끼.

비유. 요리 대회 채점표. "레시피 충실(텍스트 정합)", "접시 안정(배경·움직임)", "먹음직스러움(미적·화질)" 항목별 점수 후 종합.

단계별 전개. ↑=높을수록 좋음.

  1. Text Alignment (TA↑) — 결과가 지시문 c와 얼마나 일치. CLIP 코사인 유사도. 0.270.
  2. Background Consistency (BC↑) — 미편집 영역이 프레임 사이 안 변하는가. 스트리밍 편집의 생명. 0.956.
  3. Motion Smoothness (MS↑) — 덜컹거림·깜빡임 없는가. 0.992.
  4. Dynamic Degree (DD↑) — 움직임이 충분히 살아 있는가. 0.256.
  5. Aesthetic Quality (AQ↑) — 구도·색감이 보기 좋은가. 0.581.
  6. Imaging Quality (IQ↑) — 흐림·노이즈 없이 선명한가. 0.708.

핵심 결과: LiveEdit(W/ Cache)은 TA·BC·MS 세 지표에서 baseline 대비 최고 — 스트리밍 편집에서 가장 중요한 축에서 1등. DD·AQ·IQ에서는 최고가 아닐 수 있는데, 4-step·캐시로 속도를 얻는 대신 약간의 디테일을 내준 절충(분석 7절 "고충실도 디테일 열화" 한계와 연결).

예시(토이). "소파 위 강아지를 고양이로". BC는 편집 영역 바깥 벽·바닥·창문이 1프레임과 30프레임에서 얼마나 같은지로 채점(같으면 0.95↑). TA는 결과 벡터와 "a cat on the sofa" 벡터의 코사인. 두 점수가 동시에 높아야 좋은 편집.

Ablation(제거 실험): "어느 부품이 무슨 일을 하는지 하나씩 떼보기"

직관. 부품을 하나씩 빼보며 성능 변화를 확인 → "운인가, 설계 덕분인가"를 가림. LiveEdit은 (1) 증류 단계별(S1→2→3) 효과와 (2) 캐시를 어느 층에 넣는지를 검증.

비유. 요리에서 재료를 하나씩 빼보는 실험. "소금 빼니 맛이 죽네 → 핵심이었구나".

(1) 증류 단계 ablation.

  1. Stage 1만: 양방향 baseline. 품질 좋지만 100 NFE, CFG 필요, 스트리밍 불가.
  2. + Stage 2: causal mask로 스트리밍(인과) 추가. Stage 1 구조 감각 유지.
  3. + Stage 3: 4 NFE, CFG 불필요 → 속도 결정적 점프(12.66 FPS). 세 단계가 다 있어야 품질+실시간성+초고속이 모두 성립.

(2) 캐시 위치 ablation.

  1. Self-Attention 층에 캐시 → 최적. 미편집 feature 재사용해도 품질 유지.
  2. FFN 층에 캐시 → 심한 열화. FFN은 고주파에 민감해 구조가 깨지고 깜빡임. → "캐시는 Self-Attention 한정"이라는 설계 근거.

예시(토이). Stage1까지: 예쁜데 100번 통과 → 라이브 불가. Stage2까지: 미래 없이 처리되나 반쯤만 빠름. Stage3까지: 4번, CFG 끔 → 79ms/프레임 라이브. FFN 캐시: 빨라지나 배경 지글지글 → "FFN 캐시는 안 되는구나" 데이터로 확인.

User Study: "결국 사람 눈에 좋은가"

직관. 숫자 지표가 높아도 사람이 봤을 때 어색하면 실패. 실제 사람에게 결과를 보여주고 투표. 자동 점수와 주관 점수가 둘 다 좋아야 신뢰.

비유. 영화 평론가 점수(자동 지표) vs 관객 선호 투표(user study).

단계별 전개.

  1. 참가자 모집: 20명에게 여러 모델 결과를 나란히 제시.
  2. 평가 항목: ① 지시 일관성 ② 배경 보존 ③ 종합 품질.
  3. 집계: LiveEdit이 상위 3위(top-3)에 든 비율 계산.
  4. 결과: 전체 품질 top-3 선호율 95.8%, 세 항목 전반 우세. 자동 지표(TA·BC·MS 1등)와 사람 평가가 같은 방향 → 결론이 탄탄.

예시(토이). 20명 중 19명이 LiveEdit을 1~3위에 넣으면 top-3 선호율 = 19/20 = 95%대. 사람 선택으로도 1등이 확인되면 "진짜 좋다"는 주장이 완성.

FAQ: 추론 및 평가

Q1. "인과 추론"이면 통계의 인과추론인가요?
아니요. 여기서 "인과(causal)"는 시간 순서상 미래를 보지 않는다는 뜻(causal mask). 흡연→폐암 같은 통계적 인과관계 분석과는 다릅니다.

Q2. step을 100→4로 줄이면 품질이 확 떨어지지 않나요?
DMD로 "100번 걸려 도달하던 분포"를 4번 만에 흉내 내도록 학습해 TA·BC·MS 1등 유지. 다만 DD·AQ·IQ 디테일 축엔 절충(7절 한계).

Q3. 79ms·12.66 FPS와 "81프레임 7.89초"가 안 맞는데요?
프레임당 79ms → 12.66 FPS는 순수 chunk 추론 속도. "81프레임 7.89초"(≈10.3 FPS)는 인코딩·디코딩 오버헤드까지 포함한 end-to-end 수치로 읽는 것이 권고이며, 측정 범위가 달라 모순이 아닙니다.

Q4. CFG를 끄면 지시를 덜 따르게 되지 않나요?
증류 과정에서 지시 따르는 능력을 모델 안에 내재화해, CFG 없이 1패스로도 TA를 확보. 그래서 끄고도 됩니다.

Q5. 캐시를 더 많은 층(FFN 포함)에 넣으면 더 빠르지 않나요?
빠르긴 하나 FFN은 고주파에 민감해 구조 열화·깜빡임. Ablation이 보여줬기에 캐시는 Self-Attention 층에만.

3. 데이터 흐름 추적

추론(스트리밍, inference-mm.py → pipeline/causal_inference.py / mm_inference.py):

  1. 입력: source video + text instruction(test_cases/test.json, test.mp4).
  2. 전처리: T5 텍스트 인코딩(wan/modules/t5.py), VAE로 source video → source_latent(16ch). 노이즈 latent와 채널 concat → 32ch DiT 입력.
  3. chunk 루프(num_frame_per_block=3 latent frame 단위):
    • 직전 chunk 결과 generated_latent와 source_latent로 _compute_mask_from_generated → importance 맵.
    • _compute_mask_from_importance(adaptive_patch_ratio=0.3) → 상위 30% keep(편집), 나머지 ~70% prune.
    • 4-step denoise(denoising_step_list=[1000,750,500,250]): 각 step에서 generator(noisy_input, kv_cache, current_start=...) 호출. prune된 토큰은 Self-Attention에서 unpruned_fill_strategy="prev_step"로 이전 캐시 feature 재사용.
    • KV 캐시·cross-attn 캐시 갱신(current_start_frame * frame_seq_length), 롤링 eviction + sink token.
  4. 출력: 편집된 latent → VAE 디코딩(demo_utils/vae*) → 프레임 출력(스트리밍). 12.66 FPS / ~79ms per 3-frame chunk.

학습(3단계): Stage1 양방향 MSE(MMDiffusion) → Stage2 인과 mask teacher forcing(MMCausVid/causal_model) → Stage3 DMD(MMDMD, Real frozen + Fake trainable, dfake_gen_update_ratio=5).

4. 핵심 코드 발췌 + 해설

(1) DMD 분포정합 gradient — model/mm_dmd.py

grad = (pred_fake_image - pred_real_image)
if normalization:
    p_real = (estimated_clean_image_or_video - pred_real_image)
    normalizer = torch.abs(p_real).mean(dim=[1, 2, 3, 4], keepdim=True)
    grad = grad / normalizer
dmd_loss = 0.5 * F.mse_loss(
    original_latent.double(),
    (original_latent.double() - grad.double()).detach(), reduction="mean")

해설: Fake/Real score 예측 차이 (fake−real)를 정규화한 뒤 generator latent에서 빼는 형태로 MSE 손실 구성 → 분포정합(reverse-KL 근사) gradient를 표준 MSE 역전파로 흘림. y=conditional_dict["source_latent"]로 편집 조건(소스) 주입.

(2) 입력 채널 확장(소스 비디오 concat) — model/mm_dmd.py:_expand_input_layer

new_proj = torch.nn.Conv3d(in_channels=32, ...)
new_proj.weight[:, :16].copy_(old_proj.weight)   # 기존 16ch 가중치 보존
# 채널 17~32 = 0 초기화 → source latent 입력 경로

해설: 양방향 baseline의 patch-embed를 16→32ch로 확장해 source latent를 추가 주입. 신규 채널 0초기화로 학습 초기 안정성 확보.

(3) chunk-wise causal attention mask — wan/modules/causal_model.py

frame_indices = torch.arange(0, total_length,
                             step=frame_seqlen * num_frame_per_block, device=device)
# mask: (kv_idx < ends[q_idx]) | (q_idx == kv_idx)

해설: 각 query는 "현재 chunk 끝(ends) 이전" 토큰까지만 attend → 미래 누설 차단(인과화). num_frame_per_block=3로 3 latent-frame을 한 chunk로 묶음. KV 캐시는 sink token 고정 + 롤링 eviction.

(4) AR-지향 마스크 캐시(L₂ → 프루닝) — pipeline/causal_inference.py

def _compute_mask_from_generated(self, generated_latent, source_latent):
    diff = (generated_latent - source_latent).pow(2).mean(dim=2)        # L2^2
    importance = (diff - diff.min()) / (diff.max() - diff.min() + 1e-8)
    low_importance_mask = self._compute_mask_from_importance(importance)

def _compute_mask_from_importance(self, importance_map):
    keep_num = int(num_tokens * self.adaptive_patch_ratio)              # 0.3 → 30% keep
    threshold_value = torch.kthvalue(importance_frame,
                                     num_tokens - keep_num + 1).values  # 임계 τ
    high_importance_frame = (importance_frame >= threshold_value)       # 편집 영역
    low_importance_mask_flat[b, f] = ~high_importance_frame             # 정적 영역=캐시

해설: 직전 chunk의 편집 강도(L₂²)로 편집/정적 토큰을 이진 분리. 상위 30%(편집)만 재계산, 하위 ~70%(정적)는 Self-Attention 캐시 feature 재사용(unpruned_fill_strategy="prev_step"). internal_pruning_layers=["self_attn"]로 FFN 제외(논문 ablation 일치).
