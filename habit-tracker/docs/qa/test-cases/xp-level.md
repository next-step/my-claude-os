## TC-XP-01: 기본 XP 적립 — 루틴 완료 시 +10 XP

**분류:** 기능 / 해피패스
**심각도 기준:** P1 (적립 안 되면 핵심 기능 불가)

**사전 조건:** 오늘 예정된 활성 루틴 1개 이상 존재, 해당 루틴이 미완료 상태

**재현 단계:**
1. 홈 화면 접속 → 현재 totalXp 기록 (DB: `select total_xp from user_stats where id=1`)
2. 루틴 카드의 체크 버튼 클릭
3. 홈 화면 리다이렉트 후 XP 바 확인
4. DB에서 totalXp 재조회

**기대 결과:**
- totalXp가 정확히 10 증가
- `routine_checks` 테이블에서 해당 루틴의 `xp_awarded = true`
- XP 바 수치 텍스트가 업데이트된 값 표시

**실패 시 확인:** `RoutineService.toggleCheck()` 내 XP 적립 분기

## TC-XP-02: XP 중복 방지 — 체크 해제 후 재체크

**분류:** 유효성 / 도메인 정책
**심각도 기준:** P1 (중복 지급 시 레벨 인플레이션)

**사전 조건:** 미완료 루틴 1개

**재현 단계:**
1. DB totalXp 기록 (T0)
2. 루틴 체크 → totalXp 확인 (T1 = T0 + 10)
3. 동일 루틴 체크 해제 (미완료로 전환)
4. 동일 루틴 재체크
5. DB totalXp 확인 (기대: T1, 추가 +10 없음)

**기대 결과:**
- 재체크 후 totalXp = T0 + 10 (추가 증가 없음)
- `xp_awarded`가 이미 true이므로 XP 지급 로직 스킵됨

**실패 시 확인:** `RoutineService.toggleCheck()` 내 `if (!check.isXpAwarded())` 조건

**기대 결과:**
- totalXp 변동 없음 (T1 유지)

**실패 시 확인:** `RoutineService.toggleCheck()` 내 체크 해제 분기에서 XP 로직 없어야 함

## TC-XP-04: 전체 완료 보너스 — 하루 루틴 전부 완료 시 +50 XP

**분류:** 기능 / 해피패스
**심각도 기준:** P1 (핵심 게이미피케이션 기능)

**사전 조건:** 오늘 예정 루틴 N개 (N≥2), 모두 미완료, lastBonusDate != 오늘

**재현 단계:**
1. DB totalXp 기록 (T0)
2. N-1개 루틴 체크 완료 → totalXp = T0 + 10*(N-1)
3. 마지막 루틴 체크
4. DB totalXp 확인: 기대 = T0 + 10*N + 50
5. `last_bonus_date` = 오늘 확인

**기대 결과:**
- 마지막 루틴 완료 후 +50 XP 추가 지급
- `last_bonus_date = 오늘` 설정

**실패 시 확인:** `RoutineService.toggleCheck()` 전체 완료 보너스 분기

## TC-XP-05: 전체 완료 보너스 중복 방지

**분류:** 도메인 정책 / 경계값
**심각도 기준:** P1

**사전 조건:** 오늘 모든 루틴 완료 상태 (lastBonusDate=오늘), DB totalXp = T

**재현 단계:**
1. 완료된 루틴 1개 체크 해제 (전체 미완료 상태 전환)
2. 해당 루틴 재체크 (전체 완료 상태 복구)
3. DB totalXp 확인

**기대 결과:**
- totalXp 변동 없음 (추가 +50 없음, xpAwarded=true이므로 +10도 없음)
- `last_bonus_date` = 오늘 그대로 유지

**실패 시 확인:** `UserStatsService.isTodayBonusAwarded()` 조건

## TC-XP-07: 레벨업 배너 표시

**분류:** 기능 / 레벨업
**심각도 기준:** P2

**사전 조건:** DB에서 `total_xp = 195` 직접 세팅 (새싹→묘목 경계 5 XP 부족)

**재현 단계:**
1. 홈 접속 → XP 바 "195 / 200 XP" 확인 (단, 현재 구현은 "195 / 5 XP" 표시 예상 - Bug #2 참고)
2. 미완료 루틴 체크 (xpAwarded=false 상태여야 함)
3. 홈으로 리다이렉트 확인

**기대 결과:**
- 레벨업 배너가 화면 상단에서 슬라이드 인 (0.3s)
- 배너 텍스트: "🌱 → 🌿 묘목에 도달했어요!"
- 4.5초 후 배너 페이드 아웃 시작
- 5초 후 배너 DOM에서 완전 제거 (F12 Elements 탭 확인)

## TC-XP-08: 레벨업 배너 자동 제거 타이밍

**분류:** UI / 애니메이션
**심각도 기준:** P3

**PRD AC 기준:** "배너는 5초 후 자동으로 사라진다"
**디자인 스펙:** 4.5초 유지 + 0.5초 페이드 아웃 = 총 5초

**재현 단계:**
1. 레벨업 조건 충족 후 홈 접속
2. 초시계 측정: 배너 등장 시점부터 DOM 제거 시점까지

**기대 결과:**
- 4.5초 후 페이드 아웃 시작
- 약 5초 후 DOM에서 제거
- 타이머가 실행 중 닫기 버튼 클릭 시 즉시 제거 (5초 타이머 취소 확인)

## TC-XP-11: 만렙에서 추가 XP 적립 시 레벨업 배너 미표시

**분류:** 도메인 정책 / 만렙
**심각도 기준:** P2

**사전 조건:** DB에서 `total_xp = 15010` 세팅 (이미 만렙), 미완료 루틴 존재

1. 루틴 체크 완료 (xpAwarded=false 상태)

**기대 결과:**
- totalXp += 10 (XP 적립은 정상)
- 레벨업 배너 표시 안 됨

**확인 포인트:** `Level.LEGEND.nextLevel()` = `Level.LEGEND` (자기 자신 반환) → prevLevel == newLevel 조건으로 레벨업 감지 안 됨

## TC-XP-14: 과거 날짜 루틴 체크 — XP 적립 확인

**분류:** 도메인 / 엣지케이스
**심각도 기준:** P3

**사전 조건:** 과거 날짜(예: 어제)에 완료하지 않은 루틴 존재

**재현 단계:**
1. `/history?date={어제날짜}` 접속 → 체크 버튼 비활성화 확인 (UI 차단 검증)
2. POST `/check/{routineId}?returnDate={어제날짜}` 직접 요청 (curl 또는 개발자 도구)
3. DB totalXp 확인

**기대 결과:**
- UI에서는 과거 날짜 체크 불가 (버튼 disabled)
- 서버 레이어에서 과거 날짜 체크 요청 시 XP 적립됨 (PRD AC 명시)
- 과거 날짜 전체 완료 보너스는 오늘 보너스 lastBonusDate에 영향 줌 (잠재적 버그 - Bug #3)

## TC-XP-17: 홈 화면 XP 관련 모델 속성 확인

**분류:** 기능 / 서버-클라이언트 계약
**심각도 기준:** P2

**재현 단계:**
1. GET `/` 요청 후 응답 HTML 확인 (개발자 도구 Elements)

**기대 결과 (PRD 계약):**
- `userLevel`: 현재 레벨 enum (이모지, 레벨명, 번호)
- `xpInCurrentLevel`: 현재 레벨 내 진행 XP
- `xpToNextLevel`: 다음 레벨까지 남은 XP
- `xpProgressPercent`: 0~100 진행률
- `leveledUp`: 기본 false (플래시 없을 때)
