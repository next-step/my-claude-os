## 홈 화면 (index.html)

### TC-UI-01: 완료율 숫자 블록 제거 확인

**분류:** UI / 정보 위계
**심각도 기준:** P2 (XP 바 시각 우선순위 침해)

**재현 단계:**
1. 홈 화면 접속
2. `display-6` 크기의 완료율 숫자 블록 존재 여부 확인

**기대 결과:**
- `display-6 fw-bold` 완료율 텍스트 없음
- XP 바가 루틴 목록보다 시각적으로 먼저 보임 (상단 배치)
- 완료율 정보는 진행 바 + "N/M 완료" 카운트로만 표현

### TC-UI-02: 진행 바 색상 분기

**분류:** UI / 시각 피드백
**심각도 기준:** P3

**재현 단계:** 루틴 개수를 조정해 완료율을 각 구간으로 만든 뒤 홈 접속

| completionRate | 기대 progress-bar 클래스 | 색상 |
|---------------|----------------------|------|
| 0% | bg-secondary | 회색 |
| 1~49% | bg-secondary | 회색 |
| 50% | bg-primary | 파랑 |
| 50~99% | bg-primary | 파랑 |
| 100% | bg-success | 초록 |

**기대 결과:** CSS 변수 오버라이드(`--bs-primary: #3B82F6`, `--bs-success: #22C55E`) 반영 확인

### TC-UI-20: 통계 빈 상태 표시 불가 검증 (Known Issue)

**분류:** 경계값 / 빈 상태
**심각도 기준:** P3 — Bug #4 연관

## 루틴 관리 화면 (routines/list.html)

### TC-UI-21: 그만하기 2단계 확인 패턴

**분류:** UI / 위험 행동 보호
**심각도 기준:** P2

**재현 단계:**
1. 루틴 목록에서 "그만하기" 버튼 클릭
2. 확인 행 표시 확인
3. "취소" 클릭 → 원래 상태 복귀 확인
4. 다시 "그만하기" → "그만하기 최종 확인" 클릭

**기대 결과:**
- 1단계: 확인 행 인라인 표시 (`d-none → d-flex`) + 배경 `--color-danger-light`
- 취소 클릭: 원래 배경 복원, 확인 행 `d-none`
- 2단계 최종: 루틴 deactivate, PRG 리다이렉트
