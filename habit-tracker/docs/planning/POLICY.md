## 1. 앱 비전

**"루틴을 지키는 것이 게임보다 재밌어지는 앱"**

습관 형성은 지루하다는 편견을 깬다.
사용자가 체크 하나 누를 때마다 작은 성취감, 예상치 못한 보상, 성장의 시각화를 제공해 앱을 매일 열고 싶게 만든다.

**핵심 감정 목표**: 뿌듯함, 설렘, 경쟁심, 귀여움

## 3. 게이미피케이션 시스템

### 3-1. XP & 레벨 시스템

| 행동 | 획득 XP |
|------|--------|
| 루틴 1개 완료 | +10 XP |
| 하루 전체 완료 | +50 XP 보너스 |
| 3일 연속 | +30 XP |
| 7일 연속 | +100 XP |
| 30일 연속 | +500 XP |

**레벨 구간** (누적 XP 기준):

| 레벨 | 이름 | 필요 XP |
|------|------|--------|
| 1 | 새싹 🌱 | 0 |
| 2 | 묘목 🌿 | 200 |
| 3 | 나무 🌳 | 600 |
| 4 | 숲 🌲🌲 | 1,500 |
| 5 | 정원사 👨‍🌾 | 3,500 |
| 6 | 자연인 🏕️ | 7,000 |
| 7 | 전설 ⚡ | 15,000 |

> **정책**: 레벨다운 없음. 한 번 올라간 레벨은 유지된다. (좌절 방지)

### 3-2. 스트릭(Streak) 시스템

- 하루 루틴을 **1개 이상** 완료하면 스트릭 유지
- 스트릭 3일부터 불꽃 이모지 🔥 표시
- 스트릭 7일 이상은 불꽃이 커짐 (애니메이션)
- **스트릭 보호권**: 30일마다 1개 자동 지급. 하루 빠졌을 때 사용하면 스트릭 유지
- 스트릭 기록은 깨져도 "역대 최고 기록"으로 보존됨

### 3-3. 뱃지(Badge) 시스템

| 뱃지 | 조건 | 설명 |
|------|------|------|
| 🎯 첫 발걸음 | 루틴 첫 완료 | 시작이 반이다 |
| 🔥 불꽃 3일 | 3일 연속 | 연속의 시작 |
| 💎 다이아 7일 | 7일 연속 | 습관이 되고 있어 |
| 👑 전설 30일 | 30일 연속 | 당신은 전설 |
| 🌅 얼리버드 | 오전 7시 이전 완료 | 일찍 일어나는 새 |
| 🦉 야행성 | 오후 11시 이후 완료 | 밤의 수호자 |
| 🎰 올클리어 | 하루 루틴 전체 완료 | 완벽주의자 |
| 🌈 7색 루틴 | 7개 이상 루틴 활성화 | 다재다능 |
| 💪 철인 | 30일간 완료율 90% 이상 | 의지의 화신 |

### 3-4. 일일 미션 (Daily Quest)

매일 자정에 리셋되는 랜덤 미션 1개:
- "오늘 루틴 2개 이상 완료하기"
- "오전 중에 루틴 1개 완료하기"
- "가장 오래된 루틴 완료하기"

달성 시 +20 XP 추가 지급.

### 3-5. 주간 보스 챌린지 (Weekly Boss)

매주 월요일에 등장하는 "보스 루틴":
- 예: "이번 주 5일 이상 루틴 완료하기"
- 보스 처치 성공 → 특별 뱃지 + +200 XP
- 보스 처치 실패 → "다음 주에 다시 도전" (패배 메시지도 재미있게)

## 4. 핵심 UX 정책

### 4-1. 피드백 원칙

- 체크 완료 시 **반드시** 즉각적인 시각/청각 피드백 (애니메이션 or 사운드)
- 루틴 전체 완료 시 **축하 애니메이션** 재생 (confetti 등)
- 100% 달성 시 화면이 달라져야 함 (특별한 배경색, 메시지)
- 오류 메시지는 사람의 언어로: "오류가 발생했습니다" 금지 → "잠깐, 뭔가 문제가 생겼어요 🤔"

### 4-2. 빈 상태(Empty State) 정책

- 절대로 빈 화면 노출 금지
- 빈 상태마다 따뜻한 메시지 + 행동 유도 버튼 필수
- 예: "아직 루틴이 없어요. 첫 루틴을 만들어볼까요? 🌱"

### 4-3. 네이밍/톤 정책

| 금지 표현 | 대체 표현 |
|---------|---------|
| 삭제 | 그만하기 |
| 오류 | 잠깐, 문제가 생겼어요 |
| 완료율 0% | "오늘 시작해봐요!" |
| 기록 없음 | "첫 기록을 만들어보세요" |
| 로딩 중 | "불러오는 중이에요..." |

### 4-4. 날짜/시간 정책

- 미래 날짜 체크는 허용하지 않음 (당일~과거만 체크 가능)
- 하루의 끝은 **자정(00:00)** 기준
- 날짜 표시: `오늘`, `어제`, 그 외는 `M월 d일 (요일)` 형식

## 5. 기술 정책

| 항목 | 정책 |
|------|------|
| 데이터 저장 | H2 파일 기반 로컬 저장 (클라우드 동기화 없음) |
| 인증 | 없음 (단일 사용자 로컬 앱) |
| 루틴 삭제 | Soft delete만 허용 (`active=false`) |
| 날짜 형식 | `yyyy-MM-dd` 문자열 ↔ `LocalDate` 변환 |
| 빈 목록 | null 금지, 빈 배열 `[]` 반환 |
| POST 응답 | PRG 패턴 필수 (redirect after POST) |
