## 목적

오늘 해야 할 루틴을 한눈에 확인하고, 체크할 때마다 즉각적인 성취감을 얻어 내일도 앱을 열고 싶게 만든다.

## 현재 구현 검토 — 발견된 문제

이 섹션은 기존 `index.html` 검토 결과다. 아래 레이아웃·컴포넌트 명세는 이 문제들에 대한 개선안이다.

### 문제 1: 완료율 이중 표시 (정보 중복 + 위계 혼란)

**현재 구조:**
```
[XP 바]

오늘의 루틴     2026년 6월 26일
                     0%       ← display-6 (약 48px 크기)
                   완료율

[████░░░░] ← 진행 바 (8px)
```

`display-6` 크기의 완료율 숫자와 진행 바가 동일한 정보를 두 번 표현한다.
더 큰 문제는 `display-6`(48px)가 XP 바의 레벨명(fw-semibold, 약 16px)보다 시각적으로 훨씬 강해서,
게이미피케이션의 핵심인 XP 바가 부가 정보처럼 보인다.

**개선:** 헤더의 완료율 숫자 블록 제거. 진행 바를 유일한 완료율 표현 수단으로 유지.
진행 바 옆에 소형 텍스트로 카운트(`3/5 완료`) 표시.

### 문제 2: 빈 상태 메시지 — POLICY §4-2, §4-3 위반

**현재:** `bi-inbox` 아이콘 + `"오늘 예정된 루틴이 없습니다."`

- `bi-inbox`는 이메일 받은편지함 아이콘. Habit Tracker 맥락 부적합.
- 메시지가 사무적이고 차갑다. POLICY §4-2는 "따뜻한 메시지 + 행동 유도 버튼 필수"를 명시한다.
- POLICY §4-3의 네이밍 가이드에 의하면 "완료율 0% → 오늘 시작해봐요!"가 대체 표현이다.

### 문제 3: 루틴 카드 완료 상태 과잉 표현 (4중 → 3중)

**현재 완료 시 동시 적용되는 시각 신호:**
1. border-left 색상: primary blue → success green
2. 체크 아이콘: `bi-circle text-secondary` → `bi-check-circle-fill text-success`
3. 루틴 이름: `text-decoration-line-through text-muted`
4. 오른쪽 "완료" 배지: `bg-success-subtle text-success`

1, 2, 3만으로 완료 상태는 충분히 전달된다. 4번 "완료" 배지는 과잉이며 카드 우측 공간을 
소모한다. 배지를 제거하면 카드 레이아웃이 더 깔끔해진다.

### 문제 4: 100% 달성 시 특별 상태 없음 — POLICY §4-1 위반

POLICY §4-1: "루틴 전체 완료 시 축하 애니메이션 재생", "100% 달성 시 화면이 달라져야 함"

현재 구현에서는 완료율이 100%가 돼도 진행 바가 초록으로 채워지는 것 외에 아무것도 없다.
최소한 헤더 영역에 축하 메시지와 다른 배경 처리가 필요하다.

### 문제 5: 완료율 진행 바 색상 고정

현재: 항상 `bg-success` (초록).
루틴을 1개만 해도 초록이어서 "아직 갈 길이 멀다"는 피드백이 없다.
헤더의 완료율 텍스트는 색상 분기(success/primary/secondary)가 있는데 진행 바는 항상 초록 — 불일치.

## 사용자 플로우

### 플로우 A — 기본 홈 진입 (루틴 있음)
```
홈 진입
  → XP 바 확인 (내 레벨 + XP 진행률)
  → 오늘의 루틴 제목 + 진행 바 + "N/M 완료" 확인
  → 루틴 카드 목록 스캔
  → 체크 버튼 탭
  → 진행 바 즉시 업데이트 (PRG redirect 후 새 페이지에서 채워진 상태)
  → 모든 루틴 완료 시 → 100% 달성 상태로 전환
```

### 플로우 B — 빈 상태 (오늘 루틴 없음)
```
홈 진입
  → XP 바 확인
  → 빈 상태 영역: 따뜻한 메시지 + "루틴 추가하기" CTA
  → 루틴 관리 화면으로 이동
```

### 플로우 C — 100% 달성 상태
```
마지막 루틴 체크
  → PRG redirect → 홈 로드
  → 진행 바 100% 초록
  → 헤더 영역: 축하 메시지 ("오늘 모든 루틴 완료! 정말 대단해요 :)")
  → 헤더 배경: --color-success-light 틴트
```

│  ╚══════════════════════════════════╝  │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ 🌱 새싹  Lv.1                    │  │  ← XP 바 (xp-level.md 참조)

- 유지: 제목(`h5 fw-bold mb-0`) + 날짜(`small text-muted`)
- 진행 바 + 카운트가 완료율을 충분히 표현한다

### 화면 2 — 진행 바 + 카운트 (개선 후)

```
┌──────────────────────────────────────────┐
│  [████████████░░░░░░░░░░░░]  3/5 완료   │
└──────────────────────────────────────────┘
```

**구조:**
```html
<div class="d-flex align-items-center gap-2 mb-4">
  <div class="progress flex-grow-1" style="height: 10px;">
    <div class="progress-bar" ...></div>
  </div>
  <span class="completion-count">3/5 완료</span>
</div>
```

- 진행 바 높이: 8px → **10px** (현재보다 약간 두꺼워져 존재감 강화)
- 색상 분기 (Thymeleaf classappend):
  - `completionRate == 100` → `bg-success`
  - `completionRate >= 50` → `bg-primary`
  - `completionRate < 50` → `bg-secondary`
- `.completion-count`: `font-size: 13px`, `color: var(--color-text-2)`, `white-space: nowrap`

## 텍스트 가이드

### 헤더

- 제목: `"오늘의 루틴"` (고정)
- 날짜: `"yyyy년 M월 d일 (E)"` 포맷, 한국어 Locale

### 완료율 카운트

- 형식: `"{완료수}/{전체수} 완료"`
- 예시: `"0/3 완료"`, `"3/3 완료"`
- 100% 시에도 `"3/3 완료"` 유지 (% 표시 금지 — 헤더 제거와 일관성)

### 100% 축하 메시지

- `"오늘 모든 루틴 완료! 정말 대단해요 :)"`
- 톤: 친구가 응원하는 느낌. 앱이 직접 말을 건다.
- 이모지: 텍스트 메시지 뒤에 추가하지 않음 (제목 옆 🎉로 충분)

### 빈 상태

- 제목: `"오늘은 쉬는 날이에요"`
- 부제: `"새 루틴을 추가하면 내일부터 시작돼요"`
- CTA: `"루틴 추가하기 →"` (→ 포함: 방향성 암시)

## 반응형 규칙

### 모바일 (< 576px)

- 헤더 영역: 제목과 날짜 세로 스택 (flex-column)
- 진행 바 + 카운트: 한 줄 유지, 카운트 텍스트 `white-space: nowrap`
- 루틴 카드: 풀 너비, 체크 버튼 44px 터치 타겟 유지
- 100% 달성 메시지: 중앙 정렬, 폰트 14px 유지 (축소 금지)

### 데스크탑 (> 768px)

- container max-width 720px 기준
- 헤더 영역: 제목 + 날짜 가로 배치 (현행 유지)
- 루틴 카드: container 전체 너비 사용
