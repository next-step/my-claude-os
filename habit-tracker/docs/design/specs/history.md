# 기록 화면 명세

> 작성일: 2026-06-27 (검토 기반 신규 작성)
> 연관 정책: docs/planning/POLICY.md §4-2, §4-3, §4-4

---

## 목적

과거의 루틴 기록을 날짜별로 돌아보면서 "나는 꾸준히 하고 있었구나"는 회고의 감정을 얻게 한다. 책망하는 화면이 아니라 기억하는 화면이다.

---

## 현재 구현 검토 — 발견된 문제

### 문제 1: 과거 미완료 루틴에 "미완료" 배지 표시 — 감정 설계 실패

**현재:** 과거 날짜의 미완료 루틴에 `<span class="badge bg-secondary-subtle text-secondary">미완료</span>` 표시

이미 지나간 날짜에서 "미완료"라는 레이블을 보여주는 것은 죄책감을 유발한다.
토스/당근의 핵심 감정 설계 원칙: "앱이 사용자를 다그치지 않는다."

기록 화면의 목적은 과거를 확인하는 것이지 못 한 걸 부각하는 게 아니다.
완료된 루틴이 눈에 띄면 충분하다. 미완료는 조용히 있으면 된다.

**추가로:** 홈 화면(오늘)의 기록 화면과 달리, 과거 날짜 기록에서는 루틴 추가 체크가 불가능(`th:disabled="${isToday == false and r.completed == false}"`)하다. 비활성화된 버튼 + "미완료" 배지가 함께 있으면 "할 수도 없는데 왜 보여주나" 느낌.

**개선:** 과거 미완료 배지 제거. 체크 버튼 비활성화만으로 미완료 상태 충분히 전달.

### 문제 2: 날짜 네비게이션 버튼 터치 타겟 미달

**현재:** `btn btn-outline-secondary btn-sm`
Bootstrap btn-sm 높이: 약 31px → 44px 터치 타겟 기준 미달

POLICY 및 디자인 시스템: "모든 탭/클릭 가능 요소는 최소 44×44px"

**개선:** `btn-sm` → `btn` (기본 크기) 또는 padding 직접 지정

### 문제 3: 미래 날짜 "다음" 버튼 비활성 표현 불명확

**현재:** `th:classappend="${isToday} ? ' disabled' : ''"`

Bootstrap disabled 클래스는 포인터 이벤트를 막지만 시각적으로 충분히 비활성 상태를 전달하지 못한다.
사용자가 "왜 안 눌리지?" 의아해할 수 있다.

**개선:** `th:disabled` 속성 추가 + `opacity: 0.4` 또는 별도 CSS 처리

### 문제 4: 빈 상태 메시지 톤 — POLICY §4-2 위반

**현재:** `"이 날 예정된 루틴이 없습니다."`

차갑고 사무적. POLICY §4-3: 기록 없음 → "첫 기록을 만들어보세요"

## 사용자 플로우

### 플로우 A — 오늘 기록 조회

```
기록 진입 (기본: 오늘 날짜)
  → 날짜 헤더에 "오늘" 뱃지 표시
  → 오늘 루틴 목록 (완료/미완료 모두 표시)
  → 완료 루틴: 체크 가능 (체크/취소 토글)
  → 미완료 루틴: 체크 가능
```

### 플로우 B — 과거 날짜 조회

```
[< 이전] 버튼 탭
  → 하루 전 날짜로 이동
  → 과거 루틴 목록 표시
  → 완료 루틴: 체크 아이콘 filled (read-only, 버튼 비활성)
  → 미완료 루틴: 빈 원 아이콘 (read-only, 배지 없음)
  → [다음 >] 버튼: 오늘이면 비활성
```

### 플로우 C — 빈 상태 (해당 날짜 루틴 없음)

```
날짜 이동
  → 해당 요일에 설정된 루틴 없음
  → 빈 상태 메시지 + 홈 CTA 표시
```

### 화면 1 — 전체 구조

```
┌──────────────────────────────────────────┐
│  [네비게이션 바]                          │
├──────────────────────────────────────────┤
│                                          │

│                    오늘                   │  ← isToday일 때만 표시
│                                          │
│  ┌──────────────────────────────────┐    │
│  │ ✓  ~~아침 스트레칭~~            │    │  ← 완료 루틴 (read-only if 과거)
│  └──────────────────────────────────┘    │
│  ┌──────────────────────────────────┐    │
│  │ ○  물 2L 마시기                  │    │  ← 미완료 (배지 없음, 버튼 비활성)
│  └──────────────────────────────────┘    │
│                                          │
└──────────────────────────────────────────┘
```

### 화면 2 — 날짜 네비게이션 (개선 후)

```
┌───────────────────────────────────────┐
│  [  <  ]   2026년 6월 27일 (토)   [  >  ] │  ← 버튼 크기 44px 이상
│                   오늘                │
└───────────────────────────────────────┘
```

**개선 사항:**

```html
<!-- 개선 전 -->
<a ... class="btn btn-outline-secondary btn-sm">
  <i class="bi bi-chevron-left"></i>
</a>

<!-- 개선 후 -->
<a ... class="btn btn-outline-secondary nav-date-btn">
  <i class="bi bi-chevron-left"></i>
</a>
```

```css
/* style.css 추가 */
.nav-date-btn {
  min-width: 44px;
  min-height: 44px;
  display: inline-flex;
  align-items: center;
  justify-content: center;
  padding: 0 var(--space-md);
}
```

**미래 날짜 비활성 처리:**
```html
<!-- 개선 전 -->
<a ... th:classappend="${isToday} ? ' disabled' : ''">

<!-- 개선 후 -->
<a ... th:classappend="${isToday} ? ' disabled' : ''"
       th:attr="aria-disabled=${isToday},
                tabindex=${isToday} ? '-1' : '0'">
```

### 화면 3 — 과거 날짜 루틴 카드 (개선 후)

```
과거 날짜 완료 루틴:
┌────────────────────────────────────────────┐  border-left: 4px --color-success
│  [✓ disabled]  ~~아침 스트레칭~~           │  opacity: 0.85
└────────────────────────────────────────────┘

과거 날짜 미완료 루틴:
┌────────────────────────────────────────────┐  border-left: 4px --color-primary
│  [○ disabled]  물 2L 마시기               │  (배지 없음)
└────────────────────────────────────────────┘
```

**변경 사항:**
- 과거 미완료 배지 제거: `th:if="${!r.completed}"` 로 표시하던 "미완료" 배지 삭제
- 완료 배지도 제거 (홈 화면과 일관성 — home.md 참조)
- 과거 날짜 여부에 관계없이 배지 없이 운영

**구현 (history/index.html):**
```html
<!-- 삭제 대상 -->
<span th:if="${r.completed}" class="badge bg-success-subtle text-success">완료</span>
<span th:if="${!r.completed}" class="badge bg-secondary-subtle text-secondary">미완료</span>
```
두 줄 모두 제거.

### 화면 4 — 빈 상태 (개선 후)

```
┌────────────────────────────────────────────┐
│                                            │
│              [bi-calendar-x]              │  ← 아이콘 3rem, --color-text-3
│                                            │
│         이 날은 루틴이 없었어요             │  ← 15px, --color-text-2, fw-semibold
│   오늘 루틴을 추가하면 내일부터 기록돼요    │  ← 14px, --color-text-2

└────────────────────────────────────────────┘
```

- 아이콘: `bi-calendar-x` (해당 날짜에 예정된 루틴이 없음을 표현)
- 제목: `"이 날은 루틴이 없었어요"` (과거 날짜 맥락 자연스럽게)
- 오늘 날짜 빈 상태: `"오늘은 루틴이 없어요"` (home.md의 빈 상태와 다른 톤)
- CTA: `"오늘 루틴 보기 →"` → `/`

## 텍스트 가이드

### 날짜 헤더

- 날짜 형식: `"yyyy년 M월 d일 (E)"`, 한국어 Locale
- "오늘" 뱃지: `isToday == true`일 때만, `text-primary` 색상 (현행 유지)

### 빈 상태

| 날짜 조건 | 제목 | 부제 |
|---------|------|------|
| 과거 날짜 | `"이 날은 루틴이 없었어요"` | `"오늘 루틴을 추가하면 내일부터 기록돼요"` |
| 오늘 | `"오늘은 루틴이 없어요"` | `"새 루틴을 추가하면 내일부터 시작돼요"` |

과거 빈 상태에서는 POLICY §4-3 "기록 없음 → 첫 기록을 만들어보세요"와 정확히 일치하진 않지만,
기록 화면 맥락상 "기록이 없는" 이유가 루틴이 없어서이므로 루틴 추가를 유도하는 것이 적절하다.

### 아이콘 맥락

| 빈 상태 조건 | 아이콘 |
|-----------|-------|
| 해당 날짜 루틴 없음 | `bi-calendar-x` |
| 기본 | `bi-inbox` 사용 금지 (Habit Tracker 맥락 부적합) |

## 반응형 규칙

### 모바일 (< 576px)

- 날짜 네비게이션: `[<]` 날짜 `[>]`가 한 줄에 들어오도록 `justify-content-between`
- 날짜 텍스트: 긴 날짜(`2026년 6월 27일 (토)`)가 버튼에 눌려 줄바꿈 되지 않도록 `text-center` + 중앙 고정
- 루틴 카드: 풀 너비 유지
- 네비게이션 버튼: 44px 터치 타겟 필수 (모바일에서 특히 중요)

### 데스크탑 (> 768px)

- 날짜 네비게이션 버튼 크기 유지 (너무 커지지 않도록 max-width 설정 가능)
- 루틴 카드: container 너비 전체 사용
