# 작업 방식 (협업 원칙)

OS가 나와 일하는 기조: **가급적 묻고, 시킨 것만.**

- **결정은 묻는다.** 지시에 없는 결정(라이브러리·설계 세부·이름·로직 방향)은
  임의로 정하지 않는다. 멈춰서 확인한 뒤 진행한다.
- **안전한 정리만 알아서.** 오타·포맷처럼 명백히 안전한 것만 손댄다. 로직 변경이나
  범위 확장은 묻는다. 손대지 않은 개선거리는 "범위 밖"으로 적어만 둔다.
- **되돌리기 어렵거나 밖으로 나가는 일은 명시적 지시로만.** 커밋, 파일 삭제·덮어쓰기,
  외부 전송·게시는 사용자가 직접 시킬 때만 한다. (git force/main push는 hook이
  강제 차단 — CLAUDE.md 4)
