# log-trade (④ 학습·기록 — 대화형 스킬)

stock-os 루프 ④의 진입점. 커널 규칙 **"기록 없는 매매 없음 — 매매 발생 시 holdings +
trade-journal 동시 갱신"**을 구현한다. 체결 상세뿐 아니라 **그때의 생각·감정**을 되물어
채우는 대화형 작업이므로 subagent가 아니라 **메인 컨텍스트 스킬**로 둔다(subagent는
사용자와 대화 불가 → os.md §4의 journal-keeper가 의도적으로 미분리인 이유).

> ⚠️ 이 스킬은 **이미 체결된 매매를 기록**할 뿐이다. 주문·매매 실행은 100% 사람(os.md §6).
> `holdings.md`가 로컬 전용(gitignore)이라 **로컬 실행 전용**이다.

## 절차

### 1. 체결 상세 수집 (대화)
사용자에게 되물어 채운다(한 번에 하나씩, 이미 말한 값은 다시 묻지 않음):
- 종목명·티커·시장(코스피/코스닥/US)
- **매수/매도** 구분 + **거래 성격**: 정기매수 / 리밸런싱 / thesis매도
- 체결가·수량·금액, (미국) 체결 시 환율
- **그때 감정**(1~5, FOMO/공포/확신/충동 등) — trade-journal의 핵심 입력값
- 매수·매도 **근거** / 매수 시 thesis
- 원칙 준수 여부(원칙에 정의된 행동이었나 / 감정 매매 아니었나)

> 프로필 약점(FOMO·물타기)을 판단하지 말고 **있는 그대로 기록**한다. 학습은 월말
> monthly-reviewer의 몫. 다만 감정 점수가 높으면(충동) 그 사실을 담담히 남긴다.

### 2. 매도면 실수익 계산 연결
**매도 기록이면 `realized-return` 스킬로 실질 실현손익**(세후·수수료·환율 차감)을
산출해 trade-journal "결과" 칸에 넣는다. 표면 수익률로 적지 않는다(커널: 실수익 기준).

### 3. trade-journal 갱신 (append, 최신순)
`03_journal/trade-journal.md`의 **기록 템플릿**을 그대로 채워 "## 기록" 아래 **맨 위**에
추가한다. 기존 기록은 건드리지 않는다.

```
### [YYYY-MM-DD] 종목명(티커) — 매수/매도 | 구분: 정기매수/리밸런싱/thesis매도
- 가격 / 수량 / 금액:
- 매수·매도 근거:
- 그때 감정 (1~5, ...):
- 원칙 준수 여부:
- 결과 (매도 시 실현손익 %·금액, 실질 기준):   ← realized-return 결과
- 복기 한 줄 (다음에 뭘 다르게?):
```

### 4. holdings 갱신 (상태)
`02_portfolio/holdings.md`를 **그 즉시** 갱신(파일의 "갱신 규칙" 준수):
- **매수**: 해당 종목의 수량·평단가 갱신(신규면 행 추가). 평단가 = 가중평균.
- **매도(일부)**: 수량 차감(평단가는 유지).
- **매도(전량 청산)**: 해당 행 제거 — 내역은 이미 3단계에서 trade-journal로 옮겨짐.
- 비중·"한:미 비중"·"최근 갱신" 날짜를 갱신. (현재가·평가손익은 데일리에서 최신
  시세로 갱신되므로 여기선 손대지 않아도 됨 — 필요 시 사용자 확인.)

### 5. 마무리
두 파일이 **모두** 갱신됐는지 확인하고(불변조건: "매매가 있었다면 holdings+journal을
함께 갱신"), 무엇을 어떻게 기록했는지 2~3줄로 요약해 present한다.

## 불변 조건 (이 스킬 종료 전 검증)
- [ ] trade-journal에 감정·근거까지 채워 기록했는가
- [ ] 매도면 결과를 **실질(세후·수수료·환율)** 기준으로 적었는가
- [ ] holdings의 수량·비중·날짜를 갱신했는가
- [ ] 두 파일을 **함께** 갱신했는가(하나만 바뀐 채 끝내지 않기)
- [ ] 주문·실행을 대신하지 않고 **기록만** 했는가
