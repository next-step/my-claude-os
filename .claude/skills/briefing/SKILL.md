# /briefing — 주제 브리핑 리포트 스킬

주제 하나를 받아 **여러 각도로 나눠 조사**한 뒤, 핵심 요약·근거·출처가 담긴
모던 HTML 브리핑 리포트를 만들어 브라우저로 띄운다.

전달된 인자: `$ARGUMENTS` — 브리핑할 주제. (예: `/briefing 2026 AI 반도체 시장`)
인자가 없으면 사용자에게 주제를 한 번 물어본다.

이 스킬의 핵심은 **공유 에이전트 `web-researcher`를 여러 관점으로 병렬 호출**하는 것이다.

```
  /briefing "<주제>"
        │  ① 주제를 2~3개 각도로 쪼갠다
        ▼
  web-researcher × N  (한 메시지에서 동시에 병렬 호출)
   각도1   각도2   각도3
        │
        ▼  ② 결과 JSON들을 data.json으로 병합
        ▼  ③ generate_report.py 실행 → HTML + 브라우저 열기
```

> `web-researcher`는 `/factcheck` 스킬도 함께 쓰는 **공유 서브에이전트**다. 조사 방식이 같으므로
> 스킬마다 새 에이전트를 만들지 않고 재활용한다.

---

## 진행 순서

### 1. 주제를 각도로 쪼갠다
주제를 서로 겹치지 않는 **2~3개의 조사 각도**로 나눈다. 예) "2026 AI 반도체 시장" →
① 시장 규모·성장, ② 주요 기업·경쟁 구도, ③ 리스크·규제. 각도가 애매한 주제면 1~2개로 줄여도 된다.

### 2. web-researcher 병렬 호출
**반드시 한 번의 응답 안에서 각도 수만큼 Agent 호출을 동시에** 실행한다(병렬).
각 호출의 프롬프트에 다음을 명시한다:
- `brief` — "주제 · 이 각도" (예: "2026 AI 반도체 시장의 주요 기업과 경쟁 구도")
- `angle` — `neutral` (브리핑은 균형 잡힌 현황 조사다)
- `max_findings` — 각 각도당 3~4개

각 에이전트는 정해진 스키마의 JSON을 반환한다(`.claude/agents/web-researcher.md` 참고).
한 각도가 실패/빈결과면 그 섹션만 비우고 나머지로 진행한다.

### 3. 결과를 data.json으로 병합
각 에이전트 결과를 합쳐 `.claude/skills/briefing/data.json`을 Write 한다:
- `topic` — 사용자가 준 주제.
- `updated_at` — 오늘 날짜(YYYY-MM-DD).
- `lead` — 각 각도의 `summary`를 종합한 2~3문장 총평(직접 작성).
- `sections` — 각도별로 `{ "angle": "각도 제목", "summary": "...", "findings": [...] }` 배열.
- `sources` — 모든 에이전트의 `sources`를 합쳐 URL 기준 중복 제거.

**키 이름과 구조를 그대로 지킬 것** — 렌더러가 이 구조에 의존한다. 기존 `data.json`을 템플릿으로 삼는다.

### 4. 리포트 생성 & 브라우저 열기
```
python3 .claude/skills/briefing/generate_report.py
```
스크립트가 `data.json`을 읽어 `.claude/briefing-report.html`을 만들고 기본 브라우저로 연다.

### 5. 결과 보고
브리핑 주제, 각도 수, 총 근거 수, 리포트 경로, 브라우저 열기 여부를 요약한다.
가장 눈에 띄는 인사이트 1~2줄을 곁들인다.

---

## 설계 메모
- **데이터/표현 분리**: 에이전트는 `data.json`만 갱신, 디자인은 `generate_report.py`만 바꾼다.
- **공유 에이전트 재활용**: 조사 로직은 `web-researcher` 하나에 모아 두고, `/briefing`·`/factcheck`가 각자 다른 방식으로 호출한다. 리서치 품질을 한 곳에서 개선하면 모든 스킬이 함께 좋아진다.
- **병렬 각도 조사**: 한 각도씩 순차로 조사하면 느리다. 각도별 전담 호출을 동시에 돌려 빠르게 모은다.

## 예외 처리
- 모든 각도가 실패하면 마지막 `data.json`으로 리포트를 만들고 "데이터 갱신 실패" 사실을 알린다.
- `python3`가 없으면 리포트 경로를 알려주고 직접 열도록 안내한다.
