# /verify — 검증 일꾼 (판단만, 결정은 호출자)

이 스킬은 검증의 **"판단(judging)" 절반**이다. 검사를 실행하고 **판정(verdict)을 산출**한다.
**"결정(deciding: 통과시킬지·재시도할지·위로 올릴지)"은 하지 않는다** — 그건 호출자(`/impl` 디스패처)의 몫이다.
여기서 루프를 돌리거나 커밋하지 않는다.

> 왜 분리: 판단(검사 실행·기준 대조)은 PR Checks·CI 등으로 **계속 자란다.** 자라는 일을 한 곳(/verify)에
> 모으면, 호출자(`/impl`)는 "판정 보고 분기"만 하는 얇은 제어로 남는다. (god-object 방지)

전달된 인자: ``
- **무엇을** 검증할지(기능 slug 또는 "현재 변경")와 **강도** `light` | `full`.
- 강도가 없으면 → **full**(안전한 쪽)로 본다.

---

## 강도 (intensity)

### light — 테스트만 (저위험)
동작 보존 리팩터링·순수 기계적 변경(import 정리·포매팅·rename)·문서/설정·**잘 덮인** 좁은 버그픽스 등.

- 프로젝트 테스트를 실행한다(예: `.venv/bin/pytest -q`). **통과/실패 수치만** 확인.
- 적대적 기준 대조는 **생략**한다 — 그린 테스트가 동작 보존의 증거다.
- **전제**: 영향 영역이 기존 테스트로 잘 덮일 때만 light가 의미 있다. 커버리지가 약하다고 판단되면
  그 사실을 판정 근거에 적고 **full을 권고**한다.

### full — 테스트 + 적대적 기준 대조 (고위험·새 동작)
새 기능·동작 변경·보안/인증·외부 입출력·약한 커버리지 등.

- **공유 검토자 서브에이전트(`reviewer`)** 를 `Task`로 띄운다(`subagent_type: reviewer`). 넘길 것:
  - **[기준]** Plan 수용 기준·PRD 시나리오 (파일 경로 또는 본문)
  - **[대상]** 구현한 코드+테스트 (경로)
  - **[객관 신호]** 테스트 명령(예: `.venv/bin/pytest -q`) — reviewer가 직접 실행해 신호로 씀
  - **[판정 형식]** 통과 | 구현결함 | 설계문제 (아래 3갈래)
- reviewer가 빠진 기준·엣지케이스·위반을 찾아 판정을 돌려준다. (적대적 검토·"테스트 재탕 금지" 행동은
  [reviewer 정의](../../agents/reviewer.md)에 내장 — 여기선 기준·대상만 준다.)
- (예정) PR Checks·CI 결과도 이 단계의 신호로 합류시킨다 — 추가는 여기서만.

---

## 판정 (verdict) — 3갈래

| 판정 | 의미 | 호출자가 할 일(참고) |
|------|------|---------------------|
| **통과** (satisfied) | 테스트 통과 + (full이면) 기준 충족 | 다음으로 |
| **구현결함** (impl-bug) | 실패/미충족이지만 **구현을 고치면 됨** | 재구현 루프 |
| **설계문제** (design-problem) | 충족하려면 **Plan/PRD를 고쳐야 함**(요구·설계 자체 문제) | 상위 단계로 에스컬레이션 |

보고 형식(이것만):

```
## 판정: 통과 | 구현결함 | 설계문제
## 근거 / 미충족 항목
(구현결함·설계문제면 항목마다: 무엇이 / 왜 / 어떻게. 없는 문제 지어내지 마라. 통과면 "없음")
## 검사 결과
(테스트 통과 N/M · 강도 light|full · 커버리지 코멘트 한 줄)
```

---

## 수용 기준 (이 스킬이 제대로 동작하는지 검증)

- [ ] `light`는 테스트만 실행해 판정한다(적대적 검증 생략).
- [ ] `full`은 테스트 + **독립 적대적 대조**로 판정한다.
- [ ] 판정이 **통과/구현결함/설계문제 3갈래**로 명확히 나온다.
- [ ] 커버리지가 약하면 light라도 그 사실을 적고 full을 권고한다.
- [ ] **루프·분기·커밋을 하지 않는다**(판단만, 결정은 호출자).

---

## 안전 규칙

- **판단만 한다.** 다음 단계 결정·루프·커밋은 호출자(`/impl`)에게 맡긴다.
- **없는 문제를 지어내지 않는다.** light면 테스트 신호에 충실하게.
- **강도가 애매하면 full.** 안전한 쪽으로.
- 서브에이전트가 빈 결과/오류를 돌려주면 그대로 알린다.
