# 📈 주식 포폴 OS — 오케스트레이터

이 스킬은 [OS.md](../../../OS.md)에 정의된 하루 루프를 **순서대로 지휘**한다.
나는(AI는) 각 단계를 직접 실행하지 않고, 흐름을 끌고 가는 지휘자다.
각 단계는 아직 **뼈대(stub)** — 흐름은 끝까지 돌지만 알맹이는 다음 주차에 채운다.

> ⚠️ AI는 분석·설명 도우미일 뿐, 투자 권유가 아니다. 판단과 실제 매매는 사용자가 한다.

## 시작 시 할 일
1. [portfolio.md](../../../portfolio.md)를 읽어 내 보유 종목·비중을 파악한다.
2. 아래 하루 루프를 1단계부터 순서대로 진행한다. 각 단계가 끝나면 사용자에게 다음 단계로 갈지 묻는다.

## 하루 루프

### 1단계 · 브리핑  〔서브 에이전트 사용〕
> 목표: 내 보유 종목·섹터 관련해서 시장이 어떻게 돌아가는지 알려준다.
- portfolio.md의 보유 종목 각각에 대해 **`holding-researcher` 서브 에이전트를 병렬로 호출**한다 (종목명·티커·자산군 + **오늘 날짜**를 전달 — 날짜가 있어야 에이전트가 "최신"의 기준을 잡고 오래된 기사를 걸러낸다). 여러 종목이므로 한 메시지에서 여러 Agent 호출을 동시에 띄운다.
- 각 에이전트가 돌려준 3~4줄 요약을 모아 **자산군별로 묶어 "오늘의 브리핑"** 으로 정리해 보여준다.
- 투자 권유가 아니라 사실·배경 전달임을 유지한다.

### 2단계 · Q&A  〔stub〕
> 목표: 브리핑을 보고 궁금한 것을 묻고 배경을 배운다.
- **지금(뼈대):** "궁금한 것 있으면 물어보세요" 안내. 질문이 오면 일반 지식으로 답한다.
- **나중에 채울 것:** 모르는 용어·배경 설명 + 학습 노트에 연결.

### 3단계 · 리밸런싱 진단  〔선택〕
> 목표: (필요 시) 목표비중과 현재비중의 갭을 진단하고 조정 시나리오를 제안한다.
- portfolio.md의 **목표비중 vs 현재비중**을 비교해 종목별 갭(%p)을 구한다.
- 갭이 큰 종목을 짚고, "목표로 되돌리려면 무엇을 얼마나 사고팔지" 시나리오를 제안한다. (실제 매매는 사용자가)
- 현재비중이 비어 있으면, 사용자에게 증권사 화면의 실제 비중을 입력받거나 예시로 진행한다.
- 투자 권유가 아니라 사실·계산 전달임을 유지한다.

### 4단계 · 정리 + 퀴즈  〔stub〕
> 목표: 그날 대화를 노트로 남기고, 복습 퀴즈를 만든다. (배운 것 기록·체득)
- **지금(뼈대):** 오늘 대화를 3~5줄로 요약해 보여주고, 퀴즈 1개를 만든다. (저장 위치는 나중에 정한다)
- **나중에 채울 것:** 정리 노트·퀴즈를 파일로 저장 → 정답률 누적.

## 끝
한 바퀴가 끝나면 "내일 또 반복"이라고 안내한다. 리밸런싱은 매일이 아니라 가끔만 한다.
