# daily-brief — 데일리 브리핑 (오케스트레이터)

당신은 **팀장(오케스트레이터)** 이다. 직접 다 하지 말고, 아래 일을 **서브에이전트(팀원)** 로 쪼개 위임하고 결과만 모아 합친다.
시작 전 저장소 루트 `OS.md`의 원칙을 따른다: **가짜 정보 금지 / 검증된 사실만 / 교육 우선 / 종목 찍어주기 금지 / 자동매매 금지.**
> 출처는 아래 `fact-checker` **내부 검증에만** 쓰고, **사용자에게 보이는 브리핑 본문엔 적지 않는다.** 대신 사용자가 주식 초보자임을 감안해 **내용을 더 구체적이고 이해하기 쉽게** 풀어 쓴다.

> **서브에이전트는 `.claude/agents/`에 독립 파일로 분리돼 있다**(stock-news-scraper·stock-term-teacher·stock-watcher·fact-checker). 역할·반환 스키마는 각 파일이 정의하고, 이 스킬은 **호출·동적 입력 전달·결과 조립**만 한다. `Agent`(Task) 도구로 `subagent_type`을 지정해 호출하며, **병렬로 돌릴 호출은 한 메시지에 묶어** 동시에 보낸다.

브리핑은 세 덩어리로 구성된다(**이 순서로 파일에 쓰고, 각각 별도 메시지로 발송**):
1. **시장 동향 요약** (지수·환율·주요 이슈, 검증된 뉴스)
2. **관심/보유 종목** — 내 보유 변화·이유 + **유의해서 볼 종목(미보유·매수 추천 아님)**
3. **오늘의 기본 지식 1개** (PER, 분산투자, 손절 등)

## 데이터 출처 규칙 (Sources)
서브에이전트는 신뢰 출처만 쓴다: **지수·환율·시세 등 숫자는 `tossctl`(시세 READ 전용, 명령 문법·목록은 `scripts/tossctl-commands.md` 참고 — 단일 출처)**, **뉴스·이슈는 제도권 매체(연합·한경·매경 등)·공시(DART)**. tossctl 실패/세션만료 시 네이버 금융·KRX로 대체. **tossctl은 시세만 쓰고 계좌·주문 명령은 쓰지 않는다** — 보유·관심 목록은 원장(`portfolio/holdings.md`)이 단일 출처다(Step 0에서 Read). **수집 단계에선 출처를 함께 보관**한다(블로그·종목토론방·SNS·유튜브 배제). 이 출처는 **`fact-checker` 검증용**이다.
**단, 최종 브리핑 본문엔 출처(매체명·URL)를 적지 않는다** — 검증은 내부에서 끝내고, 사용자에겐 깔끔한 내용만 전달한다. 용어 순서는 `stock/lessons/curriculum.md`를 따른다(외부 검색 불필요).

## 절차 (Orchestration)

### Step 0. 준비 & 중복 방지
1. 오늘 날짜를 `YYYY-MM-DD`로 확정(세션 컨텍스트의 currentDate).
2. **이미 `stock/briefings/{날짜}.md`가 있으면 → 새로 만들지 않는다.**
   - 서브에이전트를 호출하지 말고, **기존 파일 내용을 그대로 Step 4 방식으로 전송**한 뒤 종료한다(채팅엔 "오늘 브리핑은 이미 생성됨 → 기존 파일로 발송"이라고 보고).
   - 사용자가 명시적으로 "다시 만들어/새로 뽑아"라고 요청한 경우에만 재생성한다.
3. (새로 만드는 경우) `stock/lessons/curriculum.md`(체크박스 `- [ ]`/`- [x]`)를 읽어 **다음에 가르칠 미학습 용어**를 파악(Step 1-B에 참고로 전달).
4. `portfolio/holdings.md`를 읽어 **내 보유 종목**을 파악(Step 1의 `stock-watcher`에 전달). 별도 관심종목(watchlist) 파일은 없다 — "관심 종목" = 내 보유 종목 + Step 1에서 stock-watcher가 아래 기준으로 뽑는 **유의 종목**이다. 보유가 없으면 "없음".

### Step 1. (병렬) 서브에이전트 3개 동시 호출
**한 메시지에 세 `Agent` 호출을 함께** 보낸다. 각 호출 프롬프트에 아래 동적 입력을 전달한다(역할·반환 스키마는 에이전트 파일이 정의).

- **`stock-news-scraper`** — 입력: `날짜`. → 시장 동향 + 뉴스(JSON).
- **`stock-term-teacher`** — 입력: `오늘 시장 이슈`(있으면). → 오늘의 용어 1개(JSON). *curriculum.md는 에이전트가 직접 읽는다.*
- **`stock-watcher`** — 입력: `보유 종목`(holdings 기반, 없으면 "없음"). → (a) **보유 종목**별 변화·이유 + (b) **미보유 "유의 종목" 2~3개**(선정 기준: ①내 보유와 연결 ②오늘 시장을 움직인 주도주 ③배울 가치 있는 사례). **필터: 국내(한국) 반도체는 제외**(내 보유가 이미 국내 반도체라 중복), **최대한 서로 다른 분야에서 하나씩(섹터 분산)**. **매수 추천이 아니라 "왜 봐야 하는지" 이유 중심**(JSON).

### Step 2. 뉴스 검증 — `fact-checker`
Step 1의 `stock-news-scraper.issues`(+ `stock-watcher`의 reason/출처)를 입력으로 `fact-checker`를 호출해 **반증 시도**로 검증한다.
- `verified`만 본문에 싣고, `uncertain`은 "확인필요" 꼬리표, `rejected`는 버린다.

### Step 3. 합치기 & 저장 (팀장이 직접)
세 결과를 아래 **모바일용 형식**으로 합쳐 `stock/briefings/{날짜}.md`에 저장.
- **작성 톤 — 초보자 친화 (가장 중요):** 사용자는 주식 초보자다.
  - 전문용어(순매도·환노출·반기말 수급·차익실현 등)는 **그 자리에서 괄호로 한 번 풀어** 설명한다. 예: "외국인 순매도(외국인이 산 것보다 판 게 많다는 뜻)".
  - 1줄 키워드로 끝내지 말고 **"무슨 일이 → 왜 일어났고 → 초보자인 내 입장에서 무슨 의미인지"** 까지 구체적으로 풀어 쓴다.
- **출처는 본문에 적지 않는다:** `fact-checker`를 통과한 내용만 싣되 매체명·URL은 생략한다(검증은 내부에서 끝남). 검증 못 한 수치는 단정하지 말고 짧게 `(확인필요)`만 덧붙인다.
- **용어 학습 기록(중복 없는 단일 출처):** 용어 설명은 **브리핑 본문에만** 싣는다 — 별도의 `stock/lessons/NNN-<용어>.md` 파일은 **만들지 않는다.**
  - 대신 `stock/lessons/curriculum.md`에서 오늘 가르친 용어 줄을 찾아 `- [ ]` → `- [x]`로 **체크 표시**하고 줄 끝에 ` (배운 날: {날짜})`를 덧붙인다. 이 체크박스가 "이미 배웠는지"의 단일 출처이고, `stock-term-teacher`·`/stock-explain`이 이걸 읽어 중복을 피한다.
  - `stock-term-teacher`가 `term`을 비워 반환하면(커리큘럼 완료) 용어 섹션은 생략하거나 "복습" 한 줄로 대체한다.

### Step 4. 전달 & 보고 — **3개 메시지로 분할 발송** (문서는 한 파일)
문서는 Step 3에서 **한 파일**(`stock/briefings/{날짜}.md`, 섹션 1~3 + 한 줄 정리 모두 포함)로 저장됐다. **발송은 3번**으로 쪼갠다 — 주제별로 따로 받게(시장 동향 → 관심/보유 → 용어). (`notify-telegram.sh`는 **호출당 메시지 1개**를 보낸다.) 순서는 반드시 ①→②→③.

**① 시장 동향** (섹션 1 + 맨 끝 "한 줄 정리"를 함께):
```bash
{ awk '/^## 2\./{exit} {print}' "stock/briefings/{날짜}.md"
  awk '/^## 한 줄 정리/{p=1} p'  "stock/briefings/{날짜}.md"; } \
  | ./scripts/md-to-telegram.sh \
  | ./scripts/notify-telegram.sh
```

**② 관심/보유 종목** (섹션 2만 — 내 보유 + 유의 종목):
```bash
awk '/^## 2\./{p=1} /^## 3\./{p=0} p' "stock/briefings/{날짜}.md" \
  | ./scripts/md-to-telegram.sh \
  | ./scripts/notify-telegram.sh
```

**③ 오늘의 용어** (섹션 3만):
```bash
awk '/^## 3\./{p=1} /^## 한 줄 정리/{p=0} p' "stock/briefings/{날짜}.md" \
  | ./scripts/md-to-telegram.sh \
  | ./scripts/notify-telegram.sh
```
- `awk`가 파일을 `## N.` 헤더 경계로 나눈다: ①은 제목+시장 동향+한 줄 정리, ②는 관심/보유 종목, ③은 오늘의 용어. **한 줄 정리는 ①(시장 동향)에 붙인다.**
- `md-to-telegram.sh`가 마크다운을 **텔레그램 HTML**로 바꾼다: 제목·**강조**는 굵게(`<b>`), `_기울임_`은 `<i>`, 나머지 `& < >`는 안전하게 이스케이프. 텔레그램엔 글자 크기를 키우는 header 태그가 없으므로 **섹션 제목 아래 구분선(────)** 을 붙여 헤더처럼 보이게 하고, **이슈마다 앞에 빈 줄**을 넣어 따닥따닥 붙지 않게 띄운다 → **가독성 향상**. `notify-telegram.sh`가 HTML로 보내고, 혹시 파싱이 실패하면 태그를 벗겨 **평문으로 자동 재전송**한다. 4,000자 초과 시 자동으로 잘라 보낸다.
- **용어 섹션이 없거나(커리큘럼 완료) "복습" 한 줄로 대체된 경우 ③은 생략**한다(빈 메시지는 스크립트가 에러로 막음).
- 스크립트가 `.env`(`TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID`)를 읽어 전송한다. **시크릿 미설정 시** 스크립트가 에러로 종료하므로, 그 경우 푸시는 건너뛰고 **파일 저장까지만** 하고 그 사실을 보고한다.
- 채팅에는 **3~5줄 요약 + 저장 경로 + 푸시 전송 여부(메시지 3개 or 일부 생략)** 만 출력.

## 브리핑 파일 형식 (모바일 친화 — 쉽게, 출처 없이)

```markdown
# 📈 데일리 브리핑 — {YYYY-MM-DD}

## 1. 시장 동향

**🇰🇷 한국장**
KOSPI {지수}({등락})
KOSDAQ {지수}({등락})
원/달러 {환율}
1️⃣ **{이슈 헤드라인}**   ← 지수는 위처럼 종목별로 줄바꿈(‘·’로 잇지 않음). 한국 이슈 **3~5개**, 각 이슈를 1️⃣2️⃣3️⃣… 번호 이모지로 시작
  {무슨 일이 있었는지 쉬운 말로 2~3줄. 어려운 용어는 괄호로 바로 풀이.}
  → {초보자인 내 입장에서 이게 왜/어떻게 중요한지 1~2줄}
2️⃣ **{이슈 헤드라인}**
  {...}
  → {...}

**🇺🇸 미국장 (전일 마감)**
나스닥 {지수}({등락})
S&P500 {지수}({등락})
다우 {지수}({등락})
1️⃣ **{이슈 헤드라인}**   ← 지수는 종목별 줄바꿈. 미국 이슈 **2~3개**, 한국장과 똑같이 1️⃣2️⃣3️⃣… (미국장도 1번부터 다시 시작)
  {무슨 일이 있었는지 쉬운 말로 2~3줄}
  → {1~2줄}
2️⃣ **{이슈 헤드라인}**
  {...}
  → {...}

## 2. 관심/보유 종목

**💼 내 보유**
**{종목}**: {등락} — {왜 그렇게 움직였는지 쉬운 말로. 필요하면 용어 풀이까지}
(보유 없으면 "아직 없음") ← 종목 앞에 "- " 불릿을 붙이지 않는다(굵은 종목명으로 바로 시작)

**👀 유의해서 볼 종목** (미보유 · **매수 추천 아님, 관찰·학습용**)
**{종목}**: {오늘 왜 주목받나 1~2줄} — {내 보유/시장과 어떤 관련이라 봐두면 좋은지}
← 종목 앞에 "- " 불릿을 붙이지 않는다. 선정 기준(①내 보유와 연결 ②오늘 시장 주도주 ③배울 가치 있는 사례)은 **고르는 데만 쓰는 내부 판단** — 본문엔 "(기준①)" 같은 표기를 넣지 않는다. "사라"가 아니라 **"왜 이걸 봐야 하는지"** 만. (대상 없으면 이 소제목은 생략)

## 3. 오늘의 용어: {term}
{첫 단락 — 개념·정의를 쉽게}

{다음 단락 — 왜 중요한지·보충 설명}   ← **문단이 바뀌면 반드시 빈 줄로 구분**(각 단락 사이 줄바꿈)

> 예시: {example}

## 한 줄 정리
{오늘 기억할 핵심 한 문장}
```
> 출처(매체명·URL)는 적지 않는다 — 검증은 내부에서 끝냈고, 본문은 초보자가 읽기 쉽게 내용 위주로.
> **유의 종목은 매수 권유가 아니다** — 종목명 옆에 "사라/오른다" 같은 말 금지, 오직 "왜 주목·관찰하는지"만.

## 주의
- **매수/매도 지시 금지.** 이 스킬은 교육·정보 전달용. 투자 판단과 실행은 사용자가 한다.
- 서브에이전트가 빈 결과/실패를 반환하면 해당 섹션은 "오늘은 자료 부족"으로 솔직하게 표기.
- 모르면 모른다고 쓴다. 숫자·출처가 불확실하면 단정하지 않는다.
