# ⚖️ 리밸런싱 진단 — rebalance

목표비중과 현재 상태의 갭을 **라이브 현재가로 계산**해 "무엇을 얼마나 사고팔면 목표로 돌아가는지"를 보여준다.

> ⚠️ AI는 계산·설명 도우미일 뿐, 투자 권유가 아니다. 판단과 실제 매매는 사용자가 한다.

## 계산은 스크립트가 (산수를 LLM이 하지 않는다)

```bash
python3 .claude/skills/rebalance/scripts/rebalance.py
```

이 스크립트가 결정론적으로 처리한다:
- **목표비중** ← [portfolio.md](../../../portfolio.md)
- **보유수량·예수금·총원금** ← [portfolio.private.md](../../../portfolio.private.md) (비공개)
- **현재가** ← 야후 파이낸스에서 종목별 실시간 조회 (한국 종목 `티커.KS`)
- 계산: 평가액 = 현재가 × 수량 → 현재비중 → 목표대비 갭(%p) → 매매 수량(목표까지, 내림)

숫자 계산은 전부 스크립트 몫이라 LLM이 암산하지 않는다(오차 방지).

## 할 일

1. 위 스크립트를 실행한다.
2. 스크립트가 뽑은 표를 **그대로 보여준 뒤**, 사람 말로 해설한다:
   - 갭이 큰(±로 눈에 띄는) 종목을 짚는다. "무엇이 목표보다 넘치고/모자란지".
   - 매매 수량이 0이 아닌 종목만 모아 "목표로 되돌리려면: A를 n주 매도, B를 m주 매수" 식으로 정리.
   - 장기투자 관점에서 **갭이 작으면(예: ±1%p 이내) 굳이 손댈 필요 없다**는 점을 알려준다. (거래비용·세금 고려)
3. 조회 실패한 종목이 있으면(스크립트가 표시함) 사용자에게 그 종목 현재가를 물어 보완한다.

## 규칙
- 결과는 **화면에만**. 평가액·현재가는 매일 변하는 파생값이라 어디에도 저장하지 않는다.
- 투자 권유가 아니라 사실·계산 전달. "이렇게 하면 목표비중이 맞춰진다"까지만, "그러니 사라"는 하지 않는다.
