# /code-run — 코드 실행 준비 스킬 (코드실행!)

`/code` 분석 결과를 받아, **원본 구현체를 사용자가 자기 터미널에서 직접 돌릴 수 있도록** 준비한다.

> 핵심 원칙 (이 스킬의 정체성):
> 1. **원본 우선** — 재구현 토이가 아니라 **논문의 실제 레포(공식/충실한 참조)** 를 대상으로 한다.
> 2. **사용자가 실행 주체** — 클로드는 **직접 실행하지 않는다.** 복붙 가능한 터미널 명령 블록과 가이드만 만든다. 실행·관찰은 사용자가 자기 터미널에서 한다.
> 3. 재구현 데모는 **원본 실행이 구조적으로 불가할 때만** fallback으로, 그 사실을 명시해서 만든다.

> 슬래시 이름: `/code-run` (원 요청의 `/코드실행!`에 해당).

## 입력
- `output/<slug>/04_code.md`(필수). 없으면 먼저 `/code` 실행.

## 절차
1. **계획**: `04_code.md`에서 원본 레포 URL, 진입점, 의존성, 최소 재현 경로를 읽고 실행 계획을 세운다.
2. **원본 확보 방법 결정** — 실제로 clone하지는 말고, 사용자가 실행할 **명령**을 설계한다:
   - 레포 URL, 고정 커밋/태그(재현성), 서브디렉토리, 진입 스크립트.
   - 필요한 모델/데이터 다운로드 단계와 그 용량·시간·GPU 요구를 미리 고지.
3. **환경 스펙 작성**: 파이썬 버전, 가상환경(venv/conda) 생성 명령, `pip install`(가능하면 원본 `requirements.txt`/`pyproject`를 그대로 참조), 알려진 설치 함정(예: 이 환경의 Store-stub python 등)을 정리.
4. **복붙용 터미널 명령 블록 작성** — 사용자가 처음부터 끝까지 순서대로 실행할 수 있는 **하나의 명령 시퀀스**를 만든다:
   clone → 브랜치/커밋 고정 → 가상환경 → 설치 → (필요시 모델/데이터 준비) → 실제 데모/추론 실행.
   - 각 줄에 무엇을 하는지 짧은 주석. 실패 가능 지점과 복구법 병기.
   - **클로드는 이 블록을 스스로 실행하지 않는다.** "이 블록을 님 터미널에 붙여넣어 실행하세요"라고 안내한다.
5. **관찰 가이드**: 실행하면 무엇을 보게 되는지(예상 출력/포트/UI/샘플 결과), 성공 판정 기준, 흔한 에러 → 해결을 적는다.
6. **기록**: 위 전부를 `output/<slug>/05_run.md`에 정리. 사용자가 실행 후 로그를 붙여주면 "결과 해석"을 채운다.

## 실행 주체 규칙 (중요)
- **기본값: 클로드는 실행하지 않는다.** Bash로 clone·pip·데모를 자동 실행하지 말 것. 산출물은 "사용자 터미널용 명령 + 가이드"다.
- 사용자가 명시적으로 "네가 직접 돌려봐"라고 요청한 경우에만 클로드가 실행하며, 그때도 무거운 다운로드/학습 전에는 먼저 알린다.
- 실행 결과 로그는 **사용자가 붙여넣어 준 실측치**를 정본으로 삼는다. 실행하지 않았으면 로그를 지어내지 말 것("아직 미실행"으로 표기).

## Fallback (원본 실행이 불가할 때만)
- 독점 데이터/하드웨어 필수, 레포 부재 등으로 원본 실행이 구조적으로 불가하면:
  - 그 사유와 "무엇을 갖추면 원본을 돌릴 수 있는지"를 먼저 명시하고,
  - `output/<slug>/run/`에 **재구현 토이 데모**(requirements + run_demo)를 만들되, "원본이 아닌 문서 기반 재구현"임을 05_run.md 상단에 굵게 고지한다.

## 출력 (`output/<slug>/05_run.md`)
```markdown
# 실행 준비 리포트: <제목>
## 대상 (원본 레포 URL / 고정 커밋 / 진입점)   ← 재구현이면 그 사실을 굵게 명시
## 환경 요구 (OS / Python / GPU / 디스크·다운로드 용량·예상 시간)
## 복붙용 터미널 명령 (clone→env→install→run, 한 블록)
## 실행하면 보게 될 것 (예상 출력 / 포트 / 성공 판정 기준)
## 흔한 에러 → 해결
## 실제 실행 로그 (사용자가 붙여넣은 실측치 — 미실행 시 "아직 미실행")
## 결과 해석 (출력이 논문 동작과 일치하는가)
```

## 품질 기준
- 명령 블록은 사용자가 **그대로 복붙해 자기 터미널에서** 처음부터 끝까지 실행 가능해야 한다.
- "돌렸다"는 주장은 **사용자 실측 로그**로만 뒷받침. 미실행이면 솔직히 "미실행"으로 표기.
- 재구현 fallback을 썼으면 반드시 원본이 아님을 상단에 고지.
- 완료 후 `output/<slug>/05_run.md` 경로와 "다음: 이 명령 블록을 터미널에서 실행하세요" 안내를 보고.
