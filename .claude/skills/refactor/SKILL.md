# Refactor 스킬

`backend-dev` / `frontend-dev` 에이전트에게 유지보수성·가독성 중심의 리팩토링을 요청한다.
기능 변경 없이 코드 구조와 표현을 개선하는 것이 목적이다.

---

## 모드 판별

| 상황 | 범위 |
|------|------|
| `/refactor` (인자 없음) | 전체 코드베이스 |
| `/refactor RoutineService` | 해당 클래스/파일만 |
| `/refactor XP 시스템` | 해당 기능 관련 파일 |

범위가 없을 때는 아래 명령으로 최근 변경 파일을 자동 탐지한다:

```bash
git diff main..HEAD --name-only | grep -E "\.java$|\.html$|\.css$|\.js$"
```

---

## Step 1: 리팩토링 대상 파악

범위가 정해지면 아래 기준으로 Backend / Frontend 파일을 분류한다.

| 경로 | 담당 |
|------|------|
| `src/main/java/` | Backend |
| `src/main/resources/templates/` | Frontend |
| `src/main/resources/static/css/` | Frontend |
| `src/main/resources/static/js/` | Frontend |

---

## Step 2: backend-dev 에이전트 실행 (Java 파일이 있을 때만)

```
아래 파일들을 유지보수성과 가독성 중심으로 리팩토링해줘.

대상 파일:
{Step 1에서 분류한 Backend 파일 목록}

리팩토링 원칙:
1. 기능은 절대 변경하지 않는다 — 동작이 동일해야 한다
2. 기존 테스트가 모두 통과해야 한다 (리팩토링 후 테스트 실행 필수)
3. 아래 항목을 우선 개선한다:
   - 메서드 길이: 20줄 초과 메서드는 분리
   - 매직 넘버/문자열: 상수로 추출
   - 중복 코드: 공통 메서드로 추출
   - 이름 개선: 의도가 드러나는 변수명·메서드명으로 변경
   - Optional 체이닝: null 체크 대신 Optional 활용
   - Stream 활용: 반복문을 Stream으로 간결하게
   - 불필요한 주석 제거: 코드가 자기 설명적이면 주석 삭제
4. 변경 사항이 없는 파일은 건너뛴다

완료 후 파일별로 어떤 개선을 했는지 bullet로 보고해줘.
```

## Step 3: frontend-dev 에이전트 실행 (HTML/CSS/JS 파일이 있을 때만)

```
아래 파일들을 유지보수성과 가독성 중심으로 리팩토링해줘.

대상 파일:
{Step 1에서 분류한 Frontend 파일 목록}

리팩토링 원칙:
1. 기능과 화면 결과는 절대 변경하지 않는다
2. 아래 항목을 우선 개선한다:

   [HTML/Thymeleaf]
   - 중복 마크업: Thymeleaf fragment로 추출
   - 인라인 스타일 제거: CSS 클래스로 이동
   - 의미없는 div 중첩 제거 (div soup)
   - 접근성 속성 누락: aria-label, role, alt 보완

   [CSS]
   - 중복 선언: CSS 변수(--var) 또는 공통 클래스로 통합
   - 마법 숫자 제거: 의미있는 변수명으로 추출
   - 사용하지 않는 클래스 정리

   [JavaScript]
   - 함수 분리: 한 함수가 하나의 일만 하도록
   - var → const/let 전환
   - 콜백 지옥 → Promise/async-await 전환
   - 하드코딩 셀렉터: data-* 속성으로 분리

3. 변경 사항이 없는 파일은 건너뛴다

완료 후 파일별로 어떤 개선을 했는지 bullet로 보고해줘.
```

## 실행 원칙

- Step 2(Backend)와 Step 3(Frontend)는 **병렬 실행 가능** — 수정 파일이 겹치지 않는다
- 리팩토링은 기능 변경이 아니다. 개발자가 "이 코드가 무엇을 하는지" 5초 안에 파악할 수 있어야 한다
- 테스트가 깨지면 리팩토링이 아니라 버그 도입이다 — backend-dev는 반드시 테스트 통과 후 보고한다
- 범위가 너무 넓어 한 번에 처리하기 어려우면 파일 단위로 나눠 순차 실행한다
