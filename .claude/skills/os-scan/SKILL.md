# /os-scan — 코드베이스 빠른 파악

전체 개발 사이클을 돌리지 않고 **1단계(코드베이스 파악)만** 단독으로 실행한다.
`/os`의 1단계와 **같은 `os-mapper` 서브에이전트를 공유**하므로,
새 저장소 온보딩이나 "지금 구조가 어떻게 되어 있지?"를 빠르게 확인할 때 쓴다.

## 절차

### 1. 위임
`Task` 도구로 `subagent_type: os-mapper`를 호출한다. 프롬프트에 포함:
- "`OS.md`의 [1단계] 규칙을 따라 저장소 구조·컨벤션을 파악하라"
- "`docs/CONVENTIONS.md`가 있으면 현재 코드와 대조해 달라진 부분만 갱신하고, 없으면 새로 작성하라"
- "재사용 가능한 자산(유틸·공통 모듈·테스트 헬퍼·빌드 의존성 위치)을 경로와 함께 목록화하라"
- "빌드/런타임 환경 프리플라이트(설치 런타임 vs 빌드도구 지원범위 vs IDE 설정) 불일치를 점검하라"

### 2. 결과 정리
에이전트 보고를 사용자에게 한눈에 요약한다:
- **구조·컨벤션 요약** (디렉토리·네이밍·테스트 관행·빌드 도구)
- **재사용 자산 목록** (경로 — 용도)
- **환경 프리플라이트** 결과 (불일치 있으면 권장 정렬값)
- `docs/CONVENTIONS.md` 생성/갱신 경로

## 참고
- 이 스킬은 **파악·문서화까지만** 한다. 코드 구현은 하지 않는다(그건 `/os`의 2단계).
- 대상이 넓으면 `os-mapper`를 영역별(구조/컨벤션/테스트관행)로 **여러 개 병렬** 호출해도 된다.
- `os-mapper`는 `/os`의 1단계에서도 쓰이는 **공유 에이전트**다.
