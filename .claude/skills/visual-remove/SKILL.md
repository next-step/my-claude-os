# visual-remove — 컴포넌트 시각 검증 등록 해제 (visual-add 의 짝)

지정한 컴포넌트(들)를 시각 검증 시스템(`demo-app`)에서 **등록 해제**한다.
`visual-add` 가 만든 등록(변형 목록 + 레지스트리 항목)과 검증 산출물(스크린샷)을 지운다.

**범위(중요):**
- 기본은 **"검증 시스템에서만 제거"** 다 — 변형 파일 / 레지스트리 항목 / `screenshots/<키>/`.
- **제품 컴포넌트 파일(`src/components/<Name>.tsx`)은 사용자의 실제 코드**이므로 **기본적으로 지우지 않는다.** 지울지는 4단계에서 따로 확인한다.
- **여러 개 동시 제거 가능** — 받은 키들을 각각 같은 절차로 처리한다.

## 절차

### 1. 대상 키 확정 (여러 개 가능)
- 사용자가 말한 컴포넌트(들)를 **키**로 환산한다(`Modal` → `modal`). 여러 개면 목록으로 모은다.
- 현재 등록 상태를 확인한다:
```bash
cd demo-app && grep -oE "^  [a-z][a-zA-Z0-9-]*: \{ label: '[^']*'" src/gallery/registry.ts
```
- 받은 키가 등록돼 있지 않으면 그 키는 건너뛰고 알려준다(없는 걸 지우려 하지 않는다).
- 지울 목록을 사용자에게 **먼저 보여주고 확인**받는다(되돌리기 어려우므로). 예: "card, modal 2개를 검증에서 제거합니다 — 진행할까요?"

### 2. 각 키별로 등록 해제
받은 키마다 다음을 한다.

**(a) 변형 파일 삭제**
```bash
cd demo-app && rm -f src/gallery/variants/<키>.ts
```

**(b) 레지스트리에서 import 2줄 + REGISTRY 항목 1줄 제거** (`src/gallery/registry.ts`)
- `import { <Name> } from '../components/<Name>'`
- `import { <키>Variants } from './variants/<키>'`
- REGISTRY 안의 `  <키>: { label: ... },` 줄
→ Edit 로 정확히 그 줄들만 지운다.

**(c) 스크린샷 산출물 삭제**
```bash
cd demo-app && rm -rf screenshots/<키>
```

### 3. 타입체크 (참조가 깨끗이 빠졌는지)
```bash
cd demo-app && ./node_modules/.bin/tsc -b --noEmit && echo "타입 OK"
```
- 에러가 나면(남은 import/참조) 찾아서 마저 지운다.

### 4. 제품 컴포넌트 파일은 지울지 확인 (기본: 보존)
- 등록 해제가 끝난 뒤, **"제품 컴포넌트 파일 `src/components/<Name>.tsx` 도 삭제할까요? (기본: 남김)"** 을 물어본다.
  - 다른 곳(`App.tsx` 등)에서 그 컴포넌트를 import 하는지 먼저 확인하고, 쓰이고 있으면 **삭제 위험을 경고**한다.
  - 승인하면 그때 지운다. 승인 없으면 컴포넌트 파일은 그대로 둔다.

### 5. 보고
- 키별로 무엇을 지웠는지 정리한다: 변형 파일 / 레지스트리 / 스크린샷 (그리고 컴포넌트 파일은 보존/삭제 여부).
- 남은 등록 대상 목록을 한 줄로 보여준다(= 현재 검증 가능한 것).

## 규칙
- **되돌리기 어려운 삭제**이므로 2단계에서 지울 목록을 확인받고 진행한다.
- **제품 컴포넌트(.tsx)는 기본 보존** — 명시 승인 시에만 삭제.
- 등록 안 된 키는 조용히 건너뛰지 말고 "등록돼 있지 않다"고 알린다.
