# /interview — 논문 파이프라인 의도 확정 인터뷰

`paper-os`(analyzer→detail→code→code-run→html)를 돌리기 **전에** 사용자 의도를 확정하는 대화형 스킬.
채팅으로 끝내지 않고 **`output/<slug>/00_intent.md`** 라는 산출물을 남겨, 이후 모든 단계가 그 의도를 기본값보다 우선 적용하게 만든다. (code-run이 "재구현 토이를 클로드가 자동실행"으로 어긋났던 사고를 시작 5초에 잡는 것이 목적.)

> 이 스킬은 **대화형**이다(사용자에게 되묻는다). 백그라운드 워크플로우 안에서 돌리지 말고, `paper-os` 실행 전에 메인 대화에서 단독 실행할 것.

## 입력
- 논문 링크(필수) + (있으면) 사용자의 초기 요청 문장.

## 절차 (세 동작 루프 → 확정 → 저장)

1. **맥락 흡수 & slug 산정**
   - 링크를 WebFetch로 가볍게 훑는다. *이미 아는 것은 묻지 않는다.*
   - paper-os와 **동일 규칙**으로 파일시스템 안전 slug를 짓는다(영문소문자/숫자/._- 만, 예: `liveedit_2606.26740`). 이후 산출물은 `output/<slug>/00_intent.md`.

2. **① 가정 추출** — 사용자 요청·논문에서 클로드가 **암묵적으로 채우게 될 가정**을 3~5개 드러낸다.
   - 형식: "이 요청은 이런 가정을 깔고 있는 것 같습니다: …"
   - 반드시 아래 **숨은 변수 체크리스트**를 훑어 가정을 뽑는다.

3. **② 빈틈 탐지** — 아직 정의되지 않아 결과를 좌우할 **미정 항목**을 짚는다.
   - 형식: "아직 안 정해진 건 이겁니다: …" (특히 **성공 기준** — "됐다"의 판정)

4. **③ 대안 제시** — 해석이 갈리는 지점만 **A vs B 갈림길**로 만들어 `AskUserQuestion`으로 묻는다.
   - 형식: "이 부분은 A와 B로 갈립니다. 어느 쪽에 가깝나요?" — 각 옵션에 기본값/추천 표시.
   - **결과물을 바꿀 갈림길만.** 사소한 건 묻지 말고 합리적 기본값으로 채우고 "이렇게 가정함"만 알린다.

5. **정지 조건** — ①②③은 최대 **2라운드**. 결과를 바꿀 만한 가정·빈틈이 더 없으면 즉시 멈춘다. (인터뷰가 20문항 설문이 되면 실패.)

6. **되짚기(confirm-back)** — "정리하면 님 의도는 …" 한 문단으로 재진술. 사용자가 틀린 데만 고친다.

7. **저장** — 확정 의도를 `output/<slug>/00_intent.md`로 Write. 완료 후 경로와 "다음: `/paper-os <링크>` 실행 — 각 단계가 이 파일을 읽습니다" 안내.

## 숨은 변수 체크리스트 (가정·빈틈을 여기서 뽑는다)

| 축 | 묻는 것 | 자주 어긋나는 예 |
|---|---|---|
| **What** 대상·범위 | 원본 레포 vs 재구현 / 어느 단계까지 / 어느 모듈 | code-run이 원본 아닌 재구현 토이로 감 |
| **Where** 실행·출력 위치 | 클로드 샌드박스 vs 사용자 터미널 / 출력 폴더 | 클로드가 자기 안에서 돌림 |
| **Who** 실행 주체 | 클로드가 실행 vs 명령만 주고 사용자가 | 클로드가 대신 실행 |
| **How** 형식·깊이 | 간단 vs 상세 / 언어 / 포맷 | |
| **Why** 성공 기준 | 무엇을 보면 "됐다"인가 | 판정 기준 부재 |
| **제약** | GPU/OS/시간/비용/네트워크 한계 | 환경 가정 어긋남 |

## 출력 (`output/<slug>/00_intent.md`)
```markdown
# 의도 스펙 (Intent Spec) — <논문 제목>
- 링크: <url>
- slug: <slug>
- 작성일: <YYYY-MM-DD>

## 확정 의도
| 축 | 확정 내용 |
|---|---|
| What 대상·범위 | … |
| Where 실행·출력 위치 | … |
| Who 실행 주체 | … |
| How 형식·깊이 | … |
| Why 성공 기준 | … |
| 제약 | … |

## 단계별 지침 (해당 단계 에이전트가 기본값보다 우선 적용)
- analyzer: …
- detail: …
- code: …
- code-run: …   ← 원본 vs 재구현 / 실행 주체(클로드 vs 사용자)를 여기서 못박음
- html: …

## 남긴 가정 (미확정 → 합리적 기본값으로 채운 것)
- …
```

## 품질 기준
- 질문은 **결과를 바꿀 것만**, 최대 2라운드. 아는 건 안 묻는다.
- 반드시 산출물(`00_intent.md`)을 남긴다 — 대화로만 끝내지 않는다(안 그러면 의도가 증발).
- 단계별 지침에는 특히 **code-run의 원본/재구현·실행 주체**를 명시한다.
- 완료 후 `output/<slug>/00_intent.md` 경로와 다음 단계(`/paper-os`) 안내를 보고.
