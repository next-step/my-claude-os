# /plan — PRD → Plan 단계 (② 설계)

[OS.md](../../../OS.md) 개발 흐름의 **② Plan 단계**만 담당하는 스킬이다.
(배경·원칙은 OS.md를 따른다. 여기서는 그 단계를 실제로 어떻게 굴리는지만 적는다.)

입력은 **`/prd`가 확정한 PRD**(`docs/prd/<slug>.md`)다. 출력은 구현 설계 문서
(`docs/plan/<slug>.md`)다. 설계가 끝나면 **사람 검토 게이트에서 멈춘다.**

> 이 스킬의 핵심 설계: 재활용 조사를 **하나의 에이전트에 통째로 맡기지 않는다.**
> 방대한 코드를 한 번에 다 읽는 건 불가능하므로, **메인(이 스킬)이 1차로 빠르게 스캔해
> "찾음/불확실/못찾음"으로 나눈 뒤, 불확실한 부분만 병렬 서브에이전트로 깊게 조사**한다.
> (서브에이전트는 또 서브에이전트를 못 띄우므로, 병렬 조율은 반드시 메인이 한다.)

전달된 인자: ``
- 인자가 있으면 → 기능 설명/slug로 사용한다.
- 인자가 없으면 → "어떤 기능의 Plan을 짤까요?"라고 한 줄로 묻고 시작한다.

이 스킬의 범위는 **② Plan(설계)까지**다. ④ 구현은 게이트 통과 후 별도 스킬에서 한다.

---

## 진행 순서

### [준비] slug 확인 + PRD 존재 확인

기능 설명에서 slug를 정하고, **확정 PRD가 있는지** `Bash(test:*)`로 확인한다.

```
docs/prd/<slug>.md     # 입력 (있어야 함)
docs/plan/<slug>.md    # 출력
```

- **PRD가 없으면** → "먼저 `/prd`로 PRD를 만들고 확정하세요"라고 안내하고 **멈춘다.**
- **PRD가 있으면** → 그 PRD를 `Read`로 읽어 본문을 확보하고 1단계로 간다.

---

### 1단계 (메인): 재활용 1차 스캔 + 갭 식별

이 단계는 **메인(이 스킬)이 직접** 한다. 서브에이전트에 넘기지 않는다.

1. PRD에서 **이 기능에 필요한 능력**을 항목으로 뽑는다. (예: "로그 파싱", "날짜 비교", "표 출력")
2. 각 능력마다 `Grep`/`Glob`로 **빠르게 1차 검색**한다. (관련 키워드·심볼·파일명)
3. 결과를 세 갈래로 분류한다:

| 분류 | 의미 | 다음 처리 |
|------|------|-----------|
| ✅ 찾음 | 재활용할 코드를 명확히 찾음 | 더 안 판다 (확정) |
| ❓ 불확실 | 있는 것 같은데 1차 검색으론 확신 못 함 | 2단계에서 깊게 조사 |
| ⬜ 못찾음 | 1차 검색에 안 잡힘 | 2단계에서 한 번 더 확인 (없으면 직접 구현) |

> 핵심: **확실한 곳(✅)엔 시간을 안 쓰고, 애매한 곳(❓⬜)만 다음 단계로 넘긴다.**

---

### 2단계 (메인이 병렬 fan-out): 불확실한 부분만 깊게 조사

**갭(❓·⬜)이 하나라도 있으면**, 항목마다 **`Explore` 서브에이전트를 하나씩, 한 번에(병렬) 띄운다.**
각 에이전트는 "그 한 부분"만 집중 조사한다. (여러 `Task` 호출을 한 응답에 함께 보내면 동시 실행.)

- `subagent_type`: `Explore`
- `description`: `"재활용 조사: <능력 이름>"`
- `prompt` (항목마다 채워서):

```
이 저장소에서 "<능력 이름>"에 재활용할 수 있는 기존 코드가 있는지 조사하라.
넓게 읽지 말고, 아래 단서로 표적 검색해 관련된 파일만 발췌해서 확인하라.

검색 단서(키워드/심볼): <1단계에서 뽑은 키워드들>

다음만 간결히 보고하라:
- 찾음: <파일경로> — 무엇이고 어떻게 재활용하나
- 못 찾음: "없음 — 직접 구현 필요"
없는 걸 지어내지 마라. 표적 검색 결과만 보고하라.
```

- **갭이 0개면 이 단계를 통째로 건너뛴다.** (작은 레포·재활용 거리가 없을 때 비용 낭비 방지)
- 각 에이전트가 돌려준 결과를 모아, 1단계의 ✅ 찾음과 합쳐 **"재활용 지도"** 를 완성한다.
  (각 항목: 재활용 가능 → 파일·사용법 / 불가 → 직접 구현)

> 왜 병렬·표적인가: 코드가 방대해도 **다 읽지 않는다.** 불확실한 곳만 골라, 각자 좁은 영역을
> 깊게 보는 에이전트를 병렬로 돌린다 → 빠르고 정확하며 컨텍스트 낭비가 없다.

---

### 3단계: Plan 설계 — Plan 서브에이전트 호출

재활용 지도가 확정됐으면, `Task` 도구로 **Plan 설계 서브에이전트**를 띄워 설계를 맡긴다.
**재활용 조사는 이미 끝났으니, 지도를 신뢰하고 설계만** 하게 한다(코드를 다시 뒤지지 않게).

- `subagent_type`: `Plan`
- `description`: `"구현 Plan 설계"`
- `prompt`:

```
당신은 구현 Plan 설계자다. 아래 PRD와 "재활용 지도"를 바탕으로 구현을 설계하라.
재활용 조사는 이미 끝났다. 아래 지도를 신뢰하고 설계에 집중하라(코드를 새로 뒤지지 마라).

[중요] 지도에서 "재활용 가능"으로 표시된 것은 새로 만들지 말고 그대로 활용/확장하라.
(OS.md 원칙 2: 재활용 우선)

PRD (확정본):
<[준비]에서 읽은 PRD 본문>

재활용 지도 (1·2단계 조사 결과):
<능력별: 재활용 가능(파일·사용법) / 직접 구현 필요>

다음 형식의 마크다운으로만 답하라:
# Plan — <기능 이름>
## 재활용할 기존 코드 (지도 기반: 파일/스킬과 어떻게 쓸지)
## 직접 구현이 필요한 부분 (지도에서 "없음"으로 확인된 것)
## 손댈 파일 (신규/수정)
## 구현 단계 (순서대로)
## 테스트 전략 (단위/통합 — OS.md 원칙 3, 필수)
## 위험 요소 / 롤백
```

서브에이전트가 돌려준 Plan 본문을 **그대로** `docs/plan/<slug>.md` 에 `Write` 로 저장한다.

---

### 검토 게이트 — ★반드시 멈춤★

Plan을 만들었으면 **요약해서 사람에게 보고하고 STOP 한다.**

```
✅ ② Plan 작성 완료 → docs/plan/<slug>.md

[재활용 조사 요약]
- 재활용: ... (찾은 것)
- 직접 구현: ... (못 찾아 새로 만들 것)
- 병렬 조사한 항목: N개 (갭이 없었으면 "1차 스캔으로 충분, 병렬 조사 생략")

[Plan 요약]
- 손댈 파일: ...
- 테스트: ...
- 위험: ...

여기는 검토 게이트입니다. Plan을 확인해 주세요.
- ❓ 이 "재활용 목록"에 빠진 게 있나요? (당신이 아는 기존 코드가 있으면 알려주세요)
- 수정이 필요하면 어디를 고칠지 알려주세요(다시 설계합니다).
- 승인하시면 ④ 구현 단계로 넘어갑니다.
```

**이 게이트를 넘어 임의로 구현(코드 작성)으로 진행하지 않는다.** (OS.md 원칙 1 — 가장 중요한 안전장치)
사용자가 명시적으로 "승인", "구현 시작"이라고 말하기 전에는 ④로 가지 않는다.

> 게이트에서 "빠진 재활용 있나요?"를 꼭 묻는 이유: 1차 스캔+병렬 조사도 완벽하진 않다.
> 코드를 잘 아는 사람이 누락을 메워주면, 정직한 1차 초안이 정확한 설계로 완성된다.

---

## 수용 기준 (이 스킬이 제대로 동작하는지 검증)

- [ ] PRD가 없으면 안내하고 멈춘다(Plan을 PRD 없이 시작하지 않는다).
- [ ] 1단계에서 메인이 PRD 기반으로 `Grep`/`Glob` 1차 스캔을 하고 찾음/불확실/못찾음을 나눈다.
- [ ] 갭이 있으면 2단계에서 항목별 `Explore` 서브에이전트를 **병렬로** 띄운다.
- [ ] 갭이 0개면 2단계를 건너뛴다.
- [ ] 3단계에서 **확정된 재활용 지도**를 Plan 서브에이전트에 넘겨 설계하게 한다.
- [ ] `docs/plan/<slug>.md` 가 생성된다.
- [ ] 검토 게이트에서 멈추고, "빠진 재활용 있나요?"를 묻고, 승인 없이 구현으로 안 넘어간다.

---

## 안전 규칙

- **게이트를 넘지 않는다.** 사람의 승인 전에는 절대 ④ 구현으로 가지 않는다.
- **병렬 조사는 갭이 있을 때만.** 비용(토큰·시간)을 의식해 무조건 띄우지 않는다.
- **모든 산출물은 저장소 안에** 둔다(`docs/plan/`). (CLAUDE.md 규칙 1)
- 서브에이전트가 빈 결과/오류를 돌려주면 그대로 알리고 멈춘다. 추측으로 채우지 않는다.
- 이 스킬은 **읽기·검색·문서 작성**만 한다. 코드 파일을 수정하지 않는다(그건 ④ 구현의 몫).
