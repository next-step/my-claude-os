# /os-status — OS 진행현황 대시보드

`/os` 오케스트레이터가 단계 전이마다 갱신하는 **`.claude/os/state.md`** 를 읽어,
지금 어느 단계에 있고 어떤 게이트·DoD가 통과됐는지 한눈에 보여준다.
(에이전트 위임 없는 **얇은 스킬** — 상태 파일만 읽어 정리한다.)

## 절차

### 1. 상태 파일 확인
```bash
cat .claude/os/state.md 2>/dev/null || echo "__NO_STATE__"
```
- `__NO_STATE__` 이거나 파일이 없으면: **"진행 중인 OS 작업이 없습니다. `/os <요구사항>` 으로 새 작업을 시작하세요."** 라고 안내하고 종료한다.

### 2. 핵심 필드 파싱
state.md에서 다음을 읽는다:
- `stage:` (1 | 2 | 3 | 4 | done) 와 `started_at` / `updated_at`
- `## 요구사항`의 한 줄 요약
- `## 게이트 통과 기록`의 체크박스 3종(결정 게이트 ①, 리뷰 게이트, 수용 게이트 ②)
- `## DoD 체크리스트`의 단계별 `[x]`/`[ ]` 개수
- `## 산출물 경로`

### 3. 대시보드로 정리
사용자에게 한눈에 들어오게 보고한다:
- **현재 단계**를 파이프라인 막대로 표시 — 예: `[1]✅ → [2]✅ → [3]🔵 → [4]⬜ → done⬜` (완료 ✅ / 진행중 🔵 / 대기 ⬜)
- **요구사항 한 줄 요약**과 시작·갱신 시각
- **게이트 현황**: 결정①·리뷰·수용② 각 통과 여부
- **DoD 진행률**: 단계별 `통과/전체` (예: 3단계 2/2)
- **다음 할 일** 한 줄: stage 기준으로 "다음은 `/os`로 N단계 이어가기" 또는 done이면 "완료 — `/commit` 안내"
- **산출물 경로**가 채워져 있으면 함께 보여준다.

## 참고
- 이 스킬은 **읽기 전용**이다. state.md를 수정하지 않는다(그건 `/os`만 한다).
- 컨텍스트가 리셋돼도 이 스킬로 "지금 어디까지 했는지"를 즉시 복원할 수 있다 — OS의 "상태는 항상 파일에" 원칙과 직결.
