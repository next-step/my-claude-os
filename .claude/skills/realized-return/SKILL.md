# realized-return (온디맨드 계산 스킬)

stock-os의 **매도 실현손익을 실질(세후·수수료·환율) 기준으로 계산**하는 스킬.
커널 규칙 "실수익 기준"(모든 수익률은 세금·수수료·환율 차감 후로 말한다)을 도구로 구현한다.
웹조회 없이 로컬 파일 읽기 + 계산뿐이라 메인 컨텍스트에서 실행한다(subagent 아님).

## 입력

1. **Read** `05_reference/tax-and-fees.md` — 세율·수수료·**매도 손익 공식**의 SSOT.
   (공식·세율을 재구현하지 말고 이 파일에서 읽어 그대로 적용.)
2. **매매 상세** — 사용자에게 질문하거나 `holdings.md`에서 참조:
   - 시장(코스피/코스닥/US), 티커, 수량
   - 매수가·(미국) 매수 시 환율
   - 매도가·(미국) 매도 시 환율

## 처리

`tax-and-fees.md`의 **매도 손익 판단 공식**을 그대로 적용:

```
실수익 = (매도금액 − 매수금액) − 거래세 − 수수료(매수·매도·환전) − (해당 시) 양도세
```

### 국내 주식
- 매도 시 **증권거래세 0.20%**(코스피=거래세0.05%+농특세0.15%, 코스닥=0.20%).
- 소액주주 상장주식 **양도세 비과세**(대주주 요건 시에만 과세 — 해당하면 사용자 확인).
- 배당은 별도(이 스킬은 매도차익만 다룸).

### 미국 주식 (한국 거주자)
- 양도차익은 **거래 시점 환율**로 원화 환산 → **환차익도 과세 대상**에 포함.
- 양도세는 연 **250만원 공제 후 초과분 22%**(양도세 20%+지방세 2%).
- ⚠️ **이 한 건만으로 양도세를 확정할 수 없다** — 연간 손익통산·250만원 공제는
  그 해 전체 해외주식 실현손익에 걸쳐 적용된다. 따라서 양도세는 **추정치**로 표기하고,
  "확정치는 연말 손익통산 기준"임을 명시한다.

### 수수료 (의존성 — 중요)
`tax-and-fees.md §3`의 **국내/미국 매매 수수료·환전 우대율이 현재 빈칸**이다.
- 값이 있으면 그대로 사용.
- 비어 있으면 (a) 사용자에게 물어 그 값으로 계산하고, (b) `tax-and-fees.md §3`에
  채워 넣을지 제안한다(연 단위 갱신 항목이므로 한 번 채우면 재사용).

## 출력

**표면 vs 실질을 나란히** 보여 괴리를 드러낸다(공식 문서의 취지: "표면 +5%여도 세금·수수료
빼면 본전 근처일 수 있다").

| 항목 | 금액 |
|------|------|
| 표면 손익 (매도−매수) | +___원 (표면 +__%) |
| − 증권거래세 / 양도세(추정) | −___원 |
| − 수수료(매수·매도·환전) | −___원 |
| **실질 실현손익** | **+___원 (실질 +__%)** |

- 미국 종목은 **환차손익**을 별도 줄로 분해해 보여준다(주가차익 vs 환차익).
- 마지막에 `03_journal/trade-journal.md`의 "결과(매도 시 실현손익 %·금액, 실질 기준)"
  칸에 **그대로 붙일 수 있는 한 줄**을 제공한다.
  예: `결과: 실질 +3.2% / +42만원 (표면 +5.1%, 거래세·환전 차감)`

## 쓰기
계산 결과는 사용자에게 present. 매매 기록(holdings·trade-journal) 갱신은 사람/기록
흐름의 몫이며, 이 스킬은 붙여넣을 문구까지만 제공한다.
