# skill-stat 스킬

`.claude/hooks/skill-counter.sh` 훅이 누적해 둔 **스킬별 호출 횟수**(`.claude/skill-stats.json`)를
사람이 보기 좋은 통계로 정리해 보여준다.

## 데이터 출처

- 파일: `.claude/skill-stats.json`
- 구조:
  ```json
  {
    "skills": {
      "<스킬이름>": { "count": <호출횟수>, "last_called": "<UTC 시각>" }
    },
    "total": <전체 호출 합계>
  }
  ```
- 이 파일은 Skill 도구가 호출될 때마다 PreToolUse 훅이 자동으로 갱신한다.

## 통계 만들기 절차

### 1단계: 데이터 읽기 + 정렬 (Bash + jq)

아래 명령으로 호출 횟수 내림차순 정렬된 표 데이터를 뽑는다:

```bash
jq -r '
  (.total // 0) as $total
  | "전체 호출: \($total)회",
    "---",
    (.skills | to_entries
      | sort_by(-.value.count)
      | to_entries[]
      | "\(.key + 1)\t\(.value.key)\t\(.value.value.count)\t\(.value.value.last_called)"
    )
' .claude/skill-stats.json
```

- 파일이 없거나 `total`이 0이면 **"아직 기록된 스킬 호출이 없습니다"** 라고 안내한다.
  (훅이 등록된 뒤 스킬을 한 번도 안 썼거나, 막 설정해서 아직 안 잡혔을 수 있음 →
  이 경우 `/hooks` 메뉴를 한 번 열거나 재시작하면 훅이 활성화된다고 덧붙인다.)

### 2단계: 표로 정리해서 보여주기

읽어온 데이터를 다음 형식의 마크다운 표로 렌더링한다. 비율은 `count / total * 100`으로 계산한다.

| 순위 | 스킬 | 호출 횟수 | 비율 | 마지막 호출 |
|----|------|--------:|----:|-----------|
| 1 | update-config | 5 | 50% | 2026-06-25 11:21 |
| 2 | git-commit | 3 | 30% | 2026-06-25 10:40 |
| 3 | skill-stat | 2 | 20% | 2026-06-25 11:50 |

- 마지막 줄에 **전체 호출 합계**와 **기록된 스킬 종류 수**를 요약한다.
- 가장 많이 쓴 스킬(1위)을 한 줄로 짚어주면 좋다. 예: "🏆 가장 많이 쓴 스킬: update-config (5회)".
- 시각은 UTC로 저장돼 있으니, 그대로 보여주되 필요하면 "UTC" 표기를 덧붙인다.

## 주의

- 통계 파일을 **수정/삭제하지 않는다**. 읽기 전용으로만 다룬다.
  (사용자가 "통계 초기화"를 명시적으로 요청하면 그때만
  `echo '{"skills":{},"total":0}' > .claude/skill-stats.json` 으로 리셋한다.)
- 이 skill-stat 스킬 자신도 호출되면 카운트에 잡힌다 — 통계에 보이는 게 정상이다.
