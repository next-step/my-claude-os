이 프로젝트의 `.claude/data/skill-stats.json`을 읽어 **스킬별 호출 통계**를 사용자에게 보여줘.

추가 맥락: $ARGUMENTS

## 데이터 출처

- 파일 경로: `.claude/data/skill-stats.json` (프로젝트 루트 기준)
- 이 파일은 `PostToolUse(matcher: Skill)` hook(`.claude/hooks/record-skill.sh`)이 스킬 호출마다 자동 갱신한다.
- **사용자 직접 호출(`/skill-name`)과 Claude의 자동 호출 모두** 같은 tool 경로를 거치므로 한 곳에 집계된다.

## 진행 순서

### 1. 통계 파일 읽기
- Read tool로 `.claude/data/skill-stats.json`을 읽는다.
- 파일이 없거나 `skills`가 비어있으면 다음과 같이 안내하고 종료:
  > 아직 기록된 스킬 호출이 없어요. 스킬을 한 번 사용하면 자동으로 집계가 시작됩니다.

### 2. 인자 파싱 ($ARGUMENTS)
- `top N`: 호출 수 기준 상위 N개만 출력 (예: `top 3`)
- 인자가 없으면 전체 출력
- 알 수 없는 인자는 무시하고 전체 출력하되, 한 줄로 알려준다

### 3. 출력 포맷
다음 항목을 **한국어**로 정리해 사용자에게 보여준다.

**요약 헤더**
- 총 호출 수 (`total_invocations`)
- 마지막 갱신 시각 (`updated_at`) — KST가 아닌 UTC임을 표기

**스킬별 표** (count 내림차순)

| 스킬 이름 | 호출 수 | 비율 | 첫 호출 (UTC) | 마지막 호출 (UTC) |
|-----------|---------|------|---------------|--------------------|
| git-commit | 5 | 62.5% | 2026-06-24 10:00 | 2026-06-25 20:00 |

- 비율 = `count / total_invocations * 100`, 소수점 1자리
- 시각은 ISO 문자열에서 `T`를 공백으로 바꾸고 `Z`는 제거해서 간결하게

### 4. 자기 자신을 카운트한다는 점 (학습용 안내)
이 스킬도 hook을 거치므로 **실행 직후 자기 자신의 count가 1 증가**한 상태를 보게 된다.
출력 맨 아래에 작게 한 줄로 그 사실을 알려주면 사용자가 데이터를 혼동하지 않는다:

> 참고: `skill-stat` 자체도 호출 시 카운트되므로, 위 표에는 이번 실행이 포함됩니다.

## 하지 않는 것
- `.claude/data/skill-stats.json` 파일 자체를 수정하거나 초기화하지 않는다 (별도 요청 시에만).
- 다른 디렉토리에 통계 파일을 새로 만들지 않는다.
