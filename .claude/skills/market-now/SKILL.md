# 🌐 시장 브리핑 — market-now

지금 시장에서 무슨 일이 벌어지는지 **출처 기반으로** 알려주는 스킬.
조사는 직접 하지 않고 **`market-researcher` 서브 에이전트에 맡긴다** (study-loop의 브리핑과 **같은 엔진을 공유**한다).

> ⚠️ AI는 분석·설명 도우미일 뿐, 투자 권유가 아니다. 판단과 실제 매매는 사용자가 한다.

## 할 일

1. **조사 테마를 정한다.**
   - 사용자가 특정 주제를 물었으면(예: "코스피 왜 폭락?") → 그 테마 1개.
   - 막연히 "오늘 시장 어때?"면 → 핵심 테마 몇 개로 쪼갠다. 기본 세트: **코스피/코스닥 동향, 미국 증시(S&P500·나스닥), 미국 기준금리, 원/달러 환율.**

2. **테마별로 `market-researcher`를 병렬 호출한다.**
   - 조사 대상으로 **시장 테마 + 오늘 날짜 + 조사 기간**을 전달한다.
   - **조사 기간은 질문의 의도에서 읽는다:**
     - "오늘 무슨 일", "오늘 왜 폭락" → **오늘 하루(1일)**
     - "이번 주", "요즘 며칠" → **약 1주**
     - "요즘 시장 분위기", 막연한 질문 → **최근 2주**
   - 테마가 여러 개면 한 메시지에서 여러 Agent 호출을 동시에 띄운다.

3. **모아서 "오늘의 시장 브리핑"으로 정리한다.**
   - 각 에이전트가 돌려준 3~4줄 요약을 테마별로 묶어 보여준다.
   - 각 줄에 붙은 `(출처, 날짜)`를 유지한다. 출처 없는 주장은 싣지 않는다.
   - 사실·배경 전달이지 투자 권유가 아님을 유지한다.

4. (선택) 사용자가 원하면, 이 시장 이슈가 **내 포폴과 어떻게 엮이는지** 한 줄 덧붙인다 — 단, 권유가 아니라 "이 종목이 이 테마에 노출돼 있다" 수준의 사실 연결까지만. 비중 조정 진단은 `/rebalance`로 넘긴다.

## 다른 입구와의 차이
- **study-loop**: 내 *보유 종목* 중심 매일 공부 루프 (브리핑→Q&A→정리·퀴즈).
- **rebalance**: 목표비중 대비 비중 조정 진단 (가끔).
- **fire-progress**: 파이어 목표 대비 진척도 (가끔).
- **market-now**: 보유와 무관하게 *시장 전반*을 수시로 파악. study-loop와 같은 `market-researcher`를 쓴다.
