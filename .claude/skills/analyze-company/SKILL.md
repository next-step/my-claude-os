# analyze-company — 회사 분석 스킬

회사 이름 하나를 받아, 세 가지 데이터를 모아 분석 리포트를 만든다.
**수집은 전문 서브 에이전트들이 병렬로**, **종합은 메인(너)이** 한다.

## 분석에 쓰는 데이터 (이 3가지로 고정)

1. 최근 1년 주가 흐름 → `stock-trend-researcher`
2. 회사 관련 뉴스 → `company-news-researcher`
3. 대한민국 경제 뉴스(거시 환경) → `market-news-researcher`

> 데이터 소스는 모두 **웹검색(WebSearch/WebFetch)** 기반이다. 별도 API 키는 쓰지 않는다.

## 실행 절차

### 1단계 — 대상 확정
- 분석할 회사 이름을 확인한다. 인자로 안 받았으면 사용자에게 묻는다.
- 동명이의/모호하면(예: 상장사 여부, 한국/해외) 한 번 확인한다.

### 2단계 — 병렬 수집 (서브 에이전트 동시 호출)
**한 번의 응답에서 Agent 도구를 3번 호출**해 세 서브 에이전트를 **동시에** 띄운다.
(순차로 돌리면 느리다. 서로 의존이 없으니 반드시 병렬로.)

- `stock-trend-researcher` ← "회사명: OO. 최근 1년 주가 흐름 + 주요 지지·저항선 + (있으면) 컨센서스 목표주가를 조사해줘."
- `company-news-researcher` ← "회사명: OO. 이 회사 관련 최근 뉴스를 조사해줘."
- `market-news-researcher` ← "대한민국의 최근 경제·시장 환경을 조사해줘." (회사명은 참고만)

각 에이전트는 정해진 출력 형식으로 결과를 돌려준다.

### 2.5단계 — 현재가·시세 확정 (결정론적 소스) ⚠️ 중요
**현재가·52주 고저 같은 정밀 시세는 웹검색 결과를 신뢰하지 않는다.** (웹검색은 출처마다 값이 달라 엉뚱한 값을 채택하는 사고가 있었다 — 기아 검증에서 실제 24% 오차.) 반드시 스크립트로 확정한다:
```
python3 .claude/skills/analyze-company/scripts/quote.py --name "OO"   # 또는 --code 000270
```
- 반환: `현재가 / 52주 고저 / 거래량`. **이 현재가가 진입/손절의 앵커**다.
- 서브에이전트가 보고한 현재가가 이 값과 크게 다르면, **스크립트 값을 채택**하고 차이를 유의에 적는다.
- 스크립트가 실패하면(비상장·해외 등) 그때만 서브에이전트 값으로 폴백하되 "확정 못 함"을 명시.

### 3단계 — 종합 분석
세 결과를 받아 **하나의 리포트로 엮는다.** 단순 나열이 아니라 **연결**한다. 예:
- 주가가 빠졌는데(주가) 그 원인이 회사 악재(뉴스)인지, 시장 전체 하락(거시)인지 구분
- 호재 뉴스가 이미 주가에 반영됐는지(주가 흐름과 대조)
- 거시 환경이 이 회사에 우호적인지 역풍인지

### 3.5단계 — 매매 가격대 산출 (진입 / 목표 / 손절)
**앵커는 2.5단계 quote.py의 현재가**다(웹검색 현재가가 아니다). 가격대 판단은 메인(너)이 한다. 재료가 없으면 "산출 불가"로 적는다(지어내지 않는다).
- **진입가**: quote.py **현재가** 기준. 현재가가 지지선 부근이면 현재가, 한참 위면 가까운 지지선까지 기다리는 가격. (진입가가 현재가보다 높으면 안 된다 — 그건 시세보다 비싸게 사라는 뜻이니 재검토.)
- **손절가**: 진입가 **아래** −5~8%(또는 52주 저점·지지선 하향 이탈 수준).
- **목표(기대치)**: 컨센서스 목표주가(웹검색)가 있으면 그것을 기준, 없으면 52주 고점·저항선으로 **보수적으로**. 현재가 대비 기대수익률(%)을 함께.
- 세 값의 **근거를 한 줄씩** 남긴다(현재가/52주 고저/컨센서스 중 무엇에서 나왔는지). 거시·뉴스가 역풍이면 보수적으로 조정.

### 4단계 — 리포트 출력
아래 형식으로 사용자에게 보여준다.

```
# 📊 OO 분석 리포트  (작성일: YYYY-MM-DD)

## 한 줄 요약
- (주가·뉴스·거시를 종합한 핵심 한두 문장)

## 1. 주가 흐름 (최근 1년)
- (stock-trend-researcher 결과 요약)

## 2. 회사 이슈 (최근 뉴스)
- (company-news-researcher 결과 요약)

## 3. 시장 환경 (한국 경제)
- (market-news-researcher 결과 요약)

## 종합 해석
- 세 축을 연결한 분석 (왜 이렇게 움직였나 / 지금 어떤 상황인가)

## 매매 가격대 (참고용)
- 진입가: (값) — 근거:
- 목표(기대치): (값, +%) — 근거:
- 손절가: (값, −%) — 근거:
- (재료 부족 시: "산출 불가 — 지지/저항·목표주가 데이터 없음")

## ⚠️ 유의
- 이 리포트는 웹검색 기반 참고 자료이며 투자 권유가 아니다. 매매·주문은 사람이 결정한다.
- 가격대는 관찰된 지지/저항·컨센서스에서 도출한 **참고 수준**이며 확정 예측이 아니다.
- 핵심 수치/주장에는 출처가 달려 있는지 확인할 것.
```

## 원칙
- **출처 없는 수치는 리포트에 싣지 않는다.** 서브 에이전트가 출처를 안 주면 "확인 불가"로 표기.
- 데이터를 못 구한 축이 있으면 숨기지 말고 "데이터 부족"이라고 적는다.
- 이 스킬은 **분석·정보 제공까지만** 한다. (OS 가드레일: 실제 매매 주문은 사람 몫)
- 향후 펀더멘털·공시(DART)·수급 등 데이터 축을 추가할 때는 **서브 에이전트를 하나 더 붙이는 방식**으로 확장한다. (이 구조의 핵심 장점)
