# analyze-company — 회사 분석 스킬

회사 이름 하나를 받아, 여러 축의 데이터를 모아 분석 리포트와 매매 가격대를 만든다.
**정성 정보는 서브 에이전트가 병렬 웹검색으로**, **정밀 수치는 결정론적 스크립트로**, **종합은 메인(너)이** 한다.

## 분석에 쓰는 데이터

**A. 정성 — 서브 에이전트 병렬 웹검색** (내러티브·맥락. 출처마다 값이 흔들려도 '해석'에 쓴다)
1. 최근 1년 주가 흐름 → `stock-trend-researcher`
2. 회사 관련 뉴스 → `company-news-researcher`
3. 대한민국 거시·시장 환경 → `kr-macro-researcher` (retrospect 와 공유하는 수집 전용 에이전트)

**B. 정밀 정량 — 결정론적 스크립트** (네이버 직접 파싱. 진입/목표/손절의 *근거 숫자*는 여기서만 취한다)
4. 현재가·52주 고저 → `scripts/quote.py`
5. 지지/저항·변동성(일별 OHLCV) → `scripts/ohlcv.py`
6. 펀더멘털(매출·영업이익 추세·ROE·부채비율·PER/PBR·배당) + 컨센서스 목표주가 → `scripts/fundamentals.py`

> **왜 둘로 나누나:** 웹검색은 출처마다 숫자가 달라 정밀값엔 위험하다(기아 검증에서 현재가 24% 오차 사고). 그래서 **정성=웹검색, 정밀 수치=스크립트**로 못 박는다. 스크립트는 네이버만 쓰고 API 키가 필요 없다.

## 실행 절차

### 1단계 — 대상 확정
- 분석할 회사 이름을 확인한다. 인자로 안 받았으면 사용자에게 묻는다.
- 동명이의/모호하면(예: 상장사 여부, 한국/해외) 한 번 확인한다.

### 2단계 — 병렬 수집 (서브 에이전트 동시 호출)
**한 번의 응답에서 Agent 도구를 3번 호출**해 세 서브 에이전트를 **동시에** 띄운다.
(순차로 돌리면 느리다. 서로 의존이 없으니 반드시 병렬로.)

- `stock-trend-researcher` ← "회사명: OO. 최근 1년 주가 흐름 + 주요 지지·저항선 + (있으면) 컨센서스 목표주가를 조사해줘."
- `company-news-researcher` ← "회사명: OO. 이 회사 관련 최근 뉴스를 조사해줘."
- `kr-macro-researcher` ← "기간: 최근 2~4주(현재 환경 모드). 대한민국의 거시·시장 환경을 조사해줘." (회사명은 참고만 — 시장 전반에 초점)

각 에이전트는 정해진 출력 형식으로 결과를 돌려준다.

### 2.5단계 — 정밀 수치 확정 (결정론적 소스) ⚠️ 중요
진입/목표/손절의 근거가 될 숫자는 **웹검색이 아니라 스크립트로 확정**한다. 세 스크립트를 돌린다(코드를 알면 `--code`가 가장 안정적, 모르면 quote.py가 이름→코드 변환):
```
python3 .claude/skills/analyze-company/scripts/quote.py        --name "OO"        # 현재가·52주 고저 → code도 얻음
python3 .claude/skills/analyze-company/scripts/ohlcv.py        --code 000270      # 지지/저항·변동성
python3 .claude/skills/analyze-company/scripts/fundamentals.py --code 000270      # 펀더멘털 + 컨센서스 목표주가
```
- **quote.py**: `현재가 / 52주 고저 / 거래량`. **이 현재가가 진입/손절의 앵커**다. 서브에이전트 보고가 이와 크게 다르면 스크립트 값을 채택하고 차이를 유의에 적는다.
- **ohlcv.py**: `support[] / resistance[]`(가까운 순, `touches`=터치 횟수로 강도) + `realized_vol_pct`(연율 변동성)·`atr_pct`. 진입은 지지, 손절은 지지 하단, 변동성은 손절 폭 조절에 쓴다.
- **fundamentals.py**: `financials`(고/저평가 판단)와 `consensus.target`(목표가 근거). 컨센서스는 커버리지가 없으면 `null` — 그땐 저항선으로 목표를 잡는다.
- 스크립트가 실패하면(비상장·해외·커버리지 없음) 그 축만 "확정 못 함"으로 두고 서브에이전트 값으로 폴백하되 근거 약함을 명시.

### 3단계 — 종합 분석
정성(주가·뉴스·거시) + 정밀(펀더멘털·지지저항·컨센서스)을 받아 **하나의 리포트로 엮는다.** 단순 나열이 아니라 **연결**한다. 예:
- 주가가 빠졌는데(주가) 그 원인이 회사 악재(뉴스)인지, 시장 전체 하락(거시)인지 구분
- 호재 뉴스가 이미 주가에 반영됐는지(주가 흐름과 대조)
- **펀더멘털 대조**: 주가는 시장의 평가, 재무는 그럴 만한가. PER/PBR·ROE·영업이익 추세로 고/저평가를 본다(예: PER 낮고 영업이익 우상향이면 저평가 신호).
- 거시 환경이 이 회사에 우호적인지 역풍인지

### 3.5단계 — 매매 가격대 산출 (진입 / 목표 / 손절)
**근거 숫자는 2.5단계 스크립트 값**을 쓴다(웹검색 수치가 아니다). 앵커는 quote.py 현재가. 판단은 메인(너)이 한다. 재료가 없으면 "산출 불가"로 적는다(지어내지 않는다).
- **진입가**: quote.py **현재가** 기준 + ohlcv.py **지지선**. 현재가가 지지선 부근이면 현재가, 한참 위면 가까운 지지선까지 기다리는 가격. (진입가가 현재가보다 높으면 안 된다 — 시세보다 비싸게 사라는 뜻이니 재검토.)
- **손절가**: 진입가 **아래** −5~8%. ohlcv.py의 **가까운 지지선 하단**이나 **변동성(atr_pct)** 으로 폭을 정한다(변동성 크면 손절 폭도 넓게). 지지선 하향 이탈 수준이 1순위 근거.
- **목표(기대치)**: fundamentals.py **컨센서스 목표주가**가 있으면 그것을 기준, 없으면 ohlcv.py **저항선**(가까운 순)으로 **보수적으로**. 현재가 대비 기대수익률(%)을 함께.
- 세 값의 **근거를 한 줄씩** 남긴다(지지/저항/변동성/컨센서스 중 무엇에서 나왔는지). 거시·뉴스가 역풍이면 보수적으로 조정.

### 4단계 — 리포트 출력
아래 형식으로 사용자에게 보여준다.

```
# 📊 OO 분석 리포트  (작성일: YYYY-MM-DD)

## 한 줄 요약
- (주가·뉴스·거시를 종합한 핵심 한두 문장)

## 1. 주가 흐름 (최근 1년)
- (stock-trend-researcher 결과 요약)

## 2. 회사 이슈 (최근 뉴스)
- (company-news-researcher 결과 요약)

## 3. 시장 환경 (한국 거시)
- (kr-macro-researcher 결과 요약)

## 4. 펀더멘털 (fundamentals.py)
- 매출/영업이익 추세, ROE·부채비율, PER/PBR·배당수익률 → 고/저평가 한 줄 해석
- 컨센서스: 투자의견 / 목표주가 (없으면 "커버리지 없음")

## 종합 해석
- 정성·정밀을 연결한 분석 (왜 이렇게 움직였나 / 지금 어떤 상황인가 / 싼가 비싼가)

## 매매 가격대 (참고용)
- 현재가(앵커): (quote.py 값)
- 진입가: (값) — 근거: (지지선 등)
- 목표(기대치): (값, +%) — 근거: (컨센서스/저항선)
- 손절가: (값, −%) — 근거: (지지선 하단/변동성)
- (재료 부족 시: "산출 불가 — 지지/저항·목표주가 데이터 없음")

## ⚠️ 유의
- 이 리포트는 웹검색 기반 참고 자료이며 투자 권유가 아니다. 매매·주문은 사람이 결정한다.
- 가격대는 관찰된 지지/저항·컨센서스에서 도출한 **참고 수준**이며 확정 예측이 아니다.
- 핵심 수치/주장에는 출처가 달려 있는지 확인할 것.
```

### 5단계 — 분석 기록 저장 (회고용, append-only) ⚠️ 항상 한다
리포트를 보여준 뒤, **이 분석 1건을 회고용으로 저장한다.** 단독 분석이든 `recommend-stocks`가 부른 것이든 **항상** 저장한다.
*왜:* 진입/목표/손절·근거·status의 **단일 진실원천**이 이 기록이다. 주 1회 회고 스킬이 이 기록을 현재가와 대조해 status(`hit_target|stopped|watching`)를 갱신한다. (추천 기록 `save_run.py`는 이 기록을 **참조만** 하고 숫자를 복제하지 않는다.)

조립한 JSON을 표준입력으로 넘긴다(스키마는 `scripts/save_analysis.py` 상단 docstring):
```
echo '<조립한 JSON>' | python3 .claude/skills/analyze-company/scripts/save_analysis.py
```
- 저장 위치: `data/analyses/YYYY-MM-DD-<회사>.md` (frontmatter 정량 + 본문 근거). 같은 날 같은 회사 재분석은 `-2`,`-3`…으로 **과거를 덮지 않는다.**
- 출력으로 `{"ref": "data/analyses/...", ...}`를 돌려준다. **`recommend-stocks`가 부른 경우 이 `ref`를 메인에 반환**해, 추천 기록(`save_run.py`)의 `picks[].analysis`에 넣게 한다.
- `axes`에는 정성 3축 요약뿐 아니라 **펀더멘털·지지저항·컨센서스 요약도 함께** 담는다(회고 때 "그때 어떤 근거로 봤나"를 복기). `price`/`entry`/`target`/`stop`은 2.5·3.5단계 스크립트 값으로 채운다.
- 재료가 없어 가격대가 "산출 불가"여도 분석 자체는 기록한다(무엇을 못 구했는지가 회고 자료).

## 원칙
- **정성 정보는 웹검색(서브에이전트), 정밀 수치는 스크립트.** 진입/목표/손절의 근거 숫자는 항상 스크립트(quote/ohlcv/fundamentals)에서 취한다.
- **출처 없는 수치는 리포트에 싣지 않는다.** 서브 에이전트가 출처를 안 주면 "확인 불가"로 표기.
- 데이터를 못 구한 축이 있으면 숨기지 말고 "데이터 부족"이라고 적는다.
- 이 스킬은 **분석·정보 제공까지만** 한다. (OS 가드레일: 실제 매매 주문은 사람 몫)
- 축 확장 방식: **정성 축은 서브 에이전트 추가**(웹검색), **정밀 정량 축은 스크립트 추가**(네이버/DART 등 결정론적 소스). 공시(DART)·수급이 다음 후보.
