# /list 스킬 — 할일 전체 조회 오케스트레이터

지금까지 쌓인 할일을 한눈에 본다.
capture(생성) → plan(구체화) → done(완료) 흐름 어디에 무엇이 걸려 있는지 상태별로 보여준다.

## 사용법

```
/list            ← 전체 할일을 상태별로 묶어서 표시
/list draft      ← draft 항목만 (planned / done 동일)
/list 장보기      ← 제목 키워드로 부분 일치 검색 (상태 무관)
```

---

## 실행 절차

> **결정론적 조회 = LLM 우회, 공유 포매터 한 번 호출**
> `/list`는 입력이 정해지면 출력도 정해지는 조회다. AI 판단이 전혀 필요 없다.
> 조회·필터·상태별 그룹핑·번호·일정 요약·경과일 같은 **표시 규칙은 전부
> `_shared/list-view.sh` 한 곳**에 들어 있다. 스킬은 이 스크립트를 그대로 호출하고
> 출력만 사용자에게 전달한다.
>
> 이렇게 단일 소스로 둔 이유:
> 1. 텔레그램 리스너도 폰에서 온 `/list`를 **같은 스크립트**로 처리한다.
>    풀 `claude -p`(콜드 스타트 ~15초) 대신 직접 호출(~1초)이라 폰 응답이 빠르다.
> 2. 표시 규칙이 스킬·리스너 두 곳에 중복되지 않아, 고칠 때 한 곳만 고치면 된다.

**한 단계로 끝난다.** 사용자 인자(`$ARGUMENTS`)를 그대로 넘겨 실행하고, 출력을 그대로 보여준다.

```bash
.claude/skills/_shared/list-view.sh {ARGUMENTS}
```

- 인자 없음 → 전체를 상태별로 묶어 표시
- `draft` / `planned` / `done` → 해당 상태만
- 그 외 문자열 → 제목 키워드 부분 일치 검색

스크립트가 인자 파싱·빈 결과 안내·헤더(`📋`/`🔍`)·그룹핑까지 모두 처리하므로,
스킬은 **출력을 가공하지 말고 그대로 전달**한다.

### 출력 예시

```
📋 할일 전체 (5개)

📝 구체화 대기 (draft) · 2개
  1. 장보기 (생활) — 3일 전 캡처
  2. 스프링 강의 듣기 (학습) — 1일 전 캡처

📅 예정 (planned) · 2개
  3. 운동하기 (건강) — 매일 저녁
  4. 병원 예약 (건강) — 2026-07-10 오전

✅ 완료 (done) · 1개
  5. 보고서 제출 (업무)
```

> **표시 규칙을 바꾸려면** `_shared/list-view.sh`만 수정한다. (이 문서가 아니라 스크립트가 단일 출처)
