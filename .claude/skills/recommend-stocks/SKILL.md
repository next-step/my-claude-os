# recommend-stocks — 종목 추천 스킬 (안정성 우선)

코스피 ~1000개에서 **싼 필터로 먼저 줄이고**, 살아남은 소수만 **무겁게 정밀 분석**하는 깔때기.
설계 배경·결정 이유는 `docs/OS.md`의 "종목 추천 스킬" 절을 따른다.

> ⚠️ 이 스킬은 **정보·조언 제공까지만** 한다. 실제 매매 주문은 사람 몫(OS 가드레일).
> 투자 권유가 아니며, 모든 수치엔 출처/근거가 있어야 한다.

## 파이프라인

```
코스피 전체(~1000)
  ↓ 1차 필터  screen_kospi.py (네이버 시총 표: 시총·거래대금·PER·ROE)   — 싼 하드컷
 ~50
  ↓ 2차 필터  개별 재무(부채비율·흑자추세) → 안정성 점수로 순위
 ~10
  ↓ 정밀 분석  analyze-company 를 10개 각각에 실행
최종 추천 = 종목별 [진입가 · 목표(기대치) · 손절가 · 안정성 점수],  안정성 우선 정렬
```

핵심 원칙: **상승 기대치가 높다고 위로 올리지 않는다.** 정렬 기준은 안정성 점수다.
("A: 상승15%·불안정"보다 "B: 상승8%·안정"을 더 추천.)

## 필터 기준

### 1차 하드컷 (통과/탈락) — `screen_kospi.py`가 적용

| 기준 | 임계값 | 왜 |
|------|--------|----|
| 시가총액 | ≥ 5,000억 | 소형주·품절주 제외. 규모는 안정성·유동성의 하한 |
| 일 거래대금(현재가×거래량) | ≥ 10억 | 못 빠져나오는 종목 제외(유동성) |
| PER | 0 < PER ≤ 25 | 적자(PER 없음/음수) 제외 + 과열 제외 |
| ROE | ≥ 5% | 최소한의 자본 수익성 |
| 우선주·스팩 | 제외 | 본주 분석에 집중 |

> 임계값은 **튜닝 대상**이다. 예: PER ≤ 25면 삼성전자(과열 구간)도 탈락한다 — 안정성·밸류에이션을 우선한 의도된 결과지만, 운용하며 조정한다. 변경 시 `screen_kospi.py`의 `DEFAULTS`와 이 표를 함께 고친다.

### 2차 안정성 스코어링 (50→10, 0~100점) — `score_stocks.py`가 적용

좁혀진 ~50개만 개별 종목 재무(네이버 "기업실적분석" 표)를 보고 점수를 매겨 상위 ~10개를 고른다.

- **하드컷:** 부채비율 ≤ 150% **그리고** 가장 최근 연도 영업이익 흑자. (둘 중 하나라도 어기면 탈락)

| 항목 | 배점 | 데이터 |
|------|------|--------|
| 재무 건전성(부채비율 50%↓=만점, 150%=0) | 30 | 기업실적분석 부채비율 |
| 이익 안정성(최근 3년 흑자비율 15 + 영업이익 변동성 낮을수록 10) | 25 | 최근 3년 영업이익 |
| 주가 변동성(연율 20%↓=만점, 70%=0) | 15 | `ohlcv.py` 일별 시세 |
| 배당(시가배당률 0%=0, 4%↑=만점) | 10 | 기업실적분석 시가배당률 |
| 수익성(ROE 5%=0, 20%↑=만점) | 10 | ROE |
| 밸류에이션(PER 10↓=만점, 25=0) | 10 | 1차 PER |

> 안정성 계열(재무건전성+이익안정성+주가변동성+배당)=**80**, 성장/밸류(ROE+PER)=20. **안정성 우선** 의도.
> **상승 기대치는 점수에 넣지 않는다.** 기대치는 3단계 산출물일 뿐, 정렬 기준이 아니다.
> 주가 변동성은 `analyze-company/scripts/ohlcv.py`를 재사용한다(이 스킬은 이미 analyze-company 스킬에 의존). 모듈/네트워크 실패 시 변동성 가점만 **중립값(절반)**으로 두고 점수는 계속 낸다.
> 배점·임계값은 튜닝 대상 — 바꾸면 `score_stocks.py`의 `DEFAULTS`/`score()`와 이 표를 함께 고친다.

## 실행 절차

### 1단계 — 1차 스크리닝 (구현됨)
```
python3 .claude/skills/recommend-stocks/scripts/screen_kospi.py --json
```
- 네이버 코스피 시총 페이지를 순회하며 1차 하드컷을 적용해 후보(~50)를 JSON으로 돌려준다.
- 표준 라이브러리만 쓰므로 별도 설치 불필요. 네트워크가 막히면 사용자에게 알리고 중단한다.

### 2단계 — 2차 안정성 스코어링 (구현됨)
```
python3 .claude/skills/recommend-stocks/scripts/screen_kospi.py --json \
  | python3 .claude/skills/recommend-stocks/scripts/score_stocks.py --top-n 10 --json
```
- 1차 결과(JSON)를 받아 종목별 재무로 하드컷+점수화하고, **안정성 점수 상위 ~10개**를 JSON으로 돌려준다.
- 1차→2차를 파이프(`|`)로 잇는다. 1차 결과를 따로 저장해 두면 회고(탈락 표본)에 쓰기 좋다.
- **변동성 가점**은 종목마다 일별 시세 여러 페이지를 더 읽어 **느리다**(50종목×수페이지). 빠르게 돌리려면 `--no-vol`(변동성 중립 처리), 정밀도를 높이려면 `--vol-days 250`. 기본 `--vol-days 60`.

### 3단계 — 정밀 분석 (analyze-company 재사용)
- 2단계 통과 ~10개 **각각에 `analyze-company` 스킬을 실행**해 주가·뉴스·거시 종합 리포트와 매매 가격대를 얻는다.
- **현재가 앵커는 2차 결과의 `price`(네이버에서 검증된 값)** 를 쓴다. analyze-company도 자체적으로 `quote.py`로 현재가를 확정하므로, 진입/손절이 실제 시세에 붙는다. (웹검색 현재가는 신뢰하지 않는다 — 기아 검증에서 24% 오차 사례.)
- 산출 규칙은 analyze-company의 3.5단계를 따른다: 진입=현재가 기준 지지, 손절=−5~8%, 목표=컨센서스/52주 고점. **진입가가 현재가보다 높게 나오면 잘못된 것**이니 재검토.
- **각 analyze-company 실행은 분석 기록을 저장하고 `ref`(예: `data/analyses/2026-06-30-기아.md`)를 돌려준다.** 이 `ref`를 종목별로 모아 둔다 — 5단계에서 추천 기록이 진입/목표/손절을 **복제하지 않고 이 ref로 참조**한다(진실원천은 분석 기록 한 곳).
- *보강 예정:* 정밀 지지/저항은 아직 약함(52주 고저 의존). OHLCV 기반 지지저항 계산을 추가하면 더 단단해진다(`docs/OS.md`).

### 4단계 — 정렬 & 출력
- **안정성 점수 내림차순**으로 정렬(상승률 아님). 동점이면 기대수익률로 보조 정렬.
- 아래 형식으로 출력.

```
# 📈 종목 추천 (작성일: YYYY-MM-DD) — 안정성 우선

| 순위 | 종목 | 안정성 | 진입가 | 목표(기대) | 손절가 | 한 줄 근거 |
|------|------|--------|--------|-----------|--------|-----------|
| 1 | ... | 84 | ... | ... (+8%) | ... | ... |

## 필터 요약
- 1차 통과 N개 → 2차 M개 → 정밀분석 상위 K개
- 적용 기준: (시총/거래대금/PER/ROE 임계값)

## ⚠️ 유의
- 웹 데이터 기반 참고 자료이며 투자 권유가 아니다. 실제 매매·주문은 사람이 결정한다.
```

### 5단계 — 회고용 기록 (구현됨, append-only)
- 이번 실행의 **선정 결정**을 한 JSON으로 조립해 저장기에 넘긴다: 입력(필터 기준·단계별 개수·탈락 표본) + 출력(추천 종목과 각 종목의 **분석 기록 ref**).
```
echo '<조립한 JSON>' | python3 .claude/skills/recommend-stocks/scripts/save_run.py
```
- 각 `picks[].analysis`에는 3단계에서 받은 분석 기록 ref를 넣는다. **진입/목표/손절은 여기 적지 않는다** — save_run.py가 ref의 분석 기록을 역참조해 본문 표에 끌어온다(진실원천 중복 방지).
- `data/recommendations/YYYY-MM-DD-run.md`에 frontmatter(정량)+본문(근거)으로 저장. 같은 날 재실행 시 `-2`,`-3`… 으로 **과거 기록을 덮어쓰지 않는다.**
- status는 추천 기록이 아니라 **각 분석 기록**에 산다(open→hit_target|stopped|watching). 회고 스킬이 분석 기록 쪽을 갱신.
- JSON 스키마는 `scripts/save_run.py` 상단 docstring 참고.

## 원칙
- **출처 없는 수치는 싣지 않는다.** 못 구한 값은 "확인 불가"로 표기.
- **안정성 우선**: 정렬·추천 우선순위는 항상 안정성 점수. 상승 기대치로 뒤집지 않는다.
- **깔때기 유지**: 무거운 analyze-company는 ~10개로 좁힌 뒤에만 실행한다.
- 이 스킬은 분석·조언까지. **실주문은 사람**(OS 가드레일).
