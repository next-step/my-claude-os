## DoD 체크리스트 (OS.md 기준)
### 1단계 — 코드베이스·컨벤션 파악
- [ ] CONVENTIONS 문서가 현재 코드와 일치
- [ ] 재사용 가능한 기존 자산 식별됨
### 2단계 — 분석·개발·테스트 작성
- [ ] 모든 요구사항이 코드로 구현됨
- [ ] 각 요구사항에 대응하는 단위·통합 테스트 존재
- [ ] 기존 자산 재사용(또는 불가 사유 명확)
### 3단계 — 검증 루프
- [ ] 모든 단위·통합 테스트 green
- [ ] skip/가짜 통과 테스트 없음
### 4단계 — 문서화
- [ ] 새 기능/변경점이 docs에 반영
- [ ] API 변경이 HTTP 문서에 반영
