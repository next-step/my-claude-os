# Claude OS — 할일 리마인더 시스템

할일이 많을 때 제때 완료하지 못하는 현대인을 위한 자동 리마인더 OS.
빠른 캡처 → AI 자동 분류 → 구체화 3단계로 마찰을 최소화한다.

---

## 핵심 원칙

**캡처와 구체화를 분리한다.**
기존 앱은 등록 시점에 너무 많은 입력을 요구해서 귀찮아 안 쓰게 된다.
이 시스템은 키워드 하나만 던져두고, 구체화는 나중에 별도 시간을 잡는다.

```
[1단계: 캡처]      [2단계: 분류]        [3단계: 구체화]
키워드 입력   →   AI 카테고리 자동 배정  →  /plan 실행 시 상세 입력
(3초 완료)        (애매하면 한 번만 질문)   (언제 / 왜 / 어떻게)
```

---

## 카테고리

| 카테고리 | 예시 키워드 |
|---------|------------|
| 📚 스터디 | 강의, 책, 자격증, 세미나 |
| 💼 업무   | 미팅, 리포트, 데드라인, 리뷰 |
| 🏠 일상   | 장보기, 청소, 약속, 취미 |
| 🏥 건강   | 병원, 운동, 약, 검진 |
| 💰 금융   | 납부, 이체, 세금, 보험 |
| 🔧 기타   | 위 카테고리에 해당하지 않는 것 |

## 리마인더 흐름

```
[매일 저녁 고정 시간]
  CronCreate 트리거
       ↓
  미완료(draft) 항목 있음?
  ├── Yes → "📌 구체화 안 된 항목 X개 있어요" 알럿 (강제 수행 X)
  │          2일 이상 초과 항목 있으면 → "⚠️ 'OOO' N일째 미완료, 지금 작성해요" 별도 강조
  └── No  → 조용히 종료

[사용자가 /plan 실행]
  → 미완료 항목을 오래된 순으로 꺼내 구체화 인터뷰 시작
```

**규칙 요약**
- 매일 저녁: 알럿만 발송, 강제 수행 없음
- `/plan` 실행 시: 그때 구체화 진행
- 2일 이상 초과 항목: 별도 경고 메시지로 강하게 리마인드

## 저장소: Notion DB

iPad · MacBook · iPhone 크로스 디바이스 지원을 위해 Notion을 사용한다.

```
[Claude (MacBook)]  →  Notion API  →  Notion DB
                                          ↓
                                   iPad 앱 / iPhone 앱 / 웹
```

### Notion DB 스키마

| 필드 | 타입 | 설명 |
|------|------|------|
| `title` | text | 입력한 키워드 |
| `category` | select | 스터디/업무/일상/건강/금융/기타 |
| `status` | select | `draft` / `planned` / `done` |
| `captured_at` | date | 최초 입력 시각 |
| `due_date` | date | 대략적 날짜 (미정 가능) |
| `detail` | text | 구체화 후 채워지는 내용 |

---

## 스킬 구성

| 스킬 | 역할 |
|------|------|
| `/capture` | 키워드 입력 → AI 분류 → Notion draft 저장 |
| `/plan` | draft 항목을 꺼내 구체화 인터뷰 진행 |
| `/done` | 미완료 항목을 골라 done 상태로 완료 처리 |
| `/remind` | Cron이 매일 호출하는 내부 알럿 스킬 |
