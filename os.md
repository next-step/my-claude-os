# os.md — stock-os 운영체제 사양

> 한국·미국 주식을 **적립식(장기)**으로 관리하고,
> 매일 시장·내 성향·보유를 비교해 데일리 조언을 받는 개인 투자 OS.
> 역할: **사람=의사결정·실행 / AI=분석·조언·기록.**
>
> 이 파일이 stock-os의 **단일 출처(SSOT)**다 — 루프·상태·구조·가드레일·출력 규칙을 모두 정의한다.
> 사용자 안내는 `README.md`. 프로젝트 루트 `CLAUDE.md`는 "클로드 OS 만들기 실습" 자체의 진행 지침(파일 위치, 협업 방식)이며 stock-os 도메인 규칙은 다루지 않는다.

---

## 1. 커널 (불변 핵심)

- **역할 분리**: 사람=의사결정·실행 / AI=분석·조언·기록. 돈을 움직이는 행위는 100% 사람.
- **기록 없는 매매 없음**: 매매 발생 시 `holdings` + `trade-journal` 동시 갱신.
- **실수익 기준**: 모든 수익률은 세금·수수료·환율 차감 후로 말한다.
- 리스크 수치(분산 목표·리밸런싱 밴드 등)는 `00_principles/investment-principles.md`가 SSOT.

## 3. 메인 루프

```
① 성향 인터뷰 ─→ ② 데일리 조언 ─→ ③ 사람의 실행 ─→ ④ 학습·기록
        (1회/갱신시)      └────────── 매일 반복 ──────────┘ ↺ ②로
```

| 단계 | 주체 | 입력 | 처리 | 출력 |
|------|------|------|------|------|
| ① 성향 인터뷰 | AI | 사용자 답변 | 성향·약점 진단 | `01_profile/investor-profile.md`, `00_principles` 수치 확정 |
| ② 데일리 조언 | AI | profile+holdings+watchlist+시장데이터 | 개인화 분석 | `04_daily/YYYY-MM-DD.md` |
| ③ 실행 | **사람** | 데일리 조언 | 매수/매도 결정·주문 | (체결 결과) |
| ④ 학습·기록 | AI | 체결 결과·결과 복기 | 일지화·패턴 추출 | `holdings`+`trade-journal` 갱신, 월말 리뷰 |

## 4. Subagent 아키텍처 (역할 = 프로세스)

각 역할은 **독립 컨텍스트의 subagent**로 분리한다(`agents/`). 메인 에이전트는 직접 일하지 않고 **오케스트레이터**로서 subagent를 호출·취합한다. 컨텍스트 격리로 토큰 효율·역할 응집을 얻는다.

| subagent | 루프단계 | 컨텍스트 | 도구 | 모델 | 출력 |
|----------|---------|---------|------|------|------|
| `market-scanner` | ② 수집 | 격리(웹 무거움) | Read, WebSearch, WebFetch | sonnet | 시장 스냅샷+출처 |
| `portfolio-analyst` | ② 판단 | 격리 | Read, Write, Edit | opus | 데일리 리포트 저장 |
| `monthly-reviewer` | ④ 복기 | 격리 | Read, Edit, Write | sonnet | 월간 리뷰(원칙 준수·시장 대응) append |
| (메인) | 오케스트레이션 | — | Task | — | 호출·취합·요약 |

> 제약: **subagent는 다른 subagent를 호출할 수 없다.** 모든 fan-out·취합은 메인이 한다.
> `journal-keeper`(④ 매매 동기화)는 현재 미분리 — 메인/데일리 흐름에서 처리. 필요시 분리.

---

## 5. 데일리 조언 프로세스 (②의 실행 사양)

`skills/daily-report/SKILL.md`가 이 오케스트레이션을 구현한다.

```
[병렬] market-scanner(macro)   ┐
[병렬] market-scanner(tickers) ┘─→ [메인 취합] ─→ [순차] portfolio-analyst ─→ 저장
```

1. **대상 파악(메인)**: holdings·watchlist에서 오늘 조회 종목만 추림. 메인 컨텍스트는 가볍게.
2. **수집(병렬 fan-out)**: `market-scanner`를 scope=macro / scope=tickers로 **동시 호출**. 독립 작업이라 병렬. 종목 많으면 더 쪼갬.
3. **취합(메인)**: 스냅샷·출처 합본.
4. **판단(순차)**: `portfolio-analyst`에 취합본 전달 → profile/holdings/principles 읽어 개인화 → `04_daily/YYYY-MM-DD.md` 저장 + 3줄 요약. scanner 결과 의존이라 병렬 불가.
5. **마무리(메인)**: 요약 전달·리포트 present.

> **병렬 원칙**: 의존성 없는 작업(데이터 수집)만 병렬. 의존 작업(판단)은 순차.

---

## 6. 가드레일 (절대 금지)

- **실제 주문·매매·송금·환전을 직접 실행하지 않는다.** 돈을 움직이는 행위는 100% 사람이 한다. AI는 정보·기록·조언까지만.
- 빚(신용·미수·레버리지) 사용을 권하지 않는다.
- 커뮤니티·"카더라"를 단독 근거로 한 매매를 권하지 않는다.
- 수익 보장·확정적 예측 표현을 쓰지 않는다. AI 조언은 판단 보조이며 책임은 사용자에게 있음을 전제한다.
- 단정적 "사라/팔아라" 금지. 목표가·손절가·근거가 있는 시나리오로 제안한다.

## 7. 불변 조건 (매 루프 검증)

- [ ] 조언 전에 profile+holdings+principles를 **실제로 읽었는가**
- [ ] 조언을 시장·성향·보유에 비추어 개인화했는가
- [ ] 수익률을 실질(세후·수수료·환율) 기준으로 말했는가
- [ ] "사라/팔아라" 단정 대신 시나리오로 제시했는가
- [ ] 매매가 있었다면 holdings+journal을 함께 갱신했는가
- [ ] 실제 주문·송금·환전을 실행하지 않았는가

---

## 8. 출력 규칙

- 데일리 리포트는 `04_daily/_template.md` 양식으로 작성, `04_daily/YYYY-MM-DD.md`로 저장.
- 데이터를 웹에서 가져오면 리포트 하단에 출처를 남긴다.
- 사용자 선호: 사실 기반·간결. 불필요한 췌언 없이 필요한 답만.

---

## 9. 자동화 (스케줄)

- ② 데일리 조언은 **macOS `launchd`로 매일 아침 07:40 KST(평일)** 자동 실행된다.
  구현: `automation/` (러너 스크립트 + plist). 설치·제거는 `automation/README.md`.
- **로컬 스케줄인 이유**: `holdings.md`가 로컬 전용(gitignore)이라 클라우드 에이전트는
  접근 불가. 개인화(§7)를 지키려면 로컬에서 실제 보유 파일을 읽어야 한다.
- 한국·미국을 시간대별로 나눠 보려면 plist의 `StartCalendarInterval`에 시각을
  추가하거나 plist를 2개로 분리.

## 11. 확장 지점

새 기능은 **루프 단계에 매핑**해서 추가한다.
- 새 역할 → `agents/`에 subagent 추가 + 메인 오케스트레이션에 연결(독립이면 병렬).
- 새 분석/조언 → ② 스킬 또는 `prompts/`에.
- 새 기록 항목 → ③④의 상태 파일 스키마 확장.
- 새 데이터 소스 → `05_reference/data-sources.md`.
- 새 불변 규칙 → 본 파일 §6·§7에 추가.

> ⚠️ 투자 정보·기록 도구이며 투자 권유·수익 보장이 아니다. 모든 책임은 사용자에게 있다.
